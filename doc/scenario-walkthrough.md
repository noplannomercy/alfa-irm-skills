# 대본형 시나리오 — 성진오토텍 풀사이클

> 실제 상황을 가정한 대본. 각 단계에서 뭘 넣고 뭐가 나오는지 구체적으로 보여줌.
> 전체 산출물: `samples/test-제조사/` 참고

---

## 배경

영업팀 슬랙: "성진오토텍이라고 자동차 부품 제조사인데, 품질관리 쪽에서 AI 도입 미팅 잡혔습니다"

---

## 1단계: 미팅 전 준비

```
[상황] 영업팀에서 고객 정보 전달받음. 아는 거 거의 없음.
[나]   /alfa-prep 실행
[입력] "성진오토텍, 자동차 부품 제조, 800명, 품질관리 AI 도입 검토, CTO 관심"
[결과] 질문 22개 + 사전요청서 + 프로필 초안 (🟡2개, ⬜4개)
[행동] 사전요청서를 고객 담당자에게 이메일 발송
       "미팅 전에 이것 좀 준비해주시면 시간 절약됩니다"
```

**포인트:** 업종이 "제조"라서 자동으로 온프레미스 분기 질문이 포함됨 ("데이터 외부 반출 가능?", "불량코드 명칭 통일?")

---

## 2단계: 1차 미팅

```
[상황] 고객사 방문. CTO + 품질관리팀장 참석. 전반적 현황 파악.
[미팅] 브레이크 디스크/서스펜션 제조, 현대기아 1차 벤더
       MES(미라콤) + SAP, 온프레미스, GPU 없음
       "문서가 너무 많고 불량 원인 찾는 데 시간이 너무 걸린다"
       "같은 불량인데 화성이랑 아산이 다르게 기록해서 분석이 안 된다" ← GraphRAG 신호!

[나]   /alfa-live 실행 (미팅 메모 붙여넣기)
[결과] 프로필 업데이트 (🟢3개, 🟡3개)
       신호 감지: #GraphRAG (용어 불일치), #on-premise (데이터 반출 금지)

[나]   /alfa-wrap 실행
[결과] 1st_summary.md (고객용) + 1st_internal.md (내부용) + Phase 문서 초안
[행동] 1st_summary.md를 고객에게 이메일 회신
       "이렇게 파악했습니다. 다음 미팅에서 구체적으로 어떤 업무를 먼저 해볼지 좁혀봅시다"
```

---

## 3단계: 2차 미팅

```
[상황] 품질관리팀장이 후보 3개 가져옴. IT과장도 동석.
[미팅] 후보: 불량 원인 분석 / 검사기준서 검색 / 협력사 품질 평가
       → "불량 원인 분석 보조"로 확정 (한 건에 반나절 걸림, 효과 가시적)
       MES ODBC 직접 조회 가능, 문서는 파일서버
       파일럿 3천만, 8월 말 마감, IT 1명 파트타임

[나]   /alfa-live 실행
[결과] 프로필 업데이트 (🟢5개, 🟡1개) ← 유즈케이스 확정됨!

[나]   /alfa-wrap 실행
[결과] 2nd_summary.md + 유즈케이스 정의서 + IRM 투입 판단(조건부)
[행동] 2nd_summary.md 고객 회신
       "보안 관련 확인 필요해서 3차 미팅 때 보안담당도 동석 부탁드립니다"
```

---

## 4단계: 3차 미팅 (규제환경 확정)

```
[상황] 보안담당(경영지원팀) 동석. OEM 보안 요구사항 확인.
[미팅] 품질 데이터 "대외비", 외부 전송 금지, 보안검토 2주
       감사로그 필요, 로컬 LLM이면 학습 데이터 이슈 없음
       불량코드 매핑 엑셀 수령 — 200개 중 30% 불일치 확인
       GPU RTX 4090 x2 품의 승인됨

[나]   /alfa-live 실행
[결과] 프로필 완전체 (🟢6개)

[나]   /alfa-wrap 실행
[결과] 3rd_summary.md + Phase 2 GOVERN 확정 + ALFA 입력 프로필(확정)
       IRM 투입 판단: "가능"으로 갱신
[행동] 3rd_summary.md 고객 회신
       "설계 작업 들어가겠습니다. 파일럿 상세 제안서는 2주 내 드리겠습니다"
```

**ALFA 단계 완료. 산출물 15개 파일.**

## 5단계: IRM Phase 0 — 리소스 분류

```
[상황] 유즈케이스 확정됨. 이제 기술 설계 시작.
[나]   /irm-init 실행
[결과] 리소스 6건 (✅4 / ⚠️2)
       행위 유형: 탐색 50% / 접근 25% / 실행(유연) 25%
       질의 13건, 체이닝 3패턴
       GraphRAG 사전 판단: 후보 (3/4 충족 가능성 높음)
```

## 6단계: IRM Phase 1 — 설계 3종

```
[나]   /irm-search 실행
[결과] 컬렉션 3개 (quality_standards / defect_history / investigation_reports)
       pgvector + HNSW, BGE-m3-ko (로컬), 하이브리드 검색
       GraphRAG 승인 (3/4) — pgvector + Apache AGE

[나]   /irm-access 실행
[결과] CLI 2개 (cli-mes, cli-defectcode) + FastAPI 게이트웨이
       둘 다 ★★★ 우선순위 (체이닝 시작점)

[나]   /irm-exec 실행
[결과] 실행(유연) 2개 — 원인 분석 보조(7단계) + 8D 보고서 생성(3단계)
       체이닝: 접근→탐색→실행 6단계 메인 플로우
       예상 효과: 반나절 → 15~25분 (80% 단축)

[나]   /irm-design 실행
[결과] 종합: 12주, 3천만원 적합
       크리티컬 패스: GPU 입고 → HWP 파싱 → 8D 보고서 위치 확인
```

## 7단계: IRM Phase 2 — 구현 설계

```
[나]   /irm-validate 실행
[결과] 체크리스트 3개 (탐색/접근/실행)
       검증 기준: 원인 추천 적중률 70%, 소요 시간 30분 이내
       구현 계획: Phase A~G, 병렬화 포함
       PoC: 화성공장 2주, 엔지니어 3~5명

[행동] 이 결과를 기반으로 "파일럿 상세 제안서" 작성
```

**IRM 완료. 산출물 추가 11개 파일. 총 26개.**
