# 유즈케이스 정의서: CT 판독문 보조

> 고객: 병원그룹
> 작성일: 2026-03-25
> 상태: 확정 (PoC 대상)

## 개요
- 목적: 영상의학과 의사의 CT 판독문 작성/검색을 AI로 보조
- 사용자: 영상의학과 의사, 방사선사
- 입력: CT 영상 판독문 (텍스트), 과거 판독 이력, 소견 키워드
- 출력: 유사 판독 사례 검색, 판독문 초안 보조, 용어 표준화 제안

## 행위 유형 분포 (예상)

| 행위 유형 | 비중 | 설명 |
|----------|------|------|
| 탐색 | 50% | 과거 유사 판독문 검색, 소견별 사례 탐색 ("이 소견과 비슷한 판독문 찾아줘") |
| 접근 | 20% | PACS/EMR에서 특정 환자의 과거 판독 이력 조회 |
| 실행(유연) | 25% | 판독문 초안 생성, 소견 요약, 용어 표준화 제안 |
| 실행(확정) | 5% | 판독문 서식 자동 채움 (환자 정보, 검사 정보 등 정형 필드) |

## 리소스 예상

| 리소스 유형 | 예시 | 규모 |
|------------|------|------|
| 문서/파일 | CT 판독문 (과거 데이터) | 약 12만건/년 × 수년치 |
| 시스템/DB | PACS (Infinitt, FHIR), EMR (자체개발) | 2개 시스템 연동 |
| 절차/매뉴얼 | 판독 가이드라인, 소견 기술 표준 | 진료과별 존재 여부 확인 필요 |
| 용어 사전 | 의학용어 동의어/약어 매핑 | 자체 구축 필요 (GraphRAG) |

## GraphRAG 필요성
- **필요** (강함)
- 근거:
  - "같은 소견인데 의사마다 표현이 달라서 검색이 안 돼요" (1차 미팅)
  - "의학용어를 못 알아들어서 엉뚱한 답변이 나왔어요" (2차 미팅, 이전 PoC 실패 원인)
- 적용 방향:
  - 의료 용어 동의어 그래프 (예: "간 종괴" = "hepatic mass" = "liver mass" = "간 SOL")
  - 의사별 표현 패턴 → 표준 소견 코드 매핑
  - 약어 확장 (예: "HCC" → "간세포암종")

## 핵심 성공 기준
1. 의료 용어 검색 정확도 — 동의어/유사 표현으로 검색해도 정확한 결과 반환
2. 판독문 초안 품질 — 의사가 수정 없이 또는 최소 수정으로 사용 가능
3. 응답 속도 — 판독 워크플로우를 방해하지 않는 수준 (수 초 이내)

## 참고
- PoC 범위: 본원 CT 판독문만 (분원, MRI 제외)
- 확장 경로: CT → MRI → 간호기록 요약 → (장기) 환자 설명용
- 이전 실패 교훈: 범용 LLM으로는 의료 용어 처리 불가 → 도메인 특화(GraphRAG + Fine-tuning 검토) 필수
