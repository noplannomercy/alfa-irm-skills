## 구현 순서 (의존성 기반)

```
Phase A: 인프라 + 선행 (2주) [IRB/데이터 준비와 병렬]
  ├─ A100 환경 세팅 (로컬 LLM 배포)
  ├─ Milvus/Qdrant 설치
  ├─ Neo4j Community 설치
  ├─ FastAPI 기본 구조
  └─ 가명처리 파이프라인 구현 (데이터 준비에 필수)

Phase B: 탐색 (3주) [Phase A 완료 후]
  ├─ 판독문 파싱 + 임베딩 파이프라인
  ├─ ct_reports 컬렉션 인덱싱 (3년치)
  ├─ 하이브리드 검색 (벡터+BM25) 구현
  ├─ 리랭킹 적용
  ├─ GraphRAG 용어 그래프 초기 구축 (영상의학과 감수)
  └─ 질의 확장 로직 구현

Phase C: 접근 (2주) [Phase A 완료 후, Phase B와 병렬 가능]
  ├─ PACS FHIR CLI 구현 + 테스트
  ├─ EMR CLI 구현 + 테스트
  ├─ 가이드라인 CLI 구현
  ├─ REST 엔드포인트 래핑
  └─ 감사로그 파이프라인 구현

Phase D: 실행 (2주) [Phase B + C 완료 후]
  ├─ 판독문 초안 생성 (LLM 프롬프트 + 검색 연동)
  ├─ 용어 표준화 (GraphRAG + LLM)
  ├─ 소견 요약 (LLM)
  └─ Human-in-the-loop UI 연동

Phase E: 통합 + 검증 (1주) [Phase D 완료 후]
  ├─ 체이닝 통합 테스트 (5개 시나리오)
  ├─ 엔드투엔드 검증
  └─ 성능 테스트 (응답 시간, 동시 접속)

Phase F: 파일럿 (2주) [Phase E 완료 후]
  ├─ 영상의학과 의사 3~5명 실사용
  ├─ 피드백 수집 + 개선
  └─ 최종 평가 + 확장 판단
```
