# 병원그룹 고객 프로필

> 작성일: 2026-03-25
> 상태: 2차 미팅 완료 — 프로필 거의 완성

## 1. 회사 개요
- 업종: 의료 (병원 그룹)
- 규모: 중견 — 서울 본원 500병상 + 지방 분원 3개 (각 150~200병상) [2차 미팅 확인]
- 주력 사업: 병원 운영
- 연간 영상 촬영: 약 12만건 (본원 기준) [2차 미팅 확인]

## 2. AI/LLM 현황
- 기존 시도: 작년 외부 업체 챗봇 PoC → 정확도 낮아 중단 [2차 미팅 확인]
- 실패 원인: "의학용어를 못 알아들어서 엉뚱한 답변" → 도메인 특화 부족 #poc-history [2차 미팅 확인]
- 남은 자산: 미확인 (챗봇 PoC에서 남은 데이터/모델 유무 확인 필요)
- 현재 관심: 영상의학과 판독 보조 1순위, 간호기록 요약 2순위 [1차 미팅 확인]
- GPU: 연구용 A100 2장 보유, 주중 업무시간 우선 사용 합의 완료 [2차 미팅 확인]

## 3. 인프라
- 클라우드/온프레미스: **온프레미스 전용** — 데이터 외부 반출 절대 불가 [1차 미팅 확인]
- EMR: **자체개발**, 온프레미스 서버 운영 [1차 미팅 확인]
- PACS: **Infinitt** 제품, 본원/분원 동일, **HL7 FHIR 연동 가능** [2차 미팅 확인]
- GPU: A100 2장 (연구팀 공유, 주중 업무시간 우선 사용) [2차 미팅 확인]
- 네트워크/보안: 미확인 (본원-분원 연동 구조 상세 미확인, 단 PoC는 본원만)
- 접근권한: 기존 EMR 권한체계 따름, PACS 읽기전용 연동 [2차 미팅 확인]
- 감사로그: 기존 EMR 감사로그에 AI 조회 이력 추가 [2차 미팅 확인]
- 데이터 보관: 원내 서버, 학습용 데이터셋은 별도 격리 스토리지 [2차 미팅 확인]

## 4. 규제 환경
- 감독기관: 보건복지부, 식약처 (현재 내부 보조수단으로 판단 → 식약처 인허가 불필요)
- 컴플라이언스: 의료정보보안팀 3명 별도 운영, 개인정보보호책임자는 행정부원장 [2차 미팅 확인]
- 데이터 분류: 개인건강정보(PHI) — 가명처리 후 LLM 투입 가능 (내부 검토 완료) [2차 미팅 확인]
- 가명처리 기준: 보건복지부 가이드라인, 이름/생년월일/주민번호 마스킹 [2차 미팅 확인]
- IRB: **간편심의 가능** — 후향적 데이터, 2주 통과 예상 [2차 미팅 확인]
- 보조수단 판정: "AI 결과를 의사가 최종 확인하는 구조면 보조수단" (의료정보보안팀장) [2차 미팅 확인]
- 의사결정 기구: 기존 의료정보위원회에서 AI 안건 심의 [2차 미팅 확인]

## 5. 원하는 것
- 경영진 지시: CTO가 "AI 도입 검토하라" 지시 — **CTO가 스폰서** [1차 미팅 확인]
- 현업 니즈:
  - 1순위: 영상의학과 판독 보조 — **CT 판독문 먼저, 잘 되면 MRI 확장** [2차 미팅 확인]
  - 2순위: 간호기록 요약 [1차 미팅 확인]
- 유즈케이스: CT 판독문 보조 (PoC 범위)
- 사용자: **의사/간호사만** — 환자 직접 노출 당분간 없음 [1차 미팅 확인]
- 향후 희망: "나중에 환자 설명용으로도 쓸 수 있으면 좋겠다" (장기) [2차 미팅 확인]
- RACI: 실행=의료정보팀, 검토=영상의학과, 승인=의료정보위원회, 보고=CTO [2차 미팅 확인]

## 6. 제약사항
- 예산: **PoC 기준 5천만원 이내** [2차 미팅 확인]
- 일정: **4월 중순 PoC 시작 목표** — IRB 2주 + 데이터 준비 2주 감안 [2차 미팅 확인]
- 인력: 의료정보팀 3명 (인프라 1명 + 개발 2명) [2차 미팅 확인]
- 고객 대면 여부: **내부 전용** (의사/간호사) [1차 미팅 확인]
- PoC 범위: **본원 데이터만** — 분원은 1차에서 제외 [2차 미팅 확인]

## 내부 메모
- 거버넌스 무게: **Medium** — 내부 전용 + 보조수단 판정 + 가명처리 프로세스 확립
- GraphRAG 신호: **강함** — 1차 "의사마다 표현이 달라서" + 2차 "의학용어를 못 알아들어서" 이중 확인 #GraphRAG
- 온프레미스 확정: 로컬 LLM 필수 #on-premise
- PoC 실패 이력: 작년 챗봇 PoC 실패 — 도메인 특화 부족이 원인 #poc-history → 이번엔 의료 용어 특화가 핵심 차별점
- 2단계 신호: "나중에 환자 설명용으로도" → 장기적 2단계 접근 인지 #2-phase
- 스폰서: CTO 확정, 예산 5천만원 확보
- GPU: A100 2장, 연구팀과 시간 분할 합의 완료
- 예상 로드맵: 온프레미스 3개월 — IRB 2주 → 데이터 준비 2주 → PoC 개발 8주
- PACS HL7 FHIR 연동 가능 → 기술적 리스크 낮음
- 핵심 성공 요인: 의료 용어 특화 (이전 PoC 실패 원인 해소)
