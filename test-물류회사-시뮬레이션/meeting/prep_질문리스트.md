# 물류회사 미팅 질문 리스트

## 미팅 기본 정보
- 예상 소요: 60분
- 거버넌스 무게: Light~Medium (현재 판단) — 상담사 보조면 Light, 고객 직접 노출이면 Medium
- 2차 미팅 필요 예상: 아니오 (규제 light 업종, 단 고객 대면 시 재검토)

## 반드시 물어볼 것 (프로필 빈칸)

### ⬜ AI/LLM 현황
- Q1. "혹시 이전에 AI나 챗봇 관련해서 시도해본 적 있으세요?"
- Q2. "CS 자동화 솔루션을 검토하거나 도입한 적 있나요?"
- (왜 물어보는지): 기존 시도가 있으면 실패 원인 파악 → 이번 PoC 차별점 설계에 핵심

### ⬜ 인프라
- Q1. "현재 CS 상담 시스템이 뭘 쓰시나요? (자체개발? 솔루션?)"
- Q2. "배송 추적 시스템은 자체인가요, 택배사 API 연동인가요?"
- Q3. "클라우드 쓰시나요? AWS, Azure 등?"
- Q4. "CS 상담 이력이 DB에 쌓여있나요? 어느 정도 기간치?"
- (왜 물어보는지): 인프라 경로(클라우드/온프레미스)와 데이터 접근성 결정

### ⬜ 규제 환경
- Q1. "CS 데이터에 고객 개인정보(이름, 전화번호, 주소)가 포함되어 있나요?"
- Q2. "개인정보 처리 관련 내부 정책이 있나요?"
- (왜 물어보는지): 개인정보 포함 시 거버넌스 무게 격상, 비식별화 필요

### ⬜ 제약사항
- Q1. "이 프로젝트에 예산이 잡혀있나요? 대략 어느 수준?"
- Q2. "일정 기대가 있으신가요? 언제까지 뭔가 보고 싶으신지?"
- Q3. "IT팀에서 투입 가능한 인력이 몇 명인가요?"
- (왜 물어보는지): 예산/일정/인력에 따라 PoC 범위 결정

## 업종 특화 질문

### 물류/배송에서 꼭 확인할 것
- Q. "CS 문의 유형 중 가장 많은 게 뭔가요? (배송 조회? 반품? 클레임?)"
  → 유즈케이스 우선순위 결정
- Q. "상담사가 답변할 때 참고하는 매뉴얼이나 FAQ가 있나요?"
  → RAG 소스 파악
- Q. "물류센터별로 처리 절차가 다른 부분이 있나요?"
  → GraphRAG 필요 신호 (센터별 용어/절차 차이)
- Q. "택배사가 여러 곳인가요? 각각 시스템이 다른가요?"
  → 접근(Access) 복잡도 파악
- (놓치면 안 되는 이유): 물류 CS는 배송 추적 + FAQ + 절차 처리가 혼재, 유형 파악이 아키텍처 결정

## 초반에 반드시 확인 (거버넌스 무게 결정)
- Q. "이 AI를 상담사가 참고용으로 쓰나요, 고객한테 직접 답변하나요?"
  → 고객 대면 = 거버넌스 Medium+, 상담사 보조 = Light
- Q. "오답이 나갔을 때 책임 소재가 중요한 상황인가요?"
  → 고객 대면 시 오답 리스크 확인

## 미팅 중 감지할 신호
- "센터마다 절차가 달라서..." / "택배사마다 용어가 다르고" → GraphRAG 필요 신호
- "고객한테 직접 챗봇으로..." → 2단계 접근 필요 (상담사 보조 먼저)
- "클라우드 안 되고 사내망에서만..." → 온프레미스 분기
- "전에 챗봇 넣었는데 정확도가..." → PoC 실패 이력, 실패 원인 파악 필수
- "대표님이 빨리 하라고..." → 일정 압박, 경량 로드맵으로
- "개인정보가 많아서..." → 비식별화 필요, 거버넌스 격상
