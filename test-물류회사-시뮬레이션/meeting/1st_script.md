> 참석자: IT팀장, CS운영팀장

## 미팅 내용 (원문)

- 회사명은 "한진로지스", 연매출 3000억 규모
- CS 시스템은 자체개발 웹 기반, MySQL DB에 상담 이력 3년치 쌓여있음
- AWS 쓰고 있음, EC2 + RDS 구성
- 택배사 3곳 연동 (CJ대한통운, 한진, 롯데), 각각 API 다름
- "택배사마다 배송 상태 코드가 달라서 상담사들이 헷갈려요"
- CS 문의 유형: 배송 조회 40%, 반품/교환 25%, 클레임 20%, 기타 15%
- 상담사 매뉴얼 있음, FAQ 300개 + 반품절차서 + 클레임 처리 가이드
- "지금 상담사가 매뉴얼 찾는 데만 시간의 30%를 쓰고 있어요"
- 이전에 외부 챗봇 업체 도입했다가 3개월 만에 철수. "FAQ만 달랑 넣으니까 맥락을 못 잡아서"
- 고객 직접 노출은 아직 무서워서, 우선 상담사 보조 도구로 시작하고 싶음
- 개인정보: CS 데이터에 고객 이름/전화번호/주소 포함. 현재 접근제어는 있지만 비식별화 안 함
- IT팀 3명 투입 가능 (백엔드 2명 + 인프라 1명)
- 예산은 상반기 내 3천만원, 대표이사가 빠른 성과 원함
- "6월까지 뭔가 보여달라"
- "나중에 고객 셀프서비스 챗봇도 하고 싶은데, 그건 다음 단계로"
