# CS 상담 보조 — 실행 구현 체크리스트

> 작성일: 2026-03-26

## 실행(확정) 파이프라인

### 반품 판단 파이프라인
- [ ] Step 정의 완료 (주문확인→절차조회→배송확인→판단→안내)
- [ ] 반품 가능 규칙 정의 (기간, 배송 상태, 상품 유형)
- [ ] 각 Step 성공/실패 경계 정의
- [ ] JSON 입출력 스키마
- [ ] 단위 테스트 (가능/불가 각 5건)

### 개인정보 마스킹 파이프라인
- [ ] 정규식 패턴 정의 (이름, 전화번호, 주소)
- [ ] 마스킹 규칙 (홍길동→홍**, 010-1234-5678→010-****-5678)
- [ ] LLM 투입 전 자동 적용 훅
- [ ] 탐지율 테스트 (100건 샘플)

## 실행(유연) 프로세스

### 클레임 처리 안내
- [ ] LLM 프롬프트 설계 (클레임 유형 판단 + 보상 기준)
- [ ] 클레임 유형 목록 정의 (파손, 분실, 지연, 오배송 등)
- [ ] Human-in-the-loop UI (상담사 승인/수정)
- [ ] 피드백 루프 (상담사 수정 내용 학습)

### 답변 초안 생성
- [ ] LLM 프롬프트 설계 (검색 결과 + 문의 맥락 → 답변)
- [ ] 톤/스타일 가이드 (고객 대응 톤)
- [ ] Human-in-the-loop UI (상담사 검토 → 전송)
