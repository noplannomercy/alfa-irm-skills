# CS 상담 보조 — 검증 기준

> 작성일: 2026-03-26

## 행위 유형별 검증

### 탐색 검증
| 항목 | 기준 | 측정 방법 |
|------|------|----------|
| FAQ 검색 정확도 | Recall@5 ≥85% | CS운영팀 선정 테스트 질의 30개 |
| 응답 시간 | ≤2초 | 검색~결과 반환, P95 기준 |
| 컬렉션 커버리지 | FAQ 300건 100% | 인덱싱 건수 확인 |

### 접근 검증
| 항목 | 기준 | 측정 방법 |
|------|------|----------|
| 배송 조회 성공률 | 택배사별 ≥99% | 각 택배사 20건 = 60건 |
| 상태 코드 매핑 정확도 | 100% | 매핑 테이블 전수 검증 |
| CS 이력 조회 | 정상 동작 100% | 명령어별 10건 = 30건 |
| 개인정보 마스킹 | 미탐지 0건 | 개인정보 포함 샘플 50건 |
| REST 응답 시간 | ≤1초 | P95 기준 |

### 실행 검증
| 항목 | 기준 | 측정 방법 |
|------|------|----------|
| 반품 판단 정확도 | ≥95% | 가능/불가 케이스 각 10건 |
| 답변 초안 수용률 | ≥50% (PoC 목표) | 상담사가 수정 없이 또는 최소 수정 비율 |
| 클레임 안내 적절성 | ≥60% | CS운영팀장 판정 |

## 엔드투엔드 검증

### 체이닝 시나리오
| # | 시나리오 | 체이닝 | 기대 결과 | 합격 기준 |
|---|---------|-------|----------|----------|
| 1 | "주문번호 123 배송 어디쯤?" | 접근 | 택배사 자동 감지 + 통합 상태 코드 반환 | 정확한 배송 현황 ≤2초 |
| 2 | "반품 하고 싶대요" | 탐색→접근→실행(확정) | 절차 조회 + 주문 확인 + 가능 판단 | 전체 체인 ≤5초 |
| 3 | "파손 클레임인데 어떻게 해?" | 접근→탐색→실행(유연) | 이력 조회 + 가이드 검색 + 보상 제안 | 제안 생성 + 상담사 UI |
| 4 | "배송 지연 안내 보내야 해" | 접근→탐색→실행(유연) | 현황 조회 + 문구 검색 + 초안 생성 | 초안 품질 상담사 승인 |

## PoC 범위
- 대상: 배송 조회 + FAQ 검색 (전체의 55~60%)
- 기간: 4주
- 참여자: 상담사 5명 (20명 중 파일럿)
- 성공 기준:
  - FAQ 검색 Recall@5 ≥85%
  - 배송 조회 성공률 ≥99%
  - 상담사 매뉴얼 검색 시간 30% → 15% 이하
  - 상담사 만족도 4/5 이상
- 확대 조건: 성공 시 → 전체 상담사 20명 확대 + 클레임/반품 기능 추가
