## 체이닝 패턴 (복합 질의)

- **"반품 접수 처리해줘"**
  = 탐색(반품 절차서 조회) → 접근(주문 정보 확인) → 실행(확정, 택배 회수 요청)

- **"이 클레임 어떻게 처리하면 돼?"**
  = 접근(상담 이력 조회) → 탐색(클레임 가이드 검색) → 실행(유연, LLM 판단 + 상담사 확인)

- **"고객한테 보낼 배송 지연 안내 만들어줘"**
  = 접근(배송 현황 조회) → 탐색(안내 문구 템플릿) → 실행(유연, 초안 생성)

## Phase 0 결론

### 행위 유형 분포 요약
- **주요 유형**: 탐색 33% + 접근 33% = 66% — 검색+조회 중심
- **아키텍처 시사**: 이커머스 CS와 매우 유사한 패턴. 단순 RAG + 시스템 CLI. 실행은 보조적.

- 동음이의어/용어 불일치: 있음 — 택배사별 상태 코드 ("배달중"="배송중"="이동중")
- 인과관계 질의 비율: 0/12 (0%) — 인과관계 질의 없음
- 동일 엔티티 다중 맥락: 약함 — 배송 상태만 해당
- → 사전 판단: **불필요** (1/4만 해당, 상태 코드 매핑은 코드 테이블로 해결 가능)
