# 유즈케이스 정의서: 불량 원인 분석 보조

> 고객: 성진오토텍
> 작성일: 2026-04-17
> 상태: 확정

## 개요
- 목적: 불량 발생 시 원인 분석 시간을 단축하고, 8D 보고서 초안을 자동 생성
- 사용자: 품질관리팀 품질 엔지니어 (화성공장 브레이크 디스크 라인)
- 입력: 불량 건 정보 (불량코드, 라인, 제품, 일시)
- 출력: 유사 불량 이력 + 관련 기준서 + 원인 분석 추천 + 8D 보고서 초안

## 현재 프로세스 (5단계)

| 단계 | 내용 | 현재 소요 | 병목 |
|------|------|----------|------|
| 1 | MES에서 불량 건 접수 | 즉시 | — |
| 2 | 과거 유사 불량 이력 검색 (MES 수동 조회) | 1~2시간 | **병목** |
| 3 | 검사기준서/작업표준서에서 해당 기준 확인 | 30분~1시간 | **병목** |
| 4 | 원인 분석 및 대책 수립 (엔지니어 판단) | 1~2시간 | — |
| 5 | 8D 보고서 작성 (한글 양식) | 30분~1시간 | — |
| | **합계** | **반나절 (3~6시간)** | |

## 행위 유형 분포 (예상)

| 행위 유형 | 비중 | 설명 |
|----------|------|------|
| 탐색(내부) | 40% | 유사 불량 이력 검색, 검사기준서/작업표준서 검색 |
| 접근 | 20% | MES 불량 건 조회, 불량 상세 데이터 조회 |
| 실행(유연) | 40% | 원인 분석 추천 (LLM 판단), 8D 보고서 초안 생성 |
| 실행(확정) | 0% | 해당 없음 (모든 판단이 상황 의존적) |

## 리소스 현황

| # | 리소스명 | 유형 | 근거 | 위치/접근 방법 | 규모 |
|---|---------|------|:----:|-------------|------|
| 1 | 불량 이력 | 시스템(MES) | ✅ | MES DB — ODBC 직접 조회 | 3년치 약 15만건 |
| 2 | 검사기준서 | 문서/파일 | ✅ | \\filesvr\품질관리\ — PDF | 약 200건 |
| 3 | 작업표준서 | 문서/파일 | ✅ | \\filesvr\품질관리\ — HWP | 약 300건 |
| 4 | 8D 보고서 양식 | 절차/매뉴얼 | ⚠️ | 한글(HWP) 양식 — 위치 미확인 | 양식 1건 + 과거 작성본 다수 추정 |
| 5 | 불량코드 체계 | 절차/매뉴얼 | ⚠️ | MES 내 코드 테이블 추정 — 화성/아산 상이 | 미확인 |

**리소스 신뢰도: ✅3건 (60%) / ⚠️2건 (40%)**

## GraphRAG 필요성
- **필요 (강한 신호)**
- 근거:
  - 공장별 불량코드 불일치: "B-crack" (화성) vs "브레이크디스크_균열" (아산)
  - 제품군별 용어 불일치: "피로파괴" (서스펜션) vs "크랙" (브레이크)
  - 동일 현상에 대한 다중 표현 → 엔티티 동의어 관계 필요

## 참고 유사 케이스
- 제조업 품질 분석은 "문서 검색 + 이력 조회 + 판단 보조" 패턴이 전형적
- 용어 불일치가 심하면 RAG만으로 부족, GraphRAG로 동의어/인과관계 연결 필요
