## 1. 회사 개요
- 업종: 자동차 부품 제조
- 규모: 직원 800명 (품질관리 인력 약 60명)
- 주력 사업: 브레이크 디스크, 서스펜션 부품 [1차 미팅 확인]
- 납품처: 현대기아 1차 벤더 [1차 미팅 확인]
- 사업장: 본사 + 화성공장 + 아산공장 [1차 미팅 확인]

## 2. AI/LLM 현황
- 기존 시도: 없음 (LLM 경험 전무) [1차 미팅 확인]
- 결과: —
- 남은 자산: 비전 검사(카메라 기반) 화성공장 운영 중 — 별도 장비, LLM과 무관 [1차 미팅 확인]

## 3. 인프라
- 클라우드/온프레미스: 온프레미스 only (사내 서버만) [1차 미팅 확인]
- DB: MES(미라콤) — ODBC 직접 조회 가능, REST API 없음 [2차 미팅 확인]
- ERP: SAP [1차 미팅 확인]
- GPU: RTX 4090 x2 — 품의 승인, 2주 내 입고 예정 [3차 미팅 확인]
- 파일서버: \\filesvr\품질관리\ [2차 미팅 확인]
- 네트워크/보안: 현대기아 보안 요구사항으로 데이터 외부 반출 금지 [1차/3차 미팅 확인]

## 4. 규제 환경
- 감독기관: 직접 규제 없음 — OEM(현대기아) 요구사항이 사실상 규제 [1차 미팅 확인]
- 컴플라이언스팀: 별도 팀 없음. 경영지원팀 최유진이 보안/컴플라이언스 겸임 [3차 미팅 확인]
- 보안 가이드라인: 현대기아 협력사 정보보안 가이드라인 적용 중 [3차 미팅 확인]
- 데이터 분류: 품질 데이터 "대외비" 등급, 개인정보 없음, 외부 클라우드 전송 금지 [3차 미팅 확인]
- 시스템 도입 절차: 보안검토서(IT팀 작성) → 경영지원팀 승인 (2주 소요) [3차 미팅 확인]
- AI 관련 요구: 학습에 데이터 미사용 보장 필요 → 로컬 LLM이면 충족 [3차 미팅 확인]
- 감사로그: 데이터 조회 이력 기록 필요 [3차 미팅 확인]
- 인증: IATF 16949 유지 중, AI 보조도구는 별도 인증 불필요, 품질매뉴얼에 사용 기준 추가 권고 [3차 미팅 확인]

## 5. 원하는 것
- 경영진 지시: CTO 주도 — "우선 하나 제대로 하자" [3차 미팅 재확인]
- 현업 니즈: 불량 원인 분석에 한 건당 반나절 소요 [2차 미팅 확인]
- **유즈케이스: 불량 원인 분석 보조 (확정, 3차 미팅 재확인)** [2차/3차 미팅 확인]
  - 파일럿: 화성공장 브레이크 디스크 라인
  - 보류: 협력사 품질 평가 → 파일럿 성공 후 2차 확장으로 합의
- 데이터 현황:
  - 검사기준서 약 200건 (PDF) — 샘플 5건 수령 [3차 미팅 확인]
  - 작업표준서 약 300건 (HWP) — 샘플 3건 수령 [3차 미팅 확인]
  - 불량 이력 MES 3년치 약 15만건 — 화성 6개월 8,000건 CSV 수령 [3차 미팅 확인]
  - 불량코드 매핑 엑셀 수령 — 200개 코드 중 30% 공장 간 불일치 [3차 미팅 확인]
- 현재 프로세스 (5단계): [2차 미팅 확인]
  1. MES에서 불량 건 접수 (불량코드, 라인, 제품, 일시)
  2. 과거 유사 불량 이력 검색 (MES 수동 조회 — 병목)
  3. 검사기준서/작업표준서에서 해당 기준 확인 (파일 서버 탐색)
  4. 원인 분석 및 대책 수립 (품질 엔지니어 판단)
  5. 8D 보고서 작성 (한글 양식)

## 6. 제약사항
- 예산: 파일럿 3천만원 이내, 효과 시 내년 본예산 1억 [2차 미팅 확인]
- 일정: 8월 말까지 파일럿 결과 보고 (약 4개월) [2차 미팅 확인]
- 인력: IT팀 이정수 과장 1명 파트타임 투입 [2차 미팅 확인]
- 고객 대면 여부: 내부 직원만 사용. 납품처 노출 계획 없음 [1차 미팅 확인]
- 보안검토: 도입 전 보안검토서 제출 필요 (2주 소요) [3차 미팅 확인]
