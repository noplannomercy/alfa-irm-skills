## 유즈케이스 분석

### 불량 원인 분석 보조 — 5단계 프로세스 매핑

| 단계 | 현재 방식 | 예상 행위 유형 | AI 역할 |
|------|----------|-------------|--------|
| 1. 불량 접수 | MES 자동 | 접근 | MES에서 불량 건 조회 |
| 2. 유사 이력 검색 | MES 수동 조회 (병목) | 탐색(내부) + 접근 | 유사 불량 자동 검색 |
| 3. 기준서 확인 | 파일서버 수동 탐색 | 탐색(내부) | 관련 기준서 자동 검색 |
| 4. 원인 분석 | 품질 엔지니어 판단 | 실행(유연) | 분석 보조 + 추천 |
| 5. 8D 보고서 작성 | 한글 수동 작성 | 실행(유연) | 초안 자동 생성 |

### 예상 행위 유형 분포
- 탐색: ~40% (이력 검색 + 기준서 검색)
- 접근: ~20% (MES 조회)
- 실행(유연): ~40% (원인 분석 + 보고서 작성)
- 실행(확정): 0% (모든 판단이 상황 의존적)
