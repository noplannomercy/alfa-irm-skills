# 성진오토텍 미팅 질문 리스트

## 미팅 기본 정보
- 예상 소요: 60분
- 거버넌스 무게: Medium (현재 판단 — 제조업, 내부용 추정)
- 2차 미팅 필요 예상: 판단 불가 (온프레미스 여부에 따라 달라짐)

## 반드시 물어볼 것 (프로필 빈칸)

### ⬜ 항목 1 — 회사 개요 (보강)
- Q1. 주력 제품이 뭔가요? (브레이크, 서스펜션 등 어떤 부품?)
- Q2. 주요 납품처는 어디인가요? (현대기아, GM 등)
- (왜 물어보는지): 부품 종류에 따라 품질검사 프로세스와 데이터 형태가 완전히 다름

### ⬜ 항목 2 — AI/LLM 현황
- Q1. 이전에 AI나 머신러닝 관련 시도해본 적 있으신가요?
- Q2. 외부 업체와 PoC 진행한 경험이 있으신가요? 결과는?
- Q3. 현재 품질관리에 비전 검사(Vision AI) 같은 건 쓰고 계신가요?
- (왜 물어보는지): 제조업은 비전 AI는 있지만 LLM은 처음인 경우가 많음. 기존 자산 파악 필요

### ⬜ 항목 3 — 인프라
- Q1. 공장 데이터가 클라우드에 올라가 있나요, 아니면 사내 서버에만 있나요?
- Q2. MES(제조실행시스템)나 ERP는 뭘 쓰고 계신가요?
- Q3. 사내에 GPU 서버가 있나요?
- (왜 물어보는지): 제조업은 데이터 외부 반출 금지가 흔함 → 온프레미스/클라우드 분기 결정

### ⬜ 항목 4 — 규제 환경
- Q1. 품질 데이터에 대한 보안 등급이 있나요?
- Q2. 납품처(OEM)에서 데이터 관리 요구사항이 있나요?
- Q3. IATF 16949 등 품질 인증 유지 중이신가요?
- (왜 물어보는지): 자동차 부품은 OEM 요구사항이 사실상 규제. 데이터 외부 반출 금지 가능성

### ⬜ 항목 5 — 원하는 것 (구체화)
- Q1. "품질관리 AI"라고 하시면 구체적으로 어떤 업무를 말씀하시는 건가요?
- Q2. 지금 가장 시간이 많이 걸리는 품질 업무가 뭔가요?
- Q3. 불량 발생 시 원인 분석은 어떻게 하고 계세요?
- Q4. CTO님이 특별히 언급하신 구체적인 기대가 있나요?
- (왜 물어보는지): "품질관리 AI"는 범위가 너무 넓음. 검사자동화/불량분석/매뉴얼검색 등 구체화 필수

### ⬜ 항목 6 — 제약사항
- Q1. 예산 규모가 잡혀 있나요? (대략적으로라도)
- Q2. 언제까지 결과를 보고 싶으신가요?
- Q3. 내부에서 이 프로젝트에 투입할 수 있는 인력이 있나요? (IT팀 등)
- Q4. 이 AI는 내부 직원만 쓰나요, 아니면 납품처에도 노출되나요?
- (왜 물어보는지): 예산/일정/인력으로 로드맵 유형이 결정됨. 대면 범위로 거버넌스 무게 결정

## 업종 특화 질문

### 제조업에서 꼭 확인할 것
- Q. 공장 데이터(검사 결과, 불량 이력 등)를 외부 클라우드로 보내도 되나요?
  → (놓치면 안 되는 이유): 답이 "안 됨"이면 온프레미스 LLM 필수 → 일정/비용 2~3배
- Q. 불량 코드나 검사 항목을 부르는 명칭이 공장/라인/팀마다 다른가요?
  → (놓치면 안 되는 이유): "같은 불량인데 다르게 부른다" → GraphRAG 필요 신호
- Q. 품질 매뉴얼이나 검사 기준서가 문서로 정리되어 있나요? 몇 건 정도?
  → (놓치면 안 되는 이유): 문서화 수준이 RAG 투입 난이도를 결정

## 초반에 반드시 확인 (거버넌스 무게 결정)
- Q. "이 서비스를 고객(외부)이 직접 사용하나요, 내부 직원이 사용하나요?"
  → 고객 대면 = 거버넌스 Heavy, 내부 = Light
- Q. "규제 기관 검토를 받아야 하나요?"
  → 제조업은 보통 불필요, 단 OEM 요구사항이 사실상 규제 역할

## 미팅 중 감지할 신호
- "표현이 달라서..." / "같은 건데 다르게 불러요" → GraphRAG 필요 신호
- "데이터 외부 반출 금지" → 온프레미스/로컬 LLM 분기
- "고객 챗봇도 하고 싶고..." → 2단계 접근 필요
- "위에서 아직 말 안 해줬어" → 스폰서 미확정, 진행 보류 가능성
- "비전 검사는 이미 하고 있어요" → Vision AI ≠ LLM, 구분 설명 필요
- "MES에서 데이터 뽑으면 되지 않나?" → 접근(CLI) 유형 힌트
