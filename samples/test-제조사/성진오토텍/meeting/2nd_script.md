# 성진오토텍 2차 미팅 메모

> 일시: 2026-04-17
> 참석: 김태호 CTO, 박성민 품질관리팀장, 이정수 IT팀 과장, 우리 측 2명

## 미팅 내용 (원문)

- 1차 미팅 서머리 기반으로 대상 업무를 논의함
- 품질관리팀장이 후보 3개를 가져옴:
  1) 불량 원인 분석 보조 — "불량 나면 과거 이력 찾고, 검사기준서 뒤지고, 8D 보고서 쓰는 데 한 건에 반나절"
  2) 검사기준서 검색 — "500건 문서에서 해당 기준 찾는 게 일"
  3) 협력사 품질 평가 — "분기마다 하는데 데이터 취합이 힘들다"

- 논의 결과 1번 "불량 원인 분석 보조"를 파일럿 대상으로 선정
  이유: 가장 시간이 많이 걸리고, 효과가 눈에 보이고, CTO가 "이거 되면 바로 보고 올릴 수 있다"고 함
  2번 검사기준서 검색은 1번 하면서 자연스럽게 포함될 거라고 판단

- 불량 원인 분석 현재 프로세스:
  1단계: MES에서 불량 건 접수 (불량코드, 라인, 제품, 일시)
  2단계: 과거 유사 불량 이력 검색 (MES 수동 조회 — 여기서 시간 많이 걸림)
  3단계: 검사기준서/작업표준서에서 해당 기준 확인 (파일 서버 탐색)
  4단계: 원인 분석 및 대책 수립 (품질 엔지니어 판단)
  5단계: 8D 보고서 작성 (한글 양식)

- 품질관리팀장: "2단계에서 화성공장은 'B-crack'이라 기록하고 아산은 '브레이크디스크_균열'이라 써서 검색이 안 된다"
- 또 "같은 현상인데 서스펜션 쪽은 '피로파괴'라 하고 브레이크 쪽은 '크랙'이라 해서 연결이 안 됨"

- IT팀 이정수 과장:
  - MES DB 직접 조회 가능, REST API는 없고 ODBC로 붙어야 함
  - 문서는 사내 파일서버 \\filesvr\품질관리\ 아래 정리되어 있음
  - 검사기준서는 PDF, 작업표준서는 HWP가 대부분
  - GPU 서버는 없지만, 최근 딥러닝용으로 RTX 4090 2장 구매 품의 올려놓은 상태
  - IT팀에서 1명 파트타임 투입 가능

- 예산: CTO가 "파일럿은 3천만원 이내로 해보자. 효과 나오면 내년 본예산 1억 잡겠다"
- 일정: "8월 말까지 파일럿 결과 보고 가능하면 좋겠다" (약 4개월)
- 8D 보고서 작성까지 AI가 해주면 좋겠지만, 최종 판단은 사람이 해야 한다는 데 합의
- 파일럿 대상: 화성공장 브레이크 디스크 라인 (가장 불량 많은 라인)
