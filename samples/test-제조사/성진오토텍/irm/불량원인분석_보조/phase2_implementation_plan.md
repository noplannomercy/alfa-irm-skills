## 구현 순서 (의존성 기반)

```
Phase A: 인프라 (1주)
  └─ GPU 수령 + Ollama 세팅
  └─ PostgreSQL 16 + pgvector + Apache AGE 설치
  └─ BGE-m3-ko + BGE-reranker-v2-m3 로컬 배포

Phase B: 탐색 — RAG (3주)
  ├─ B-1: quality_standards 컬렉션 (1.5주)
  │   └─ PDF 파싱 + HWP 파싱 테스트 + 임베딩 + 인덱싱
  ├─ B-2: defect_history 컬렉션 (1주) — B-1과 병렬
  │   └─ MES 데이터 전처리 + 임베딩
  └─ B-3: investigation_reports 컬렉션 (0.5~1주) — 8D 위치 확인 후
  └─ 의존: Phase A

Phase C: 접근 — CLI (1주) — Phase B와 병렬
  ├─ C-1: cli-mes (0.5주)
  ├─ C-2: cli-defectcode (0.5주) — C-1과 병렬
  └─ C-3: FastAPI 게이트웨이 (C-1, C-2 완료 후)
  └─ 의존: Phase A (MES 접속 확인)

Phase D: 탐색 — GraphRAG (2~3주)
  └─ 매핑 엑셀 시드 로드 + 8D 보고서 트리플 추출
  └─ 질의 확장 + 멀티홉 검색 통합
  └─ 의존: Phase A + B-3 (investigation_reports)

Phase E: 실행 (1.5주)
  ├─ E-1: 원인 분석 LLM 프롬프트 설계 + 튜닝 (1주)
  └─ E-2: 8D 보고서 생성 프롬프트 (0.5주)
  └─ 의존: Phase B + C + D

Phase F: 통합 + 검증 (1.5주)
  └─ 체이닝 3패턴 엔드투엔드 테스트
  └─ 감사로그 확인
  └─ 보안검토서 제출 (병렬)
  └─ 의존: Phase E

Phase G: 파일럿 운영 (2주)
  └─ 화성공장 품질 엔지니어 3~5명
  └─ 20건 이상 실 불량 건 처리
  └─ 결과 보고
  └─ 의존: Phase F + 보안검토 승인
```

**병렬화 요약:**
```
A ──→ B-1 ──→ B-3 ──→ D ──→ E ──→ F ──→ G
  ├─→ B-2 ──┘             │
  └─→ C ───────────────────┘
```

## 리스크

| 리스크 | 영향 | 대응 |
|-------|------|------|
| HWP 파싱 품질 불량 | quality_standards 컬렉션 품질 저하 | 샘플 3건 우선 테스트, 불량 시 OCR 대안 검토 |
| 8D 보고서 위치 미확인 | investigation_reports + GraphRAG 지연 | 고객에게 즉시 확인 요청, 없으면 Phase D 축소 |
| GPU 입고 지연 | 전체 일정 밀림 | 크리티컬 패스 — 입고 지연 시 CPU 추론으로 임시 진행 |
| LLM 원인 추천 정확도 미달 | 파일럿 실패 | 프롬프트 반복 튜닝 + 유사 이력 수 늘리기 |
| 보안검토 지연 | 파일럿 시작 지연 | Phase F와 병렬 제출, 미리 초안 작성 |
