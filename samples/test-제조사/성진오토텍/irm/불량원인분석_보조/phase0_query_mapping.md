## 체이닝 패턴 (복합 질의)

복수 행위 유형을 연결하는 질의:

- **"이 불량의 원인을 분석해줘"** (질의 #9)
  = 접근(#1: MES에서 불량 상세 조회)
  → 접근(#7: 불량코드 매핑/변환)
  → 탐색(#2: 유사 불량 이력 검색)
  → 탐색(#3: 관련 검사기준서 검색)
  → 탐색(#4: 관련 작업표준서 검색)
  → 실행(유연: 종합 분석 + 원인 추천)

- **"8D 보고서 초안 작성해줘"** (질의 #10)
  = 질의 #9의 결과 (원인 분석)
  → 탐색(#6: 유사 과거 8D 보고서 참조)
  → 실행(유연: 양식에 맞게 초안 생성)
  → Human-in-the-loop: 품질 엔지니어 검토/수정

- **"피로파괴와 크랙이 같은 현상인 건 맞아?"** (질의 #12)
  = 접근(#7: 불량코드 매핑 조회)
  → 탐색(GraphRAG: 엔티티 동의어 관계 확인)

## Phase 0 결론

### 행위 유형 분포 요약
- **주요 유형**: 탐색(내부) (54%) — RAG 투자가 핵심
- **아키텍처 시사**: 탐색 중심 + 접근 보조 + 실행(유연) 마무리 구조. 전형적인 "조회→검색→판단" 체이닝.

### GraphRAG 사전 신호 (Phase 1에서 정식 판단)
- 동음이의어/용어 불일치: **있음** — "B-crack" vs "브레이크디스크_균열", "피로파괴" vs "크랙"
- 인과관계 질의 비율: 3/13 (23%) — #7, #9, #12
- 동일 엔티티 다중 맥락: **있음** — 같은 불량 현상이 공장별·제품군별로 다르게 표현
- → 사전 판단: **후보 (강한 신호 — 3/4 조건 충족 가능성 높음)**

### 데이터 품질 리스크
- HWP 파싱: 작업표준서 300건 — 파서 품질 테스트 필요
- 8D 보고서: 위치/건수 미확인 — 확인 후 ⚠️ 해소 가능
- 불량코드 30% 불일치: 매핑 엑셀 확보 → GraphRAG 시드 데이터
