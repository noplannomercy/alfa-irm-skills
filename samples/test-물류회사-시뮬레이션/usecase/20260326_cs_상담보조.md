# 유즈케이스 정의서: CS 상담 보조

> 고객: 한진로지스
> 작성일: 2026-03-26
> 상태: 확정

## 개요
- 목적: 상담사가 CS 문의에 빠르게 답변할 수 있도록 매뉴얼/이력 검색 + 배송 조회를 보조
- 사용자: CS 상담사 20명 (내부)
- 입력: 고객 문의 내용 (배송 조회, 반품/교환, 클레임 등)
- 출력: 관련 FAQ/매뉴얼 검색 결과 + 배송 현황 조회 + 답변 초안

## 행위 유형 분포 (예상)

| 행위 유형 | 비중 | 설명 |
|----------|------|------|
| 탐색 | 50% | FAQ/매뉴얼/과거 상담 이력 검색 |
| 접근 | 30% | 배송 추적 (택배사 API), 주문 정보 (CS DB) |
| 실행(유연) | 15% | 답변 초안 생성 (LLM) |
| 실행(확정) | 5% | 개인정보 마스킹 |

## 리소스 예상

| 리소스 유형 | 예시 | 규모 |
|------------|------|------|
| 문서/파일 | FAQ 300개, 반품절차서, 클레임 가이드 | ~400건 |
| 시스템/DB | CS MySQL (상담 이력), 택배사 API 3곳 | 4개 시스템 |
| 절차/매뉴얼 | 반품 절차, 클레임 처리 절차 | 2개 절차 |

## GraphRAG 필요성
- 검토 필요
- 근거: 택배사 3곳의 배송 상태 코드 불일치 ("배달중"="배송중"="이동중"), 센터별 절차 차이 가능성
- 판단: Phase 0에서 정식 판단 (조건 2/4 이상 시 승인)
