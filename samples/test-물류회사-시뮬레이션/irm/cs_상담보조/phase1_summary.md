# CS 상담 보조 — Phase 1 종합 요약

> 작성일: 2026-03-26
> 고객: 한진로지스

## 아키텍처 개요

### 행위 유형별 배치

| 행위 유형 | 비율 | 배치 방식 | 핵심 구성요소 |
|----------|------|----------|-------------|
| 탐색 | 33% | 단순 RAG (벡터 검색) | cs_knowledge 단일 컬렉션 |
| 접근 | 33% | CLI + REST 게이트웨이 | delivery CLI (통합), cs CLI |
| 실행(확정) | 17% | 결정적 파이프라인 | 반품 판단, 개인정보 마스킹 |
| 실행(유연) | 17% | LLM + Human-in-the-loop | 클레임 안내, 답변 초안 |

### GraphRAG 판단
- 결정: **불필요** (0/4)
- 택배사 상태 코드 불일치는 매핑 테이블로 해결

### 인프라 요약
- LLM: **Bedrock Claude** (AWS 네이티브)
- 임베딩: **Titan Embeddings V2** (Bedrock)
- 벡터DB: **pgvector** (RDS PostgreSQL)
- REST: **FastAPI** (내부)
- CLI: **2개** (delivery, cs)

### 기존 인프라 재사용
- AWS 인프라 전부 재사용 (EC2, RDS)
- 신규: Bedrock 개통 + pgvector 활성화 + FastAPI 배포

## 전체 아키텍처 플로우

```
상담사 (CS 시스템) → FastAPI 게이트웨이
                       │
         ┌─────────────┼─────────────┐
         ↓             ↓             ↓
     [탐색]        [접근]        [실행]
   cs_knowledge   delivery CLI   반품 판단 PL
   (pgvector)     cs CLI         마스킹 PL
       ↓              ↓          LLM 답변 초안
   Bedrock Claude  JSON 반환     LLM 클레임 안내
       ↓                            ↓
   답변 생성                    상담사 확인
       ↓                     (Human Gate)
   상담사 응답
```
