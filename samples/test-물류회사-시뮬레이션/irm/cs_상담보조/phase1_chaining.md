# CS 상담 보조 — 체이닝 설계

> 작성일: 2026-03-26

## 체이닝 패턴

| # | 질의 | 체이닝 순서 | GraphRAG 필요 |
|---|------|-----------|:------------:|
| 1 | "반품 접수 처리해줘" | 탐색→접근→실행(확정) | N |
| 2 | "이 클레임 어떻게 처리해?" | 접근→탐색→실행(유연) | N |
| 3 | "배송 지연 안내 만들어줘" | 접근→탐색→실행(유연) | N |

## 체이닝 플로우 상세

### 패턴 1: 반품 접수
```
"반품 처리해줘" →
  ① 탐색: cs_knowledge에서 반품 절차 조회
  ② 접근: cs order로 주문 정보 + delivery track으로 배송 상태
  ③ 실행(확정): 규칙 기반 반품 가능 판단 → 상담사에게 절차 안내
```

### 패턴 2: 클레임 처리
```
"이 클레임 어떻게 해?" →
  ① 접근: cs history로 이전 상담 이력 조회
  ② 탐색: cs_knowledge에서 클레임 가이드 검색
  ③ 실행(유연): LLM이 유형 판단 + 보상 기준 제안 → 상담사 확인
```

### 패턴 3: 답변 생성
```
"고객한테 뭐라고 해?" →
  ① 접근: delivery track으로 배송 현황 조회
  ② 탐색: cs_knowledge에서 안내 문구/절차 검색
  ③ 실행(유연): LLM이 답변 초안 생성 → 상담사 검토
```
