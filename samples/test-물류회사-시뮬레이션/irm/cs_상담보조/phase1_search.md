# CS 상담 보조 — 탐색 배치 설계

> 작성일: 2026-03-26
> 탐색 질의: 4건 / 전체 12건 (33%)

## 데이터 소스 분석

| # | 소스명 | 형태 | 갱신 주기 | 이력형/최신형 | 규모 |
|---|--------|------|----------|-------------|------|
| 1 | FAQ | 텍스트 | 수동 (월 1~2회 업데이트) | 최신형 | 300건 |
| 2 | 반품 절차서 | 문서 | 수동 (분기 1회) | 최신형 | 1건 (10p) |
| 3 | 클레임 처리 가이드 | 문서 | 수동 (분기 1회) | 최신형 | 1건 (15p) |

## 컬렉션 설계

### 분리 기준 판단
| 기준 | FAQ vs 절차서/가이드 | 결과 |
|------|-------------------|------|
| 갱신 주기 같은가? | 모두 수동, 비슷 | 합침 |
| 메타데이터 같은가? | doc_type, category 공통 | 합침 |
| 질의 패턴 섞이는가? | "반품 어떻게?" → FAQ+절차서 동시 검색 | 합침 |

→ **단일 컬렉션** `cs_knowledge`

### 컬렉션 구조
| 컬렉션명 | 포함 소스 | 메타데이터 | 인덱싱 범위 | 갱신 방식 |
|----------|----------|-----------|-----------|----------|
| `cs_knowledge` | FAQ + 반품절차서 + 클레임가이드 | doc_type, category, keyword | 전체 (소규모) | 수동 (변경 시 재인덱싱) |

## GraphRAG 판단

| # | 조건 | 충족 | 근거 |
|---|------|:----:|------|
| 1 | 동음이의어/용어 불일치 빈번 | ✗ | 택배사 상태 코드는 매핑 테이블로 해결, FAQ 내 용어는 통일 |
| 2 | 인과관계 질의 >20% | ✗ | 0/12 = 0% |
| 3 | RAG 적중률 목표 미달 | ✗ | 소규모 데이터 + 단순 검색, 기본 RAG로 충분 예상 |
| 4 | 동일 엔티티 다중 맥락 | ✗ | 없음 |
| | **합계** | **0/4** | |

→ 결정: **불필요** — 단순 RAG로 충분

## 검색 전략
- 검색 방식: **벡터 단독** — 소규모 데이터 (300건+α), 하이브리드 불필요
- 리랭킹: **미적용** — 데이터 규모 작고 false positive 낮음
- 크로스 컬렉션: 해당 없음 (단일 컬렉션)

## 임베딩/모델 선택
- 임베딩: **Titan Embeddings V2** (Bedrock) — AWS 네이티브, 한국어 지원
- LLM: **Claude** (Bedrock) — AWS 네이티브
- 벡터DB: **pgvector** (RDS PostgreSQL 또는 기존 MySQL + pgvector 호환 옵션)
