## 구현 순서 (의존성 기반)

```
Phase A: 인프라 (0.5주)
  ├─ Bedrock 개통
  ├─ pgvector 활성화 (RDS PostgreSQL)
  ├─ FastAPI 기본 구조 배포
  └─ 개인정보 마스킹 파이프라인

Phase B: 탐색 (1주) [Phase A 완료 후]
  ├─ FAQ/매뉴얼 파싱 + 임베딩
  ├─ cs_knowledge 컬렉션 인덱싱
  └─ 벡터 검색 구현

Phase C: 접근 (1주) [Phase A 완료 후, Phase B와 병렬]
  ├─ delivery CLI (택배사 3곳 통합 + 상태 코드 매핑)
  ├─ cs CLI (MySQL 연동)
  └─ REST 엔드포인트 래핑

Phase D: 실행 (1주) [Phase B + C 완료 후]
  ├─ 답변 초안 생성 (LLM 프롬프트 + 검색 연동)
  ├─ 반품 판단 파이프라인
  └─ 클레임 처리 안내

Phase E: 통합 + 파일럿 (0.5주)
  ├─ 체이닝 통합 테스트
  ├─ 상담사 5명 파일럿 시작
  └─ 피드백 수집
```
