## 구현 순서

```
Phase A: 인프라 (1주)
  ├─ Neo4j Community 설치 (EC2)
  ├─ 그래프 스키마 정의
  └─ FastAPI 엔드포인트 추가

Phase B: 탐색 (2주) [Phase A 완료 후]
  ├─ 매뉴얼/절차서 파싱 + logistics_docs 인덱싱
  ├─ 이상 이력 추출 + defect_history 인덱싱 (3년치)
  ├─ 하이브리드 검색 + 리랭킹 구현
  ├─ GraphRAG 용어 그래프 구축 (물류운영팀장 감수)
  └─ 질의 확장 로직 구현

Phase C: 접근 (1주) [Phase A 완료 후, Phase B와 병렬]
  ├─ wms CLI 구현 (6개 명령어)
  ├─ REST 엔드포인트 래핑
  └─ 단위 테스트

Phase D: 실행 (1주) [Phase B + C 완료 후]
  ├─ 검수 판단 파이프라인
  ├─ 이상 건 기록 (GraphRAG 용어 표준화 연동)
  ├─ 이상 건 처리 판단 (LLM + Human-in-the-loop)
  └─ 원인 분석

Phase E: 통합 + 파일럿 (1주)
  ├─ 체이닝 통합 테스트
  ├─ 1센터 파일럿 시작 (검수 담당자 10명)
  └─ 피드백 수집
```
