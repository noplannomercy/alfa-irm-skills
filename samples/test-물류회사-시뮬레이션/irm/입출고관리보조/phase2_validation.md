# 입출고 관리 보조 — 검증 기준

> 작성일: 2026-05-15

## 행위 유형별 검증

### 탐색 검증
| 항목 | 기준 | 측정 방법 |
|------|------|----------|
| 유사 건 검색 정확도 (Recall@10) | ≥75% | 물류운영팀장 선정 이상 건 20건, 관련 건이 상위 10건에 포함 |
| GraphRAG 개선율 | RAG 대비 +25% 이상 | 질의 확장 ON/OFF 비교 (동일 20건) |
| 매뉴얼 검색 정확도 | Recall@5 ≥85% | 절차/기준 질의 20건 |
| 응답 시간 | ≤3초 | P95 기준 |

### 접근 검증
| 항목 | 기준 | 측정 방법 |
|------|------|----------|
| wms CLI 정상 동작 | 성공률 100% | 명령어별 5건 = 30건 |
| 데이터 정합성 | WMS 원본과 100% 일치 | 교차 검증 10건 |
| REST 응답 시간 | ≤1초 | P95 기준 |

### 실행 검증
| 항목 | 기준 | 측정 방법 |
|------|------|----------|
| 검수 판단 정확도 | ≥95% | 합/불 케이스 각 10건 |
| 용어 표준화 정확도 | ≥90% | 자유 텍스트 50건 → 표준 유형 매핑 |
| 이상 건 처리 제안 적절성 | ≥60% | 담당자가 수정 없이 또는 최소 수정 비율 |
| 이상 건 기록 성공률 | 100% | WMS에 정상 기록 확인 20건 |

## 엔드투엔드 검증

### 체이닝 시나리오
| # | 시나리오 | 체이닝 | 기대 결과 | 합격 기준 |
|---|---------|-------|----------|----------|
| 1 | "이 입고 건 검수해줘" | 접근→탐색→실행(확정) | 품목 확인 + 기준 조회 + 합/불 판단 | ≤5초, 판단 정확 |
| 2 | "비슷한 파손 건 찾아줘" | 탐색 (GraphRAG) | "훼손","깨짐" 등 동의어로 확장 검색 | 질의 확장 동작 + 유사 건 반환 |
| 3 | "이상 건 처리해줘" | 접근→탐색→탐색→실행(유연)→실행(확정) | 전체 체인 완료 + WMS 기록 | ≤15초, 기록 확인 |
| 4 | "이번 달 파손율 원인 분석" | 접근→탐색→실행(유연) | 통계 + 패턴 + 원인 추론 | 분석 결과 + 근거 제시 |

## PoC 범위
- 대상: **1개 센터** (본원 물류센터)
- 데이터: 이상 건 이력 3년치 + 매뉴얼/절차서/기준표
- 기간: **6주**
- 참여자: 검수 담당자 10명 (1센터)
- 성공 기준:
  - GraphRAG 개선율 +25% 이상
  - 검수 판단 정확도 ≥95%
  - 이상 건 처리 시간 50% 단축
  - 담당자 만족도 4/5 이상
- 확대 조건: 성공 시 → 나머지 4개 센터 확대 (용어 그래프 센터별 확장)
