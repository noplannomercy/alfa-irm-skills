# 입출고 관리 보조 — 탐색 배치 설계

> 작성일: 2026-05-15
> 탐색 질의: 5건 / 전체 13건 (38%)

## 데이터 소스 분석

| # | 소스명 | 형태 | 갱신 주기 | 이력형/최신형 | 규모 |
|---|--------|------|----------|-------------|------|
| 1 | 입출고 매뉴얼 (센터별) | 문서 | 수동 (분기 1회) | 최신형 | 5건 (센터별) |
| 2 | 이상 처리 절차서 | 문서 | 수동 (분기 1회) | 최신형 | 5개 절차 |
| 3 | 이상 건 처리 이력 | DB (자유 텍스트) | 실시간 (이상 발생 시) | 이력형 (축적) | ~1000건/월 × 수년치 |
| 4 | 검수 기준표 | 문서 | 수동 (품목 추가 시) | 최신형 | 품목별 기준 |

## 컬렉션 설계

### 분리 기준 판단
| 기준 | 매뉴얼/절차서/기준표 vs 이상 이력 | 결과 |
|------|-------------------------------|------|
| 갱신 주기 같은가? | 문서=분기, 이력=실시간 | **분리** |
| 메타데이터 같은가? | 문서=doc_type/센터, 이력=이상유형/품목/일시 | **분리** |
| 질의 패턴 섞이는가? | "절차 찾아줘" vs "비슷한 건 찾아줘" | 분리하되 체이닝 |

→ **2개 컬렉션**

### 컬렉션 구조
| 컬렉션명 | 포함 소스 | 메타데이터 | 인덱싱 범위 | 갱신 방식 |
|----------|----------|-----------|-----------|----------|
| `logistics_docs` | 매뉴얼 + 절차서 + 기준표 | doc_type, center_id, category | 전체 (소규모) | 수동 (변경 시) |
| `defect_history` | 이상 건 처리 이력 | defect_type, center_id, product, date, handler | 전체 (이력형, 3년+) | 배치 (일 1회, WMS에서 추출) |

## GraphRAG 판단

| # | 조건 | 충족 | 근거 |
|---|------|:----:|------|
| 1 | 동음이의어/용어 불일치 빈번 | ✓ | "파손"="훼손"="깨짐"="damage"="broken", 센터별 자유 텍스트 |
| 2 | 인과관계 질의 >20% | ✗ | 2/13=15% (근접하나 미달) |
| 3 | RAG 적중률 목표 미달 | ✓ | 용어 불일치로 벡터 검색만으로는 유사 건 누락 예상 |
| 4 | 동일 엔티티 다중 맥락 | ✓ | 같은 이상 유형이 센터별/담당자별/시기별로 다르게 기록 |
| | **합계** | **3/4** | |

→ 결정: **승인**

### GraphRAG 구성
- **노드**: 이상 유형 (표준화), 품목, 센터, 담당자
- **엣지**: 동의어(synonym: 파손↔훼손↔깨짐), 발생위치(at-center), 관련품목(affects)
- **활용**: 질의 확장 — "파손" 검색 시 "훼손", "깨짐" 등 자동 확장

## 검색 전략
- 검색 방식: **하이브리드 (벡터 + BM25)** — 이상 이력은 키워드 매칭도 중요 (품목 코드, 센터 ID)
- 리랭킹: **적용** — 용어 불일치로 초기 검색 노이즈 예상
- 크로스 컬렉션: GraphRAG로 용어 확장 → `defect_history`에서 검색 + `logistics_docs`에서 절차 조회
