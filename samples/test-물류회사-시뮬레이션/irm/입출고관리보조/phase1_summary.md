## 아키텍처 개요

### 행위 유형별 배치

| 행위 유형 | 비율 | 배치 방식 | 핵심 구성요소 |
|----------|------|----------|-------------|
| 탐색 | 38% | **GraphRAG** + 하이브리드 RAG | logistics_docs + defect_terms 그래프 |
| 접근 | 23% | CLI + REST 게이트웨이 | wms CLI (기존 인프라 재사용) |
| 실행(확정) | 15% | 결정적 파이프라인 | 검수 판단, 이상 건 기록 |
| 실행(유연) | 23% | LLM + Human-in-the-loop | 이상 건 처리 판단, 원인 분석 |

### GraphRAG 판단

| # | 조건 | 충족 | 근거 |
|---|------|:----:|------|
| 1 | 동음이의어/용어 불일치 빈번 | ✓ | "파손"="훼손"="깨짐"="damage", 센터별 자유 텍스트 |
| 2 | 인과관계 질의 >20% | ✗ | 2/13 = 15% (근접) |
| 3 | RAG 적중률 목표 미달 | ✓ | 용어 불일치로 단순 벡터 검색 시 유사 건 누락 예상 |
| 4 | 동일 엔티티 다중 맥락 | ✓ | 같은 이상 유형이 센터별/담당자별 다르게 기록 |
| | **합계** | **3/4** | |

→ 결정: **승인** — CS(0/4)와 완전히 다른 결과

### 인프라 요약
- LLM: **Bedrock Claude** (기존 CS와 공유)
- 임베딩: **Titan Embeddings V2** (기존 공유)
- 벡터DB: **pgvector** (기존 공유, 컬렉션 추가)
- 그래프DB: **Neo4j Community** (신규 — GraphRAG용)
- REST: **FastAPI** (기존 게이트웨이에 엔드포인트 추가)
- CLI: **1개 신규** (wms)

### CS 유즈케이스와 공유/신규 비교

| 항목 | CS 상담 보조 | 입출고 관리 보조 | 공유 |
|------|------------|---------------|:----:|
| Bedrock | 사용 중 | 동일 | ✓ |
| pgvector | cs_knowledge | + logistics_docs (추가) | ✓ (인스턴스) |

| FastAPI | 게이트웨이 | 엔드포인트 추가 | ✓ |
| delivery CLI | 사용 중 | 해당 없음 | — |
| cs CLI | 사용 중 | 해당 없음 | — |
| wms CLI | 없음 | **신규** | ✗ |

**→ 인프라 대부분 재사용. 신규는 Neo4j + wms CLI + 컬렉션만.**

## 전체 아키텍처 플로우

```
검수 담당자 (모바일/웹) → FastAPI 게이트웨이
                            │
              ┌─────────────┼─────────────┐
              ↓             ↓             ↓
          [탐색]        [접근]        [실행]
        GraphRAG       wms CLI       검수 판단 PL
        질의 확장        ↓           이상 기록 PL
           ↓          JSON 반환     LLM 처리 판단
        하이브리드 RAG                LLM 원인 분석
        (pgvector)                      ↓
           ↓                       담당자 확인
        리랭킹                    (Human Gate)
           ↓
        LLM 응답 생성
```
