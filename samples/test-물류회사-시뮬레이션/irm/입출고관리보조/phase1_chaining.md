# 입출고 관리 보조 — 체이닝 설계

> 작성일: 2026-05-15

## 체이닝 패턴

| # | 질의 | 체이닝 순서 | GraphRAG 필요 |
|---|------|-----------|:------------:|
| 1 | "이상 건 처리해줘" | 접근→탐색→탐색→실행(유연)→실행(확정) | Y |
| 2 | "이 입고 건 검수해줘" | 접근→탐색→실행(확정) | N |
| 3 | "파손율 원인 분석해줘" | 접근→탐색→실행(유연) | Y |

## 체이닝 플로우 상세

### 패턴 1: 이상 건 전체 처리 (가장 복잡)
```
"이상 건 처리해줘" →
  ① 접근: wms defect-list → 이상 건 상세 확인
  ② 탐색: defect_history에서 유사 건 검색 (GraphRAG 질의 확장)
  ③ 탐색: logistics_docs에서 처리 절차 조회
  ④ 실행(유연): LLM이 유형 판단 + 처리 방법 제안 → 담당자 확인
  ⑤ 실행(확정): wms defect-record → 표준화된 용어로 기록
```

### 패턴 2: 입고 검수
```
"이 입고 건 검수해줘" →
  ① 접근: wms inbound → 입고 품목/수량 확인
  ② 탐색: logistics_docs에서 검수 기준 조회
  ③ 실행(확정): 기준 대비 합/불 판단 → 결과 기록
```

### 패턴 3: 원인 분석
```
"파손율 원인 분석해줘" →
  ① 접근: wms defect-stats → 통계 조회
  ② 탐색: defect_history에서 패턴 검색 (GraphRAG, 품목/센터/시기)
  ③ 실행(유연): LLM이 통계+이력 기반 원인 추론 → 물류운영팀장 검토
```
