## 구현 순서

```
Phase A: 인프라 (1주)
  ├─ Bedrock 개통
  ├─ PostgreSQL 확장 (pgvector + Apache AGE)
  ├─ FastAPI 기본 구조
  └─ 가명처리 파이프라인 (데이터 준비에 필수)

Phase B: 탐색 (3주) [A 완료 후]
  ├─ 가이드라인+약관+FAQ 파싱 → uw_guidelines 인덱싱
  ├─ 심사 이력 추출 → uw_cases 인덱싱 (27만건)
  ├─ OCR 청구서 → claim_docs 인덱싱 (품질 필터)
  ├─ 하이브리드 검색 + 리랭킹
  ├─ GraphRAG 약관 관계 그래프 구축 (심사팀장 감수)
  └─ 질의 확장 + 관계 추적 로직

Phase C: 접근 (1주) [A 완료 후, B와 병렬]
  ├─ uw CLI (PostgreSQL)
  ├─ core CLI (REST API, 신용정보 제한)
  ├─ REST 엔드포인트
  └─ 감사로그 파이프라인

Phase D: 실행 (2주) [B+C 완료 후]
  ├─ 심사 의견 초안 (LLM + GraphRAG 연동)
  ├─ 할증/특약 판단 보조
  ├─ 재심사 절차 파이프라인
  └─ Human-in-the-loop UI

Phase E: 통합 + 파일럿 (1주)
  ├─ 체이닝 테스트 (4개 시나리오)
  ├─ 감사로그 완전성 검증
  └─ 심사역 5명 파일럿 시작
```
