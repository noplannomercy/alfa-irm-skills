# 자동차보험 심사 보조 — 실행 배치 설계

> 작성일: 2026-04-03
> 설계 신뢰도: ✅ (절차서 실제 확인)

## 실행(확정) — 결정적 파이프라인

### 가명처리 파이프라인
```
Step 1: 텍스트 수신
Step 2: 패턴 매칭:
         - 이름 (한글 2~4자) → 홍*동
         - 주민번호 (6자리-7자리) → 완전 제거
         - 주소 → 시/구까지만
         - 연락처 → 마스킹
Step 3: 가명처리 적용
Step 4: 원본 미저장, 가명 텍스트만 반환
```
- 적용 시점: LLM 투입 전 자동
- 신용정보(지급 이력): 가명처리 아닌 완전 비노출 (core CLI에서 통계만 제공)

### 감사로그 파이프라인
```
Step 1: AI 보조 요청 수신
Step 2: AI 응답 생성 (검색 결과 + 판단 근거)
Step 3: 로그 기록:
         - 요청 내용
         - AI 응답 (검색 결과 + 근거 자료 참조 목록)
         - 심사역 최종 판단 (승인/거절/보류)
         - 심사역 수정 사항 (AI 의견과 다를 경우)
         - 타임스탬프 + 심사역 ID
Step 4: 기존 심사 시스템 감사로그 테이블에 저장
```
- 금감원 검사 대비: AI 판단 근거 + 심사역 최종 판단 함께 보관
- 보관 기간: 심사 이력과 동일 (5년+)

### 재심사 절차 파이프라인
```
Step 1: [접근] uw case → 원심사 건 조회
Step 2: [탐색] uw_cases에서 유사 재심사 사례 검색
Step 3: [탐색] uw_guidelines에서 재심사 기준 조회
Step 4: [판단] 규칙 기반 → 재심사 접수 가능 여부
Step 5: [안내] 재심사 절차 3단계 안내
```
- 결정적: 재심사 접수 요건 충족 여부는 규칙 기반

## 실행(유연) — 판단 기반 프로세스

### 심사 의견 초안 생성
```
Step 1: [접근] core contract → 계약 정보 조회
Step 2: [접근] uw history → 과거 심사 이력
Step 3: [탐색] uw_guidelines → 심사 기준 + 약관 조항 검색 (GraphRAG 관계 추적)
Step 4: [탐색] uw_cases → 유사 사례 검색 (GraphRAG 질의 확장)
Step 5: [LLM] 검색 결과 + 계약 정보 기반:
         - 적용 약관 조항 정리
         - 유사 사례 판단 결과 참조
         - 심사 의견 초안 생성
         - 판단 근거 명시
Step 6: [심사역 확인] 초안 검토 → 승인/수정/거절
Step 7: [감사로그] AI 의견 + 심사역 최종 판단 기록
```
- 판단 포인트: Step 5 — LLM이 약관 적용 + 유사 사례 참조 + 의견 초안
- Human-in-the-loop: Step 6 — 심사역 최종 판단 (보조수단성 충족)
- 고정화 조건: 단순 심사(60%)에서 동일 판단 5회 연속 → 규칙화 검토

### 할증/특약 적용 판단 보조
```
Step 1: [접근] core contract → 계약/보험료 정보
Step 2: [탐색] uw_guidelines → 할증 기준 검색 (GraphRAG: 할증→사고유형→약관 관계)
Step 3: [LLM] 기준 대비 적용 여부 판단 + 근거 제시
Step 4: [심사역 확인] 판단 결과 검토
```
