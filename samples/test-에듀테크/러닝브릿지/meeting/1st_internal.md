## 기술 신호
- 거버넌스 무게: **Light** — 1차 PoC는 내부 상담사 보조 한정, 수강생 대면 없음
- GraphRAG 필요: **검토 필요** — 용어 불일치보다는 강의별 콘텐츠 분리 검색이 핵심. 3,000개 강의 각각의 내용을 구분하여 검색해야 하므로 메타데이터 태깅 + per-course 인덱싱 필요
- 인프라 경로: **클라우드 API** — AWS 기반, Bedrock 활용 자연스러움
- PoC 이력: 없음 (첫 AI 도입)

## 감지된 리스크
- **답변 일관성**: "같은 질문인데 상담사마다 답변이 달라요" → FAQ 150개만으로는 부족, 판단 기준(환불 규정, 학습 경로 추천 로직 등) 표준화 필요
- **콘텐츠 질문 난이도**: 문의의 35%가 강의 내용 질문이나 강의마다 내용이 달라 단순 FAQ로 커버 불가 → PoC 범위를 일부 강의로 제한하거나, 강의 자막/교안 기반 RAG 필요
- **개인정보 리스크**: 수강생 이름/이메일/수강이력을 LLM에 넣을 경우 개인정보보호법 대상. 미성년자 포함 시 더 엄격
- **예산 제약**: 2,000만원은 PoC 수준, 본격 구축 시 추가 예산 확보 필요
- **스폰서 구체성**: 대표이사 지시는 있으나 "어떻게"는 현장에 위임 → 범위 초과 시 추가 승인 필요

## 예상 로드맵

### 경량 PoC (4~6주) — AWS Bedrock 기반
- **Week 1-2**: Bedrock 개통 + 벡터DB(OpenSearch Serverless 또는 pgvector) 세팅 + FAQ 150건 파싱·임베딩
- **Week 3-4**: RAG 파이프라인 구축 + 상담사 보조 UI 프로토타입 + 수강 방법(30%) 질문 대응
- **Week 5-6**: 강의 교안(PDF) 일부(50~100개) 추가 임베딩 + 콘텐츠 질문 대응 테스트 + 상담사 피드백 수렴
- **판단 시점**: 6주차 — 확대(전체 강의) vs 피벗 vs 중단

### 비용 추정
- Bedrock(Claude/Titan): 월 50~100만 (PoC 규모)
- 벡터DB: 월 30~50만 (OpenSearch Serverless 또는 pgvector on RDS)
- IT 인력 2명 인건비는 고객 내부 부담
- 총 외부 비용: 월 100~150만 × 2개월 = 200~300만 (인프라) + 컨설팅/개발비
