## 체이닝 패턴 (복합 질의)

복수 행위 유형을 연결하는 질의:

- **#7** "결제한 지 7일 됐는데 환불 가능한가요?"
  = 탐색(내부, #6a: 환불 규정 조항 검색) → 접근(#4a: 수강생 결제일 조회) → 실행(확정, #6b: 환불 기준 적용)

- **#8** "다른 강의로 변경하고 싶다는데 가능한가요?"
  = 탐색(내부, #6a: 변경 규정 검색) → 접근(#4a: 수강생 현재 수강 상태 조회) → 실행(확정, #6b: 변경 가능 여부 판단)

- **#9** "이 수강생 진도가 많이 느린데, 어떤 강의를 추천하면 좋을까요?"
  = 접근(#4a: 수강이력/진도 조회) → 실행(유연, #4b: 학습 패턴 분석) → 탐색(내부, #2: 관련 강의 검색)

## Phase 0 결론

### 행위 유형 분포 요약
- **주요 유형**: 탐색(내부) 57% — FAQ + 교안/자막 + 상담이력 + 환불규정에서의 의미 검색이 핵심
- **아키텍처 시사**:
  - **RAG가 시스템의 핵심** (57%)
  - Multi-source RAG 필요: FAQ 컬렉션 / 강의 콘텐츠 컬렉션(교안+자막) / 규정 컬렉션 / 상담이력 컬렉션
  - 접근(21%)은 PostgreSQL SQL + Zendesk API로 구현
  - 실행(확정)(14%)은 환불/변경 규정 기반 규칙 로직
  - 체이닝 패턴 3건 — 탐색→접근→실행 연결이 핵심 플로우

### GraphRAG 사전 신호 (Phase 1에서 정식 판단)
- 동음이의어/용어 불일치: **없음** — 강의 콘텐츠는 각각 독립적, 용어 혼동보다 콘텐츠 분리가 핵심
- 인과관계 질의 비율: 1/14 (7%) — #9 학습 추천만 해당
- 동일 엔티티 다중 맥락: **약함** — 같은 수강생이 여러 강의를 듣지만, 관계 탐색보다는 이력 조회(접근)로 해결
- → 사전 판단: **불필요** — 메타데이터 필터링 + per-course 인덱싱으로 대응. 추후 커리큘럼 기반 추천이 본격화되면 재검토
