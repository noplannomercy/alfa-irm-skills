# 학습상담 보조 — 실행 구현 체크리스트

> 작성일: 2026-03-27

## 실행(확정) 파이프라인

### 환불 판단
- [ ] 파이프라인 4 Step 구현 (탐색→접근→판단→응답)
- [ ] judge_refund() 규칙 함수 구현
- [ ] 환불규정 PDF 분석 → 규칙 조건 확정 (일수, 진도%, 환불 비율)
- [ ] 입력 JSON 스키마: {student_id, inquiry_text}
- [ ] 출력 JSON 스키마: {eligible, refund_type, refund_amount_pct, reason, rule_section}
- [ ] Step별 에러 핸들링 (규정 미발견, 수강생 미발견, 규칙 매칭 실패)
- [ ] 단위 테스트: 전액환불/부분환불/환불불가 각 1건
- [ ] 통합 테스트: 체이닝 패턴 A 전체 흐름

### 강의 변경 판단
- [ ] 파이프라인 4 Step 구현 (탐색→접근→판단→응답)
- [ ] judge_course_change() 규칙 함수 구현
- [ ] 변경규정 분석 → 규칙 조건 확정 (진도%, 상태, 차액 정산)
- [ ] 입력 JSON 스키마: {student_id, target_course_id}
- [ ] 출력 JSON 스키마: {eligible, reason, conditions, rule_section}
- [ ] Step별 에러 핸들링
- [ ] 단위 테스트: 변경가능/변경불가(진도초과)/변경불가(이수완료) 각 1건
- [ ] 통합 테스트: 체이닝 패턴 A 전체 흐름
