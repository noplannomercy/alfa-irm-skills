## 1. 회사 개요
- 업종: 에듀테크 (온라인 교육 플랫폼)
- 회사명: 러닝브릿지
- 규모: 직원 300명, 수강생 50만명 (미성년자 약 15%), 연매출 500억
- 주력 사업: 온라인 교육 플랫폼 운영, 강의 콘텐츠 3,000개 보유 (영상+PDF 교안+SRT 자막)

## 2. AI/LLM 현황
- 기존 시도: 없음 (첫 AI 도입)
- 남은 자산: 없음
- 활용 가능 자산: 학습 상담 FAQ 150개 (구글독스), Zendesk 상담이력 36만건 (API 추출 가능)

## 3. 인프라
- 클라우드: AWS (EC2 + RDS + S3)
- DB: PostgreSQL (RDS) — 수강이력/진도/성적 3년치
- 스토리지: S3 — 강의 콘텐츠 (영상+PDF 교안+SRT 자막 2,800개)
- CS 시스템: Zendesk (API 연동 가능)
- GPU: 없음 (클라우드 API 경로)

## 4. 규제 환경
- 데이터 분류: 수강생 이름/이메일/수강이력 → 개인정보보호법 대상
- 컴플라이언스팀: 별도 없음, 법무 외주 (필요 시)
- 미성년자: 전체의 약 15%, 법정대리인 동의 수집 중
- 비식별화: 이름 마스킹, 이메일 도메인만 노출 가능
- 개인정보 처리방침: 홈페이지 게시 중, 내부 정책 문서 보유

## 5. 원하는 것
- 경영진 지시: "AI로 학습 상담 효율화하라" (대표이사)
- 현업 니즈: 상담사 10명, 하루 500건 수동 처리, 답변 일관성 부족
- 유즈케이스 1차: 상담사 보조 도구 (내부용)
- 유즈케이스 2차: 수강생 직접 대면 AI (추후)
- 문의 유형: 수강 방법 30% / 콘텐츠 질문 35% / 환불·변경 20% / 기술 문제 15%
