# 에듀테크사 미팅 질문 리스트

## 미팅 기본 정보
- 예상 소요: 60분
- 거버넌스 무게: Light ~ Medium (수강생 대면 여부에 따라 결정)
- 2차 미팅 필요 예상: 판단 불가 (대면 범위 확인 후 결정)

## 반드시 물어볼 것 (프로필 빈칸)

### ⬜ AI/LLM 현황
- Q1. 이전에 AI나 챗봇 도입을 시도해보신 적이 있나요? (PoC, 외부 업체 검토 등)
- Q2. 시도했다면 어떤 결과였고, 당시 만들어진 데이터나 시스템이 남아있나요?
- Q3. 현재 상담사분들이 사용하는 도구가 있나요? (CRM, 티켓 시스템, 엑셀 등)
- (왜 물어보는지): 기존 자산이 있으면 재활용 가능. 실패 경험이 있으면 같은 실수 방지.

### ⬜ 인프라
- Q1. 교육 플랫폼은 어떤 클라우드에서 운영 중인가요? (AWS, GCP, Azure, 자체 서버 등)
- Q2. 수강생 데이터와 강의 콘텐츠는 어떤 DB에 저장되어 있나요?
- Q3. 외부 API 연동이 있나요? (결제, LMS, 영상 스트리밍 등)
- (왜 물어보는지): 인프라에 따라 LLM 배포 방식과 비용 구조가 달라짐.

### ⬜ 규제 환경
- Q1. 수강생 개인정보(이름, 연락처, 학습이력)를 AI 시스템에 활용해도 되는지 내부 검토가 되었나요?
- Q2. 개인정보보호 담당자나 법무팀이 있나요?
- Q3. 미성년 수강생이 포함되어 있나요? (아동 개인정보 별도 규제)
- (왜 물어보는지): 에듀테크는 학습이력이 개인정보에 해당. 미성년자 포함 시 규제 격상.

### ⬜ 제약사항 (예산/일정)
- Q1. 이 프로젝트에 배정된 예산이 있나요? 대략적인 범위라도 알 수 있을까요?
- Q2. 대표이사 지시의 목표 시점이 있나요? ("올해 안에", "분기 내" 등)
- Q3. AI 프로젝트에 투입할 수 있는 내부 인력이 있나요? (IT팀, 데이터팀 등)
- (왜 물어보는지): 예산·일정에 따라 단계적 접근 vs 일괄 도입이 결정됨.

## 업종 특화 질문

### 에듀테크에서 꼭 확인할 것

**상담 업무 구조**
- Q. 하루 500건 문의의 유형별 비율이 어떻게 되나요? (수강 신청, 환불, 학습 진도, 기술 문제, 강의 추천 등)
- Q. 문의 채널이 어디인가요? (전화, 채팅, 이메일, 앱 내 문의 등)
- Q. 상담사분들이 답변 시 참고하는 자료가 있나요? (FAQ, 매뉴얼, 내부 위키 등)
- Q. 답변에 "판단"이 필요한 경우가 있나요? (환불 기준 적용, 학습 경로 추천 등)
- (놓치면 안 되는 이유): 단순 FAQ 자동응답과 판단형 상담 보조는 완전히 다른 설계. 유형별 비율이 곧 ROI.

**콘텐츠 구조**
- Q. 강의 3,000개의 분류 체계가 있나요? (카테고리, 태그, 난이도 등)
- Q. 강의 간 선후수 관계(커리큘럼)가 정의되어 있나요?
- Q. 강의 설명, 목차, 자막 등 텍스트 데이터가 구조화되어 있나요?
- (놓치면 안 되는 이유): 학습 추천이나 상담 보조를 하려면 콘텐츠 메타데이터 품질이 핵심.

## 초반에 반드시 확인 (거버넌스 무게 결정)

- Q. **"이 AI 시스템을 수강생이 직접 사용하나요, 내부 상담사만 사용하나요?"**
  → 수강생 직접 사용 = 거버넌스 Medium 이상, 내부 상담사 보조 = Light
  → **이 질문이 전체 설계 방향을 결정합니다**

- Q. "잘못된 답변이 나갔을 때 책임 소재는 어떻게 보시나요?"
  → 오답 리스크 허용 범위 파악

- Q. "수강생 데이터를 외부 클라우드 AI 서비스에 보내도 되나요?"
  → 온프레미스/로컬 LLM 분기 결정

## 미팅 중 감지할 신호

- "강의마다 부르는 이름이 달라서..." / "같은 내용인데 카테고리가 다르게 되어있어요" → GraphRAG 필요 신호
- "수강생 데이터는 절대 외부에 못 보내요" → 온프레미스/로컬 LLM 분기
- "나중에 수강생한테 직접 챗봇도 만들고 싶어요" → 2단계 접근 필요 (1차: 내부 보조, 2차: 대면)
- "대표님이 말씀은 하셨는데 구체적으로 뭘 원하시는지..." → 스폰서 지시 모호, 범위 정의 선행 필요
- "환불이나 결제 관련도 AI가 처리했으면..." → 결제/환불은 별도 거버넌스, 단순 자동화 불가
