## A. 사전 질의 (담당자용)

아래 항목 중 현재 파악 가능한 내용을 정리해주세요.

### 현황 관련
- 이전에 AI, 챗봇, 자동응답 등을 도입하거나 검토해보신 적이 있으신가요? 있다면 어떤 결과였나요?
- 현재 학습 상담사분들이 사용하는 업무 도구는 무엇인가요? (상담 관리 시스템, 엑셀, 내부 위키 등)
- 하루 500건 문의의 대략적인 유형 비율을 알려주실 수 있나요? (예: 수강 신청 30%, 학습 진도 질문 25%, 환불 15%, 기술 문제 15%, 기타 15%)
- 문의가 들어오는 채널은 어디인가요? (전화, 채팅, 이메일, 앱 내 문의 등)

### 목표/기대 관련
- AI 도입으로 기대하시는 구체적인 효과가 있으시면 알려주세요 (예: 상담 처리 시간 단축, 야간 자동응답, 상담사 업무 부하 감소 등)
- 이 AI 시스템을 수강생이 직접 사용하게 할 계획인지, 내부 상담사의 업무 보조 도구로 사용할 계획인지 방향을 알려주세요
- 예산이나 일정 계획이 잡혀있으시면 공유 부탁드립니다

### 수강생/데이터 관련
- 수강생 중 미성년자(만 14세 미만)가 포함되어 있나요?
- 수강생 개인정보를 외부 클라우드 서비스에서 처리해도 되는지 내부 방침이 있나요?
- 개인정보보호 담당자 또는 법무팀이 있으신가요?

## B. 리소스 조사 요청 (IT팀용)

AI 도입 범위를 구체화하기 위해 아래 정보가 필요합니다.
미팅 전 또는 미팅 후 1주 내 회신 부탁드립니다.

### 문서/파일
- [ ] 상담사가 참고하는 FAQ, 매뉴얼, 응대 스크립트 목록
- [ ] 각 문서의 저장 위치, 파일 형식, 대략적 건수
- [ ] 샘플 문서 3~5건 (내용 확인용)

### 강의 콘텐츠
- [ ] 강의 분류 체계 (카테고리, 태그, 난이도 등)
- [ ] 강의 설명, 목차, 자막 등 텍스트 데이터 현황
- [ ] 강의 간 선후수 관계(커리큘럼) 정의 여부

### 시스템/DB
- [ ] 교육 플랫폼 운영 환경 (AWS, GCP, Azure, 자체 서버 등)
- [ ] 수강생 데이터 및 강의 데이터 저장 DB 종류
- [ ] 외부 연동 시스템 목록 (결제, LMS, 영상 스트리밍 등)
- [ ] 상담 이력 데이터 저장 여부 및 규모 (건수, 기간)

### 보안/접근
- [ ] 수강생 개인정보 접근 제한 사항
- [ ] 데이터 외부 반출 가능 여부
- [ ] 클라우드/온프레미스 제약 사항
