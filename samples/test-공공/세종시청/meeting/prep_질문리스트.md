# 세종시청 미팅 질문 리스트

## 미팅 기본 정보
- 예상 소요: 60분
- 거버넌스 무게: Heavy (현재 판단 — 공공기관, 개인정보, 행안부)
- 2차 미팅 필요 예상: 예 (범위 넓음, 보안/개인정보 담당 동석 필요)

## 반드시 물어볼 것 (프로필 빈칸)

### 🟡 항목 1 — 회사 개요 (보강)
- Q1. 시민봉사과 민원상담팀 인원이 몇 명인가요?
- Q2. 민원 유형별 비중이 어떻게 되나요? (세금/복지/인허가/건축/환경 등)
- Q3. 가장 많은 반복 질문 top 3이 뭔가요?
- (왜 물어보는지): 민원 유형에 따라 관련 부서/시스템/규제가 완전히 다름. 범위 좁히기의 시작점.

### ⬜ 항목 2 — AI/LLM 현황
- Q1. 이전에 AI나 챗봇 도입 시도한 적 있으신가요?
- Q2. 다른 시·군·구에서 AI 민원 도입 사례를 참고하신 게 있나요?
- Q3. 정부24 AI 챗봇 "민원길라잡이"를 쓰고 계신가요?
- (왜 물어보는지): 공공은 벤치마크 사례가 중요. 기존 중앙정부 AI와 중복 방지.

### 🟡 항목 3 — 인프라 (보강)
- Q1. 시청 IT 인프라가 G-클라우드인가요, 자체 전산실인가요?
- Q2. 새올 시스템 외에 민원 관련 시스템이 더 있나요?
- Q3. 정부24 연동은 어떤 방식인가요? (API, 화면 연계 등)
- Q4. 외부 클라우드(AWS, Azure) 사용이 가능한가요?
- (왜 물어보는지): 공공은 클라우드 보안인증(CSAP) 필요. G-클라우드 vs 외부 클라우드 분기.

### ⬜ 항목 4 — 규제 환경
- Q1. 개인정보보호 담당자가 누구인가요?
- Q2. 개인정보영향평가(PIA) 대상인가요?
- Q3. 시민 민원 데이터를 AI에 투입하는 것에 대한 내부 검토가 있었나요?
- Q4. 행안부 "공공분야 AI 활용 가이드라인"을 알고 계시나요?
- Q5. 정보보안 등급이 어떻게 되나요?
- (왜 물어보는지): 공공 AI는 규제가 가장 무거움. PIA, CSAP, 행안부 가이드라인 전부 확인 필요.

### 🟡 항목 5 — 원하는 것 (범위 좁히기 핵심)
- Q1. "민원 AI"라고 하시면, 전화 상담사가 참고하는 보조 도구인가요, 시민이 직접 쓰는 챗봇인가요?
- Q2. 모든 민원 유형을 다 AI로 하고 싶은 건가요, 특정 분야부터 시작할 건가요?
- Q3. 만약 특정 분야라면 어떤 걸 먼저 하고 싶으신가요?
- Q4. 현재 민원 FAQ나 매뉴얼이 정리되어 있나요?
- Q5. 민원 상담 이력이 DB에 쌓이고 있나요?
- (왜 물어보는지): "민원 AI" 범위가 너무 넓음. 부서/유형/대면 여부로 좁혀야 IRM 진입 가능.

### ⬜ 항목 6 — 제약사항 (보강)
- Q1. 5천만원 예산의 집행 기한이 언제까지인가요?
- Q2. 내부 IT 인력이 투입 가능한가요?
- Q3. 조달 절차(나라장터)를 거쳐야 하나요?
- Q4. 예산 외에 행정 절차상 걸리는 것이 있나요? (위원회 심의, 정보화사업 심의 등)
- (왜 물어보는지): 공공은 예산 있어도 조달/심의 절차 때문에 3~6개월 걸릴 수 있음.

## 업종 특화 질문

### 공공에서 꼭 확인할 것
- Q. 이 AI 서비스가 시민에게 직접 응대하나요, 내부 직원만 사용하나요?
  → (놓치면 안 되는 이유): 시민 직접 = 규제 Very Heavy (PIA, 행안부 가이드라인, 오답 책임)
- Q. 민원 데이터에 주민등록번호, 주소, 소득 정보 등 개인정보가 포함되나요?
  → (놓치면 안 되는 이유): 공공 개인정보는 민간보다 훨씬 무거움. 개인정보영향평가 필수 가능성.
- Q. 정보화사업 심의를 거쳐야 하나요?
  → (놓치면 안 되는 이유): 심의 통과에 1~2개월 소요. 예산 있어도 착수 못 함.

## 초반에 반드시 확인 (거버넌스 무게 결정)
- Q. "이 서비스를 시민이 직접 사용하나요, 내부 직원이 사용하나요?"
  → 시민 직접 = 거버넌스 Very Heavy, 내부 직원 = Heavy
- Q. "행안부나 개인정보보호위원회 검토를 받아야 하나요?"
  → 공공은 거의 필수

## 미팅 중 감지할 신호
- "부서마다 답변이 달라서" → GraphRAG 또는 부서별 지식 분리 필요
- "시민이 직접 쓰게 하고 싶다" → 거버넌스 격상 + 2단계 접근
- "정보화사업 심의가 아직..." → 일정 리스크
- "다른 시청에서 했다던데" → 벤치마크 참조 + 차별화 필요
- "전자정부법 때문에..." → 법적 제약 확인 필요
- "나라장터 거쳐야 해요" → 조달 3~6개월 추가
