# 세종시청 2차 미팅 메모

> 일시: 2026-04-18
> 참석: 김현정 민원상담팀장, 정우성 전산담당, 박도현 정보보안담당관, 우리 측 2명

## 미팅 내용 (원문)

- 정보보안담당관 박도현:
  - 개인정보영향평가(PIA): 민원 데이터에 주민번호/소득정보 포함 → PIA 대상. 다만 "상담사 보조 도구(내부용)"이면 간이 PIA로 가능. 2주 소요.
  - 외부 클라우드: CSAP 인증 필수. 네이버클라우드(G-클라우드 입점), NHN 클라우드 등 사용 가능.
  - AI 가이드라인: 행안부 "공공분야 AI 활용 가이드라인" 적용. 핵심은 "최종 판단은 사람이", "개인정보 비식별화", "AI 판단에 대한 설명 가능성".
  - 데이터 비식별화: 상담 이력을 AI에 넣으려면 주민번호/소득정보 마스킹 필수. 이름/전화번호는 상담 맥락에 불필요하므로 제거.
  - 감사로그: 누가 어떤 민원 데이터에 접근했는지 기록 필수 (공공기관 의무).

- 정보화사업 심의: 5천만원 "미만"이면 심의 면제. 이번 건은 4,990만으로 조정하면 면제 가능. 팀장이 "그렇게 하자"고 함.
- 나라장터: 5천만 미만 수의계약 가능. 절차 약 2주.

- 유즈케이스 확정:
  - 세금/복지 합쳐서 하기엔 부서가 다르고 규정도 다름
  - 팀장: "세금 쪽이 FAQ가 더 잘 정리되어 있고 반복률도 높으니 세금부터 하자"
  - **"세금/과태료 민원 상담 보조"로 확정**
  - 범위: 재산세, 자동차세, 주민세, 과태료 관련 민원
  - 복지는 세금이 잘 되면 2차로

- 세금 관련 리소스 상세:
  - 세무과 FAQ: 약 150건 (시청 홈페이지 DB에서 세금 카테고리 필터)
  - 세무 매뉴얼: 약 30건 HWP (재산세 부과 기준, 납부 방법, 감면 조건 등)
  - 지방세법 관련 법령: 법제처 API로 조회 가능 (공개 데이터)
  - 세금 민원 이력: 새올에서 세금 카테고리 필터 → 약 3만건/년
  - 부과/납부 데이터: 세외수입 시스템 — DB 직접 조회 가능하지만 개인정보(납세자) 포함

- GraphRAG 관련:
  - 전산담당: "세무 용어가 진짜 어려워요. 민원인이 '재산세'라 하면 되는데 '건물분 재산세'인지 '토지분 재산세'인지에 따라 답이 달라져요"
  - 또: "감면 조건도 장애인 감면, 다자녀 감면, 국가유공자 감면... 민원인은 그냥 '세금 깎아달라'고 하는데"
  - 팀장: "과태료도 주정차 과태료, 환경 과태료, 건축 과태료 전부 다른 법인데 민원인은 그냥 '과태료'라고만 해요"

- 전산담당:
  - 새올 DB 스키마 일부 공유 가능 (민원 접수 테이블)
  - 세외수입 시스템도 DB 직접 조회 가능
  - 법제처 API는 공개 — 바로 사용 가능
