# 세종시청 1차 미팅 메모

> 일시: 2026-04-11
> 참석: 김현정 민원상담팀장, 정우성 전산담당, 우리 측 2명

## 미팅 내용 (원문)

- 시민봉사과 민원상담팀 직원 12명. 하루 평균 전화 민원 500건 + 방문 민원 200건.
- 전화 민원 유형 비중: 세금/과태료 문의 30%, 복지 지원 문의 25%, 인허가/신고 절차 20%, 주정차/교통 15%, 기타 10%.
- 가장 많은 반복 질문: "재산세 납부 기한이 언제예요?", "기초연금 신청 어떻게 해요?", "음식점 영업허가 절차가 어떻게 되나요?"
- 민원상담팀장: "세금이랑 복지가 전체의 55%인데, 이 두 개만 해도 상담사 부담이 확 줄어들 것 같다"
- FAQ: 약 800건 — 시청 홈페이지 FAQ DB에 있음. 부서별로 각자 관리. 최근 1년 내 업데이트된 건 절반 정도.
- 민원 응대 매뉴얼: 약 120건 — 부서별 Hwp 파일. 공유 드라이브에 흩어져 있음.
- 민원 상담 이력: 새올에 3년치 쌓여 있음. 전산담당이 DB 직접 조회 가능.
- 인프라: G-클라우드 사용 중. 자체 전산실도 있음. 외부 클라우드는 CSAP 인증된 것만 사용 가능.
- 새올 시스템은 행안부 표준. REST API는 없고 DB 직접 조회만 가능.
- 정부24 연동은 화면 연계 방식. API 호출은 안 됨.
- AI 경험: 없음. 다른 시청에서 한 거 몇 개 들어봤는데 "뉴스에만 나오고 실제로 되는 건 본 적 없다"고 함.
- 개인정보: 민원 상담 내용에 이름, 전화번호, 주소 당연히 포함. 세금/복지는 주민번호, 소득 정보도.
- 개인정보보호 담당: 기획예산과 정보보안담당관. 2차 미팅에 동석 가능.
- 정보화사업 심의: 5천만원 이상은 심의 대상이지만, 이번 건은 정확히 5천만이라 담당자 확인 필요.
- 조달: 나라장터 수의계약 가능 범위 확인 중. 안 되면 제한경쟁.
- 팀장: "우선 상담사가 쓰는 내부 도구부터. 시민 직접 챗봇은 나중에. 위에서 시키면 모르지만."
- 예산 집행 기한: 올해 12월 말. 다만 9~10월에는 효과 보고가 있어야 내년 예산 확보 가능.
- IT 인력: 전산담당 정우성 1명. 다른 업무도 겸하고 있어서 파트타임만 가능.
- 팀장: "부서마다 용어가 달라요. 세무과에서는 '과세증명'이라 하고 민원인은 '세금 증명서 떼러 왔다'고 하고..."
