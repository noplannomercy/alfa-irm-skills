## 실행(확정) — 해당 없음

## 실행(유연) — 판단 기반 프로세스

### 프로세스 1: 민원 답변 초안 생성

```
Step 1: [입력] 상담사가 시민 문의 내용 입력
Step 2: [접근] cli-saeol: 해당 민원 상세 조회 (비식별화)
Step 3: [접근] cli-tax: 관련 부과/납부 현황 (비식별화, 해당 시)
Step 4: [탐색] tax_knowledge: FAQ + 매뉴얼 + 법령 검색
         + GraphRAG: 용어 확장 ("세금 깎아달라" → 감면 유형들)
Step 5: [탐색] tax_consultation: 유사 과거 민원 응대 검색
Step 6: [LLM 판단] 종합 → 답변 초안 생성
Step 7: [사람 확인] 상담사가 검토/수정 후 시민에게 답변
```

- 판단 포인트: Step 6
- Human-in-the-loop: Step 7 — 행안부 가이드라인 "최종 판단은 사람"
- 설명 가능성: 답변 초안에 "근거: FAQ #xxx, 지방세법 제OO조" 표시 (행안부 요구)
- 비식별화: Step 2, 3에서 자동 마스킹 후 LLM 전달
