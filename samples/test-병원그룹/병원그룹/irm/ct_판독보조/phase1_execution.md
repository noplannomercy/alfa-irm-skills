# CT 판독문 보조 — 실행 배치 설계

> 작성일: 2026-03-25

## 실행(확정) — 결정적 파이프라인

### 가명처리 파이프라인
```
Step 1: [접근] EMR → 판독문 원본 텍스트 획득
Step 2: [패턴매칭] 정규식 → 이름/생년월일/주민번호 식별
Step 3: [치환] 규칙 기반 → 마스킹 적용 (이름→[NAME], 생년월일→[DOB], 주민번호→[SSN])
Step 4: [검증] 패턴 재검사 → 미처리 항목 확인
Step 5: [기록] 격리 스토리지 → 가명처리 완료 데이터 저장
```

- 입력: 판독문 원본 텍스트 (PHI 포함)
- 출력: 가명처리 완료 판독문 텍스트
- 성공/실패 경계:
  - Step 2: 패턴 미탐지 시 → 수동 검토 큐로 이동
  - Step 4: 검증 실패 시 → 재처리 후 수동 확인
- 테스트: 알려진 PHI 포함 샘플 100건으로 탐지율/정밀도 검증

### 감사로그 기록 파이프라인
```
Step 1: [트리거] AI 조회 요청 발생
Step 2: [수집] 사용자ID, 조회대상, 조회시각, 질의내용 캡처
Step 3: [기록] EMR 감사로그 테이블에 INSERT
Step 4: [확인] 기록 성공 여부 반환
```

- 입력: AI 조회 이벤트 (사용자, 대상, 시각)
- 출력: 감사로그 기록 ID
- 성공/실패 경계:
  - Step 3: DB 기록 실패 시 → 로컬 파일에 백업 저장 후 재시도 큐
- 테스트: 동시 조회 부하 테스트, 로그 누락 여부 검증

## 실행(유연) — 판단 기반 프로세스

### 판독문 초안 생성 프로세스
```
Step 1: [접근] PACS → 현재 CT 영상 메타데이터 + 환자 이전 판독 이력
Step 2: [탐색] ct_reports → 유사 소견 판독문 검색 (GraphRAG 질의 확장 포함)
Step 3: [LLM 판단] 유사 판독문 + 가이드라인 참조 → 판독문 초안 생성
Step 4: [사람 확인] 의사가 초안 검토 → 수정/승인
```

- 판단 포인트: Step 3 — LLM이 유사 사례와 가이드라인을 참조하여 소견 기술, 결론 초안 생성
- Human-in-the-loop: Step 4 — **의사가 반드시 최종 확인** (보조수단 판정의 핵심 조건)
- 고정화 조건: 특정 소견 유형에서 의사 수정률이 5% 이하로 안정되면, 해당 유형은 템플릿 기반 실행(확정)으로 전환 검토

### 용어 표준화 제안 프로세스
```
Step 1: [접근] 판독문 텍스트 입력
Step 2: [탐색] medical_terms(GraphRAG) → 비표준 용어 식별 + 표준 용어 매핑
Step 3: [LLM 판단] 문맥 고려하여 표준 용어 치환 제안 생성
Step 4: [사람 확인] 의사가 제안 검토 → 수락/거부
```

- 판단 포인트: Step 3 — 같은 용어라도 문맥에 따라 다른 표준 용어가 적절할 수 있음
- Human-in-the-loop: Step 4 — 의사가 용어 치환 수락/거부 결정
- 고정화 조건: 특정 용어 쌍의 수락률 95% 이상 시 자동 치환 후보로 등록

### 소견 요약 프로세스
```
Step 1: [접근] 판독문 전문 입력
Step 2: [탐색] 가이드라인 → 해당 부위/소견의 기술 기준 참조
Step 3: [LLM 판단] 판독문 핵심 소견 추출 + 요약 생성
Step 4: [사람 확인] 의사가 요약 정확성 확인
```

- 판단 포인트: Step 3 — 어떤 소견이 핵심인지, 어떤 순서로 기술할지 판단
- Human-in-the-loop: Step 4 — 누락/오류 확인
- 고정화 조건: 부위별 요약 패턴이 안정되면 템플릿화 가능
