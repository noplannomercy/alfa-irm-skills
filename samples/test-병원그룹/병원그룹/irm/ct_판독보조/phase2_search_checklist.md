# CT 판독문 보조 — 탐색 구현 체크리스트

> 작성일: 2026-03-25

## RAG 기본 (§1~§8)

### §1 컬렉션 설계
- [ ] 컬렉션 분리 기준 확정 — 갱신주기(일vs수동) + 메타데이터(환자정보vs용어관계) → 2개 분리
- [ ] `ct_reports`: CT 판독문 과거 데이터 (PACS → 가명처리 후 인덱싱)
- [ ] `medical_terms`: 의학용어 동의어/약어 그래프 (Neo4j) — 벡터 컬렉션 아닌 그래프로 관리

### §2 메타데이터 스키마
- [ ] `ct_reports` 공통 필드: `doc_type="ct_report"`, `dept_name="영상의학과"`, `report_date`, `source="PACS"`
- [ ] `ct_reports` 도메인 필드: `patient_id_masked`(가명), `body_part`(촬영 부위), `finding_codes`(소견코드), `radiologist_id`(판독의)
- [ ] 추출 방식: `report_date`/`patient_id`=FHIR 메타데이터 자동, `body_part`/`finding_codes`=규칙 기반 파싱, `radiologist_id`=FHIR 자동

### §3 문서 파싱 & 청킹
- [ ] 파서: 판독문은 구조화 텍스트 (소견/결론 섹션) → 섹션 기반 파싱
- [ ] 청킹 전략: 판독문 단위 (1건 = 1청크, 보통 200~500토큰으로 청킹 불필요)
- [ ] 긴 판독문(>512토큰): 소견/결론 분리 청킹, 메타데이터 공유
- [ ] 토크나이저: multilingual-e5-large 기준 512 토큰 맥스

### §4 임베딩 모델
- [ ] 모델: **multilingual-e5-large** (한/영 혼용 의학용어 처리)
- [ ] 실행: **로컬** (A100, 온프레미스 제약)
- [ ] 벡터 차원: 1024
- [ ] 테스트: 의학용어 동의어 쌍 50개로 유사도 검증 (cosine similarity > 0.8 기준)

### §5 검색 전략
- [ ] **하이브리드** (벡터 + BM25) — 의학 약어/코드는 키워드 매칭 필수 (예: "HCC", "BIRADS-4")
- [ ] RRF (Reciprocal Rank Fusion) 조합, 가중치 벡터:BM25 = 0.7:0.3 초기 설정
- [ ] 크로스 컬렉션 라우팅: GraphRAG 질의 확장 → ct_reports 검색 (순차, 라우팅 불필요)

### §6 리랭킹
- [ ] 적용: **예** — 다중 소견 질의 시 false positive 높음
- [ ] 리랭커: **bge-reranker-v2-m3** (다국어, 로컬 실행 가능)
- [ ] 적용 조건: 검색 결과 상위 20건 → 리랭킹 → 상위 5건 반환

### §7 인덱싱 범위
- [ ] 이력형 (`ct_reports`): **전체** — 과거 수년치 판독문 전부 인덱싱 (유사 사례 검색 품질 위해)
- [ ] 초기 인덱싱: 최근 3년 우선, 이후 점진적 확대
- [ ] 최신형 해당 없음 (판독문은 축적형)

### §8 갱신 파이프라인
- [ ] 주기적: 일 1회 배치 — 당일 완료된 판독문 가명처리 → 인덱싱
- [ ] 트리거: PACS에서 판독 완료 이벤트 (향후 실시간 전환 검토)
- [ ] doc_id 매핑: `study_id` (PACS 검사 고유ID) → 중복 방지 + 증분 업데이트

## GraphRAG 추가 (§9~§12) — 승인됨 (3/4)

### §9 인프라
- [ ] **Neo4j Community** 설치 (온프레미스, 원내 서버)
- [ ] 그래프 스키마:
  - 노드: `Term`(용어), `Code`(표준코드), `BodyPart`(부위), `Finding`(소견), `Radiologist`(판독의)
  - 엣지: `SYNONYM_OF`, `ABBREVIATION_OF`, `PART_OF`, `EXPRESSED_AS`, `MAPS_TO_CODE`
- [ ] 초기 데이터: 영상의학과 주요 용어 500~1000개 + 동의어/약어 매핑

### §10 엔티티/트리플 추출
- [ ] 추출 방식: **반자동** — 초기 용어 사전은 영상의학과 과장 감수, 이후 LLM 보조 추출
- [ ] LLM 추출 설정: 판독문에서 의학용어 엔티티 추출 → 기존 그래프와 매칭 → 신규 용어 후보 제안
- [ ] 제약조건: 자기참조 금지, 관계는 동사형 (`IS_SYNONYM_OF`, `ABBREVIATES`), 근거 출처 필수
- [ ] 추출 품질: 영상의학과 과장이 주 1회 신규 용어/관계 검토 승인

### §11 그래프 검색 통합
- [ ] 질의 확장: 사용자 질의 용어 → Neo4j에서 1-hop 동의어/약어 조회 → 확장된 용어 목록으로 벡터+BM25 검색
- [ ] 멀티홉: 기본 1-hop (동의어 직접 연결), 필요 시 2-hop (용어→표준코드→다른 동의어)
- [ ] 컨텍스트 어셈블리: [검색된 판독문] + [관련 용어 관계 트리플] → LLM에 전달

### §12 갱신 동기화
- [ ] 벡터+그래프 동기: 신규 판독문 인덱싱 시 → LLM이 신규 용어 후보 추출 → 검토 큐에 추가
- [ ] 문서 삭제 시: 벡터 삭제 + 해당 문서에서만 나온 용어의 엣지 검토 (고아 방지)
- [ ] 용어 사전 수동 업데이트 시: 그래프만 갱신 (벡터 재인덱싱 불필요)
