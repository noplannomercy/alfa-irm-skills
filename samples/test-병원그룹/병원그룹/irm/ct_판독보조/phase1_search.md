## GraphRAG 판단

| # | 조건 | 충족 | 근거 |
|---|------|:----:|------|
| 1 | 동음이의어/용어 불일치 빈번 | ✓ | "간 종괴"="hepatic mass"="liver mass"="간 SOL", 의사마다 같은 소견을 다르게 기술 |
| 2 | 인과관계 질의 >20% | ✗ | 2/14 = 14% (기준 미달이나 근접) |
| 3 | RAG 적중률 목표 미달 | ✓ | 이전 PoC 실패 원인이 용어 불일치 — 벡터 검색만으로는 재실패 예상 |
| 4 | 동일 엔티티 다중 맥락 | ✓ | 같은 병변이 시간축(이전/현재), 의사축(표현 차이), 용어축(한글/영문/약어)으로 다중 표현 |
| | **합계** | **3/4** | |

→ 결정: **승인** (3/4 충족, 이전 PoC 실패 원인과 직결)

### GraphRAG 구성
- **노드**: 의학용어 (표준코드 기반), 의사(판독의), 부위, 소견, 진단
- **엣지**: 동의어(synonym), 약어(abbreviation), 포함관계(part-of), 의사별표현(expressed-as)
- **활용**: 질의 확장(query expansion) — 사용자가 "간 종괴"로 검색 시 "hepatic mass", "liver mass", "간 SOL" 등으로 자동 확장 후 벡터 검색

## 검색 전략
- 검색 방식: **하이브리드 (벡터 + BM25)** — 의학용어는 키워드 매칭도 중요 (약어, 코드)
- 리랭킹: **적용** — 의학 도메인 특성상 초기 검색 결과 노이즈 높음, 리랭커로 정밀도 향상
- 크로스 컬렉션: GraphRAG로 `medical_terms` 조회 → 질의 확장 → `ct_reports`에서 벡터+BM25 검색

### 검색 플로우
```
사용자 질의
  → ① GraphRAG: medical_terms에서 동의어/약어 확장
  → ② 하이브리드 검색: ct_reports에서 벡터+BM25
  → ③ 리랭킹: 검색 결과 정밀 정렬
  → ④ LLM: 최종 응답 생성
```

## 임베딩/모델 선택
- 임베딩: **multilingual-e5-large** — 한국어+영어 혼용 의학용어 처리, 로컬 실행 가능 (A100)
- 리랭커: **bge-reranker-v2-m3** — 다국어 지원, 로컬 실행 가능
- LLM: **로컬 LLM** (Llama 3 70B 또는 Qwen2.5 72B) — 온프레미스 제약, A100 2장 기준
- 벡터DB: **Milvus** 또는 **Qdrant** — 온프레미스 설치, 하이브리드 검색 지원
- 그래프DB: **Neo4j Community** — 온프레미스 설치, 용어 관계 그래프 저장
