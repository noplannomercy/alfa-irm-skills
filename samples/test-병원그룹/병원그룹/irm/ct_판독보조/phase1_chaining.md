## 체이닝 플로우 상세

### 패턴 1: 유사 사례 기반 초안 생성
```
"이 소견과 비슷한 판독문 찾아서 초안 만들어줘" →
  ① 탐색: GraphRAG로 소견 용어 확장 → ct_reports에서 유사 판독문 검색
  ② 실행(유연): LLM이 유사 판독문 + 가이드라인 참조 → 초안 생성 → 의사 확인
```

- **핵심 의존**: ①의 검색 품질이 ②의 초안 품질을 결정
- **GraphRAG 역할**: 의사별 표현 차이를 넘어 의미적으로 유사한 판독문 검색
- **실패 시**: 유사 판독문 없으면 가이드라인만으로 초안 생성 (품질 저하 가능성 알림)

### 패턴 2: 이력 비교 분석
```
"이 환자의 이전 판독 이력 보고, 이번 소견과 비교해줘" →
  ① 접근: PACS FHIR API → 환자의 과거 CT 판독 이력 조회
  ② 탐색: ct_reports → 이전 판독문과 현재 소견의 유사도 비교
  ③ 실행(유연): LLM이 변화 분석 → 비교 요약 생성 → 의사 확인
```

- **핵심 의존**: ①에서 이력을 먼저 확보해야 ②③ 진행 가능 (순차)
- **GraphRAG 역할**: 이전/현재 판독에서 같은 소견을 다른 용어로 기술한 경우 매칭
- **실패 시**: 이전 이력 없으면 "초진 환자"로 안내, 비교 대신 유사 타환자 사례 제안

### 패턴 3: 판독문 생성 + 가명처리 + 기록
```
"새 판독문 작성하고 가명처리까지 해줘" →
  ① 실행(유연): 판독문 초안 생성 → 의사 수정/승인
  ② 실행(확정): 승인된 판독문에 가명처리 적용
  ③ 실행(확정): 감사로그에 AI 생성 이력 기록
```

- **핵심 의존**: ①에서 의사 승인이 완료되어야 ②③ 진행 (Human-in-the-loop 게이트)
- **GraphRAG 역할**: 불필요 (가명처리와 로그 기록은 규칙 기반)
- **실패 시**: ②에서 가명처리 실패 시 수동 검토 큐로 이동, ③ 진행 보류

## 체이닝 오케스트레이션

- **방식**: 순차 실행 (각 Step 완료 확인 후 다음 Step)
- **타임아웃**: Step별 30초, 전체 체인 5분
- **실패 처리**: 중간 Step 실패 시 이전 결과까지 반환 + 실패 알림
- **Human-in-the-loop 게이트**: 실행(유연) Step 완료 시 의사 확인 대기 (비동기)
