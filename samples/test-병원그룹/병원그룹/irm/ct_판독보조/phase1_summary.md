# CT 판독문 보조 — Phase 1 종합 요약

> 작성일: 2026-03-25
> 고객: 병원그룹

## 아키텍처 개요

### 행위 유형별 배치

| 행위 유형 | 비율 | 배치 방식 | 핵심 구성요소 |
|----------|------|----------|-------------|
| 탐색(내부+외부) | 35% | GraphRAG + 하이브리드 RAG | ct_reports 컬렉션, medical_terms 그래프, 질의 확장 |
| 접근 | 29% | CLI + REST 게이트웨이 | PACS CLI (FHIR), EMR CLI, 가이드라인 CLI |
| 실행(확정) | 14% | 결정적 파이프라인 | 가명처리 파이프라인, 감사로그 파이프라인 |
| 실행(유연) | 21% | LLM + Human-in-the-loop | 판독문 초안 생성, 용어 표준화, 소견 요약 |

### GraphRAG 판단
- 결정: **승인**
- 조건 충족: **3/4** (동음이의어 ✓, RAG 미달 예상 ✓, 다중 맥락 ✓)
- 핵심 용도: 의학용어 질의 확장 — 의사별 표현 차이를 넘어 의미 기반 검색 실현

### 인프라 요약
- LLM: **로컬** (Llama 3 70B 또는 Qwen2.5 72B) — 온프레미스 제약, A100 2장
- 임베딩: **multilingual-e5-large** — 한/영 혼용 의학용어, 로컬 실행
- 리랭커: **bge-reranker-v2-m3** — 다국어, 로컬 실행
- 벡터DB: **Milvus** 또는 **Qdrant** — 온프레미스, 하이브리드 검색
- 그래프DB: **Neo4j Community** — 온프레미스, 용어 관계 그래프
- REST: **FastAPI** — 내부 네트워크 전용
- CLI: **3개** (PACS, EMR, 가이드라인)

### 기존 인프라 재사용
- 다른 유즈케이스(간호기록 요약, MRI 확장)와 공유 가능:
  - `medical_terms` 그래프 (용어 사전은 진료과 공통)
  - Neo4j, Milvus/Qdrant 인스턴스 (컬렉션 추가만)
  - FastAPI 게이트웨이 (엔드포인트 추가)
  - EMR CLI (환자 조회 공통)
- 신규 구축 필요:
  - `ct_reports` 컬렉션 (CT 판독문 전용)
  - PACS CLI (FHIR 연동)
  - 가명처리 파이프라인
  - GraphRAG 용어 그래프 초기 구축

## 전체 아키텍처 플로우

```
사용자(의사/간호사) → FastAPI 게이트웨이
                        │
          ┌─────────────┼─────────────┐
          ↓             ↓             ↓
      [탐색]        [접근]        [실행]
    GraphRAG       PACS CLI      가명처리 PL
    질의 확장      EMR CLI       감사로그 PL
       ↓          가이드라인       LLM 초안
    하이브리드 RAG    CLI          용어 표준화
    (Milvus/Qdrant)   ↓          소견 요약
       ↓          JSON 반환         ↓
    리랭킹                     의사 확인
       ↓                     (Human Gate)
    LLM 응답 생성
       ↓
    감사로그 기록
       ↓
    사용자 응답
```

## 예상 일정

| 단계 | 기간 | 내용 |
|------|------|------|
| 인프라 | 2주 | A100 환경 세팅, Milvus/Qdrant + Neo4j + FastAPI 설치 |
| 탐색(RAG) | 3주 | 판독문 인덱싱, 임베딩 파이프라인, 하이브리드 검색 구현 |
| GraphRAG | 2주 | 의학용어 그래프 초기 구축, 질의 확장 로직 |
| 접근(CLI) | 2주 | PACS FHIR CLI, EMR CLI, 가이드라인 CLI 구현 |
| 실행(파이프라인) | 2주 | 가명처리 + 감사로그 파이프라인, LLM 초안 생성 |
| 통합 테스트 | 1주 | 체이닝 테스트, 엔드투엔드 검증, 의사 피드백 수렴 |
| **합계** | **12주** | IRB 2주 + 데이터준비 2주는 별도 (선행 조건) |

**참고**: 선행 조건(IRB 승인 2주 + 데이터 준비 2주)과 인프라 세팅(2주)은 병렬 진행 가능 → 실질 PoC 기간 약 10주
