# CT 판독문 보조 — 실행 구현 체크리스트

> 작성일: 2026-03-25

## 실행(확정) 파이프라인

### 가명처리 파이프라인
- [ ] 파이프라인 Step 정의: 원본획득 → 패턴매칭 → 치환 → 검증 → 저장
- [ ] 성공/실패 경계:
  - 패턴매칭 실패 (미탐지 의심) → 수동 검토 큐
  - 검증 실패 (잔여 PHI 발견) → 재처리 + 수동 확인
  - 저장 실패 → 재시도 3회 후 알림
- [ ] 입력 JSON: `{ "report_text": "원본", "study_id": "검사ID" }`
- [ ] 출력 JSON: `{ "masked_text": "가명처리완료", "study_id": "검사ID", "masked_fields": ["name", "dob", "ssn"], "status": "success" }`
- [ ] 정규식 패턴: 한국 이름 (2~4자 한글), 생년월일 (YYMMDD/YYYY-MM-DD), 주민번호 (6-7자리)
- [ ] CLI 구현: `deidentify run --input <파일/텍스트> --output <경로>`
- [ ] 단위 테스트: 알려진 PHI 포함 샘플 100건 (탐지율 99% 이상, 정밀도 95% 이상)
- [ ] 에러 핸들링: 실패 시 원본 데이터는 격리 스토리지에 보존, 처리 상태 로그 기록

### 감사로그 기록 파이프라인
- [ ] 파이프라인 Step 정의: 트리거 → 수집 → 기록 → 확인
- [ ] 성공/실패 경계:
  - DB INSERT 실패 → 로컬 파일 백업 후 재시도 큐
  - 확인 실패 → 알림 (감사로그 누락은 규제 이슈)
- [ ] 입력 JSON: `{ "user_id": "사용자", "action": "ai-query", "target": "대상", "query": "질의", "timestamp": "시각" }`
- [ ] 출력 JSON: `{ "log_id": "기록ID", "status": "success" }`
- [ ] EMR 감사로그 테이블 스키마 확인 + AI 컬럼 추가
- [ ] 동시 접근 처리: 트랜잭션 격리 (동시 조회 시 로그 누락 방지)
- [ ] 단위 테스트: 정상 기록, 동시 10건 부하, DB 장애 시 백업 기록 각 검증

## 실행(유연) 프로세스

### 판독문 초안 생성
- [ ] LLM 판단 포인트: 유사 판독문 + 가이드라인 → 소견 기술/결론 초안 생성
- [ ] 프롬프트 설계:
  - 시스템: "영상의학과 전문의를 보조하는 판독문 작성 도우미. 유사 사례와 가이드라인을 참조하여 초안을 작성하되, 최종 판단은 의사에게 위임"
  - 입력: 검사 메타데이터 + 유사 판독문 Top 5 + 가이드라인 해당 섹션
  - 출력: 소견/결론 초안 (JSON 구조)
- [ ] Human-in-the-loop: **필수** — 초안 생성 후 의사 검토 화면 표시, 수정/승인 버튼
- [ ] 불일치 처리: 의사가 대폭 수정 시 → 수정 내용을 피드백으로 수집 (프롬프트 개선용)
- [ ] 패턴 모니터링: 소견 유형별 의사 수정률 추적 → **수정률 5% 이하 시 고정화 검토**
- [ ] 피드백 루프: 의사 수정 이력 → 월 1회 프롬프트/검색 파라미터 튜닝

### 용어 표준화 제안
- [ ] LLM 판단 포인트: 판독문 내 비표준 용어 식별 → 표준 용어 치환 제안
- [ ] 프롬프트 설계:
  - 시스템: "의학용어 표준화 보조. GraphRAG 용어 그래프를 참조하여 비표준 표현을 식별하고 표준 용어로 치환 제안"
  - 입력: 판독문 텍스트 + GraphRAG 매칭 결과 (동의어/약어 후보)
  - 출력: `[{ "original": "비표준", "suggested": "표준", "confidence": 0.95, "context": "문맥" }]`
- [ ] Human-in-the-loop: 제안 목록 표시 → 항목별 수락/거부
- [ ] 고정화 조건: 특정 용어 쌍 수락률 95% 이상 → 자동 치환 후보 등록 (의사 알림 후)

### 소견 요약
- [ ] LLM 판단 포인트: 판독문 전문 → 핵심 소견 추출 + 요약
- [ ] 프롬프트 설계:
  - 시스템: "판독문 핵심 소견을 간결하게 요약. 임상적 중요도 순서로 나열"
  - 입력: 판독문 전문 + 가이드라인 해당 섹션
  - 출력: 구조화 요약 (핵심소견 3~5개, 각 1줄)
- [ ] Human-in-the-loop: 요약 결과 표시 → 의사 확인 (누락/오류 수정)
- [ ] 고정화 조건: 부위별 요약 패턴 안정 시 → 템플릿 기반 반자동 전환
