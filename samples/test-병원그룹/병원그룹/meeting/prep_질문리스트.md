# 병원그룹 미팅 질문 리스트

## 미팅 기본 정보
- 예상 소요: 60분
- 거버넌스 무게: **Heavy** (의료 업종 + 환자 데이터 관련 AI)
- 2차 미팅 필요 예상: **예** (규제/컴플라이언스 확인 필요)

## 반드시 물어볼 것 (프로필 빈칸)

### 1. AI/LLM 현황 — ⬜ 정보 없음
- Q1. "영상의학과 판독 보조 AI 외에 다른 AI/자동화 시도를 해본 적 있으신가요?"
- Q2. "PoC나 파일럿을 진행한 경험이 있다면 결과가 어떠셨나요?"
- Q3. "현재 판독 보조와 관련해서 특정 벤더 제품을 검토 중이신가요, 자체 개발을 고려하시나요?"
- (왜 물어보는지): 기존 AI 시도 경험과 남은 자산이 있으면 접근 방식이 완전히 달라짐. 실패 경험이 있으면 같은 실수를 피해야 함

### 2. 인프라 — 🟡 EMR만 확인
- Q1. "EMR 시스템은 어떤 제품을 쓰고 계신가요?" (유비케어, 이지케어텍, 비트컴퓨터 등)
- Q2. "PACS(의료영상저장전송시스템)는 어떤 것을 쓰시나요?"
- Q3. "서버/인프라가 자체 전산실(온프레미스)인가요, 클라우드(AWS/Azure 등)도 쓰시나요?"
- Q4. "본원과 분원 간 네트워크/데이터 연동 구조가 어떻게 되어 있나요?"
- (왜 물어보는지): 의료 데이터는 외부 반출 제한이 많아 온프레미스/하이브리드 구성이 필수일 수 있음. EMR-PACS 연동 구조에 따라 기술 아키텍처가 결정됨

### 3. 규제 환경 — ⬜ 정보 없음
- Q1. "병원 내 개인정보보호 담당자나 정보보안팀이 있으신가요?"
- Q2. "의료기기 인허가(식약처) 관련해서 내부에서 검토해 본 적 있으신가요?"
- Q3. "환자 데이터의 비식별화/가명화 프로세스가 이미 있으신가요?"
- Q4. "IRB(기관생명윤리위원회) 심의 절차가 필요한 경우가 있을까요?"
- (왜 물어보는지): 의료 AI는 식약처 인허가, 개인건강정보 보호, 의료기기 규제가 모두 걸림. 사전에 내부 거버넌스 체계 유무를 파악해야 로드맵을 설계할 수 있음

### 4. 제약사항 — ⬜ 정보 없음
- Q1. "이번 AI 도입 검토에 별도 예산이 배정되어 있나요, 아니면 예산 확보부터 해야 하나요?"
- Q2. "CTO님이 말씀하신 '검토'의 기한이 있나요? 언제까지 결과를 보고하셔야 하나요?"
- Q3. "IT팀 인력 중 이 프로젝트에 투입 가능한 분이 몇 명이나 되시나요?"
- Q4. "이 AI가 의사/환자 중 누가 직접 사용하게 되나요?"
- (왜 물어보는지): 예산 미확정 + 일정 미정이면 PoC 단계에서 멈출 가능성이 높음. 투입 인력과 사용자 범위에 따라 접근의 무게감이 완전히 달라짐

## 업종 특화 질문

### 의료에서 꼭 확인할 것

#### 개인건강정보(PHI) 처리
- Q. "AI 학습이나 추론에 실제 환자 데이터를 사용할 계획이신가요, 아니면 비식별 데이터만 쓰시나요?"
- Q. "현재 환자 데이터가 어떻게 분류/관리되고 있나요?" (개인정보, 민감정보, 의료정보)
- (놓치면 안 되는 이유): 의료법, 개인정보보호법, 생명윤리법이 동시 적용. 환자 데이터가 LLM에 들어가면 규제 복잡도가 급격히 상승

#### 의료기기 인허가
- Q. "판독 보조 AI가 의사의 '참고용'인가요, 아니면 자동 판독 결과를 환자에게 직접 전달하나요?"
- Q. "식약처 SaMD(Software as a Medical Device) 인허가를 고려하고 계신가요?"
- (놓치면 안 되는 이유): 의사 보조 vs 독립 판독에 따라 의료기기 등급이 달라짐. 등급에 따라 인허가 기간이 6개월~2년 차이

#### 다기관 데이터 거버넌스
- Q. "본원과 분원의 환자 데이터를 통합해서 AI에 활용할 계획이신가요?"
- Q. "분원별로 EMR/PACS가 동일한 시스템인가요, 다른 시스템인가요?"
- (놓치면 안 되는 이유): 다기관 데이터 통합은 데이터 주권, 동의 범위, 네트워크 구조 문제가 추가됨

## 초반에 반드시 확인 (거버넌스 무게 결정)

- Q. **"이 AI를 환자가 직접 사용하나요, 의사/방사선사가 참고용으로만 사용하나요?"**
  - 환자 직접 사용 = 거버넌스 Very Heavy + 의료기기 인허가 필수
  - 의사 참고용 = 상대적으로 Light하지만 여전히 의료법 적용

- Q. **"규제 기관(식약처, 보건복지부) 검토를 받아야 하나요?"**
  - 예 = 인허가 로드맵 별도 필요 (최소 6개월 추가)

- Q. **"환자 데이터가 병원 밖으로 나갈 수 있나요?"**
  - 불가 = 온프레미스/로컬 구성 필수 → 아키텍처 대폭 변경

## 미팅 중 감지할 신호

- "판독문 표현이 의사마다 달라서..." / "본원이랑 분원이 쓰는 용어가 달라요" → **GraphRAG 필요 신호** (의료 용어 표준화 이슈)
- "데이터 외부 반출 금지" / "클라우드 안 돼요" → **온프레미스/로컬 LLM 분기** (GPU 인프라 확인 필요)
- "나중에 환자 챗봇도 하고 싶고..." → **2단계 접근 필요** (1차 내부→2차 외부, 규제 격상)
- "위에서 아직 구체적으로 말 안 해줬어" → **스폰서 미확정**, PoC 범위 축소 검토
- "식약처는 잘 모르겠어요" → **인허가 인식 부족**, 교육 세션 필요 신호
- "비용은 나중에..." → **예산 미확보**, 가벼운 PoC부터 시작하는 전략 필요
