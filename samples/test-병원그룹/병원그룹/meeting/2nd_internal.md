## 기술 신호
- 거버넌스 무게: **Medium** — 내부 전용 + 보조수단 판정 + 가명처리 확립 + IRB 간편심의
- GraphRAG 필요: **강함** — 1차 "의사마다 표현이 달라서" + 2차 "의학용어를 못 알아들어서" (이전 PoC 실패 원인)
- 인프라 경로: **온프레미스 로컬 LLM** — A100 2장, 연구팀 시간 분할
- PoC 이력: **있음** — 작년 챗봇 PoC 실패 (도메인 특화 부족)

## 감지된 리스크
- **GPU 공유**: 연구팀과 A100 공유 → 프로덕션 시 GPU 추가 확보 필요
- **이전 PoC 실패 트라우마**: "의학용어를 못 알아들어서" → 이번 PoC에서 의료 용어 특화가 핵심. 초기에 용어 인식 정확도를 데모해야 신뢰 확보
- **2단계 기대**: "나중에 환자 설명용으로도" → 기대 관리 필요. 환자 대면 시 거버넌스 Heavy 격상 + 식약처 인허가 이슈
- **분원 확장 복잡도**: 본원 PoC 이후 분원 데이터 통합 시 네트워크/데이터 주권 이슈 추가
