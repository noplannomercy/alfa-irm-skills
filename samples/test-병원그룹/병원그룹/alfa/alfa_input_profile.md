# 병원그룹 ALFA 입력 프로필 (확정)

> 확정일: 2026-03-25
> 버전: v1.0
> 근거 미팅: 1차 미팅 (2026-03-25), 2차 미팅 (2026-03-25)

## 1. 회사 개요
- 업종: 의료 (병원 그룹)
- 규모: 중견 — 서울 본원 500병상 + 지방 분원 3개 (각 150~200병상)
- 연간 영상 촬영: 약 12만건 (본원 기준)

## 2. AI/LLM 현황
- 기존 시도: 작년 외부 업체 챗봇 PoC → 의학용어 이해 부족으로 중단
- 교훈: 도메인 특화 없이는 의료 현장에서 쓸 수 없다는 공감대 형성
- 보유 자산: GPU 서버 A100 2장 (연구용, 주중 업무시간 우선 사용 합의)

## 3. 인프라
- 환경: 온프레미스 전용 (데이터 외부 반출 절대 불가)
- EMR: 자체개발, 온프레미스 서버 운영
- PACS: Infinitt, 본원/분원 동일, HL7 FHIR 연동 가능
- GPU: A100 2장 (연구팀 공유)
- 접근권한: 기존 EMR 권한체계, PACS 읽기전용 연동
- 감사로그: 기존 EMR 감사로그에 AI 조회 이력 추가
- 데이터 보관: 원내 서버, 학습용 데이터셋은 별도 격리 스토리지

## 4. 규제 환경
- 데이터 분류: 개인건강정보(PHI) — 가명처리 후 LLM 투입 가능 (내부 검토 완료)
- 가명처리: 보건복지부 가이드라인 준수 (이름/생년월일/주민번호 마스킹)
- IRB: 간편심의 가능 (후향적 데이터, 2주 소요 예상)
- 보조수단 판정: AI 결과를 의사가 최종 확인 → 보조수단으로 인정 (의료정보보안팀장 확인)
- 컴플라이언스: 의료정보보안팀 3명, 개인정보보호책임자는 행정부원장
- 의사결정: 의료정보위원회에서 AI 안건 심의

## 5. 원하는 것
- 스폰서: CTO
- 유즈케이스:
  - 1순위: CT 판독문 보조 (PoC 대상)
  - 2순위: 간호기록 요약 (PoC 이후 검토)
  - 장기: 환자 설명용 활용 (현재 범위 외)
- 사용자: 의사/간호사 (내부 전용, 환자 비노출)
- RACI: 실행=의료정보팀, 검토=영상의학과, 승인=의료정보위원회, 보고=CTO

## 6. 제약사항
- 예산: PoC 기준 5천만원 이내
- 일정: 4월 중순 시작 목표 (IRB 2주 + 데이터 준비 2주)
- 인력: 의료정보팀 3명 (인프라 1명 + 개발 2명)
- PoC 범위: 본원 데이터만 (분원 제외)
- 확장 조건: CT 성과 확인 후 MRI 확장
