## 실행(확정)

### 반품 판단 (절차서 기반 ✅)
- [ ] 5단계 파이프라인 구현 (주문확인→배송확인→절차조회→규칙판단→안내)
- [ ] 규칙: 수령 후 7일 이내 + 미개봉 + 미파손 → 가능
- [ ] 단위 테스트 (가능/불가 각 5건 = 10건)

### 개인정보 마스킹 (정책 기반 ✅)
- [ ] 이름 패턴 (한글 2~4자 → 홍*동)
- [ ] 전화번호 패턴 (010-XXXX-XXXX → 010-****-5678)
- [ ] 주소 패턴 (시/구까지만)
- [ ] LLM 투입 전 자동 적용 훅
- [ ] 원본 미저장 확인
- [ ] 탐지율 테스트 (50건 샘플)

## 실행(유연)

### 클레임 처리 안내 (가이드 기반 ✅)
- [ ] LLM 프롬프트: 유형 판단 (파손/분실/지연/오배송)
- [ ] 유형별 보상 기준 내장:
  - 파손: 사진 → 100% 환불/교환
  - 분실: 택배사 확인 → 보상
  - 지연: 기간별 쿠폰/환불
  - 오배송: 회수 + 재발송
- [ ] Human-in-the-loop UI
- [ ] 고정화 모니터링 (3회 연속 동일 판단 → 알림)
