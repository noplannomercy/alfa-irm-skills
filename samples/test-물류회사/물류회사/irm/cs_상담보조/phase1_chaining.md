# CS 상담 보조 — 체이닝 설계

> 설계 신뢰도: ✅

## 체이닝 패턴

| # | 질의 | 체이닝 순서 | GraphRAG | 신뢰도 |
|---|------|-----------|:--------:|:------:|
| 1 | "반품 처리해줘" | 탐색→접근→실행(확정) | N | ✅ |
| 2 | "이 클레임 어떻게?" | 접근→탐색→실행(유연) | N | ✅ |

## 체이닝 플로우 상세

### 패턴 1: 반품 접수
```
"반품 처리해줘" →
  ① 탐색: cs_knowledge에서 반품 절차 조회 [✅ 절차서 확인됨]
  ② 접근: cs order + delivery track [✅ ERD+API 확인됨]
  ③ 실행(확정): 7일/미개봉/미파손 규칙 판단 [✅ 기준 확인됨]
```

### 패턴 2: 클레임 처리
```
"이 클레임 어떻게?" →
  ① 접근: cs history 조회 [✅]
  ② 탐색: cs_knowledge에서 클레임 가이드 [✅ 가이드 확인됨]
  ③ 실행(유연): LLM 유형 판단 → 상담사 확인 [✅ 보상 기준 확인됨]
```

### 패턴 3: 답변 생성
```
"지연 안내 보내야 해" →
  ① 접근: delivery track [✅]
  ② 탐색: cs_knowledge에서 안내 문구 [✅]
  ③ 실행(유연): LLM 초안 생성 → 상담사 검토 [✅]
```
