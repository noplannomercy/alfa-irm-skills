## 구현 순서 (의존성 기반)

```
Phase A: 인프라 (0.5주)
  └─ pgvector 확장 + Bedrock 설정

Phase B: 탐색 (1주) — B-1, B-2 병렬
  ├─ B-1: knowledge_base (FAQ+매뉴얼 파싱+임베딩)
  └─ B-2: consultation_history (CRM 동기화)
  └─ 의존: Phase A

Phase C: 접근 (0.5주) — Phase B와 병렬
  └─ cli-crm + 개인정보 마스킹
  └─ 의존: Phase A

Phase D: 실행 + UI (1주)
  └─ 답변 생성 프롬프트 + 상담사 웹 UI
  └─ 의존: Phase B + C

Phase E: 통합 + 테스트 (0.5주)
  └─ 체이닝 테스트 + 개인정보 필터 검증
  └─ 의존: Phase D

Phase F: 파일럿 (1주)
  └─ 상담사 5명 실 운영
  └─ 의존: Phase E
```

**병렬화:**
```
A → B-1 ──→ D → E → F
  → B-2 ──┘
  → C ────┘
```

**합계: 4.5주 (6주 마감 대비 1.5주 여유)**
