# cs_상담보조 — 실행 배치 설계

> 작성일: 2026-04-11
> 참조: IRM 실행가이드 v10

## 실행(확정) — 결정적 파이프라인

해당 없음. 상담 답변은 문의 내용마다 다르므로 결정적 파이프라인 불가.

## 실행(유연) — 판단 기반 프로세스

### 프로세스 1: 답변 초안 생성

```
Step 1: [입력] 상담사가 고객 문의 내용 입력
Step 2: [탐색] knowledge_base에서 관련 FAQ/매뉴얼 검색
Step 3: [탐색] consultation_history에서 유사 과거 응대 검색
Step 4: [접근] CRM에서 해당 고객 이력 조회 (해당 시)
Step 5: [LLM 판단] 검색 결과 종합 → 답변 초안 생성
Step 6: [사람 확인] 상담사가 초안 검토/수정 후 고객에게 전송
```

- 판단 포인트: **Step 5** — LLM이 FAQ+매뉴얼+이력을 종합하여 답변 작성
- Human-in-the-loop: **Step 6** — 상담사가 최종 확인 후 전송
- 불일치 시 처리: 상담사가 크게 수정한 경우 피드백 기록
- 고정화 조건: 동일 카테고리 질문에서 수정 없이 5회 연속 전송 → FAQ 자동응답 후보 (고객 직접 챗봇 전환 검토 트리거)

### 개인정보 필터링 (Step 4~5 사이)
- CRM 응답에서 이름/전화/주소 마스킹 후 LLM 전달
- LLM 답변 초안에 개인정보 포함 여부 사후 검증
