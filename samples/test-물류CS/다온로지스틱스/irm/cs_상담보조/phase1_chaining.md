## 체이닝 패턴

| # | 질의 | 체이닝 순서 | GraphRAG | 신뢰도 |
|---|------|-----------|:--------:|:------:|
| 1 | "이 문의에 답변 초안 만들어줘" | 탐색→탐색→접근→실행(유연) | N | ✅ |

## 체이닝 플로우 상세

### 패턴 1: 답변 초안 생성 (메인 플로우)

```
[입력] 고객 문의 내용
    │
    ▼
① [탐색] knowledge_base: FAQ + 매뉴얼 검색
    │   → 관련 FAQ/매뉴얼 top-5
    │
    ▼ (병렬)
② [탐색] consultation_history: 유사 과거 응대 검색
    │   → 유사 상담 top-5
    │
    ▼ (선택적 — 고객 정보 있을 때)
③ [접근] cli-crm: 해당 고객 이력 조회
    │   → 개인정보 마스킹 후 전달
    │
    ▼
④ [실행(유연)] LLM 답변 초안 생성
    │   → ①②③ 결과 종합
    │
    ▼
⑤ [Human-in-the-loop] 상담사 검토/수정/전송
```

**병렬화:**
```
① ──┐
② ──┤→ ④ → ⑤
③ ──┘
```
①②③ 모두 병렬 실행 가능.

**예상 소요 시간:** 10~20초 (현재 수동 FAQ 검색 3~5분 → 90% 단축)
