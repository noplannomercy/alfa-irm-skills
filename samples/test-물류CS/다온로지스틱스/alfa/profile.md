# 다온로지스틱스 고객 프로필

> 작성일: 2026-04-08
> 최종 수정: 2026-04-11
> 상태: 1차 미팅 완료 — 프로필 완성

## 1. 회사 개요
- 업종: 택배 물류 (B2C 중심)
- 규모: 직원 200명 (CS팀 상담사 15명)
- 주력 사업: 택배 배송, 일 평균 3만건, 주요 고객 이커머스 셀러 [1차 미팅 확인]

## 2. AI/LLM 현황
- 기존 시도: 1년 전 규칙 기반 챗봇 도입 → 3개월 만에 철거 ("멍청해서") [1차 미팅 확인]
- 결과: 실패. 상담사들 AI 불신 있음 [1차 미팅 확인]
- 남은 자산: 없음 [1차 미팅 확인]

## 3. 인프라
- 클라우드/온프레미스: AWS (EC2, RDS, S3) [1차 미팅 확인]
- DB: PostgreSQL (RDS) — 자체 CRM, REST API 있음 [1차 미팅 확인]
- Bedrock: 사용 가능 환경 [1차 미팅 확인]
- 네트워크/보안: 클라우드 기반, 특별 제한 없음

## 4. 규제 환경
- 감독기관: 없음 [1차 미팅 확인]
- 컴플라이언스팀: 없음 — 대표이사 직접 승인 [1차 미팅 확인]
- 데이터 분류: 상담 내용에 개인정보(이름, 전화번호, 주소) 포함 [1차 미팅 확인]
- 개인정보처리방침: 있으나 AI 관련 조항 없음 → 추가 필요 [1차 미팅 확인]

## 5. 원하는 것
- 경영진 지시: CS팀장 실무 의사결정, 대표이사 승인
- 현업 니즈: 반복 질문(전체 60%)에 대한 상담 시간 단축 [1차 미팅 확인]
- **유즈케이스: CS 상담 보조 (내부 도구) — 확정** [1차 미팅 확인]
  - 상담사가 질문 입력 → FAQ/매뉴얼 검색 → 답변 초안 제시
  - 고객 직접 대면 아님 (내부 상담사용)
  - 향후: 고객 직접 챗봇, 물류센터 매뉴얼, 셀러 문의 대응으로 횡전개
- 반복 질문 top 3: 배송 추적 / 반품 절차 / 집하 예약 방법 [1차 미팅 확인]
- 데이터 현황:
  - FAQ 300건 — 구글 시트, 3개월 내 업데이트 [1차 미팅 확인]
  - 상담 매뉴얼 50건 — Notion [1차 미팅 확인]
  - 상담 이력 — CRM(PostgreSQL), REST API, 2년치 50만건 [1차 미팅 확인]
- 상담 채널: 카카오톡 60% / 전화 30% / 이메일 10% [1차 미팅 확인]

## 6. 제약사항
- 예산: 2천만원 [사전 정보 + 1차 미팅 확인]
- 일정: 6주 내 파일럿 결과 보고 [1차 미팅 확인]
- 인력: IT담당 오민수 1명 풀타임 투입 가능 [1차 미팅 확인]
- 고객 대면 여부: 내부 직원(상담사)만 사용. 고객 직접 대면은 향후 검토 [1차 미팅 확인]

## 내부 메모
- 거버넌스 무게: Light (물류 + 내부용 + 규제기관 없음)
- GraphRAG 신호: 없음
- 예상 미팅 횟수: 1회 (완료 — 프로필 다 채워짐)
- 예상 로드맵 유형: 경량 4주 (클라우드 OK + Bedrock + 규제 Light)
- ⚠️ 상담사 AI 불신: 이전 챗봇 실패 경험 → 파일럿 시 UX/신뢰 확보 중요
- ⚠️ 개인정보: 상담 내용에 포함 → LLM 투입 시 가명처리 또는 비식별화 검토
