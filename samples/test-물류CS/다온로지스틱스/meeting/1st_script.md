# 다온로지스틱스 1차 미팅 메모

> 일시: 2026-04-11
> 참석: 한지영 CS팀장, 오민수 IT담당(1인), 우리 측 2명

## 미팅 내용 (원문)

- 택배 물류 전문. B2C 중심, 일 평균 배송 3만건. 주요 고객은 이커머스 셀러.
- CS팀 상담사 15명, 하루 평균 상담 800건. 채널은 카카오톡(60%) + 전화(30%) + 이메일(10%).
- 반복 질문 top 3: "배송 어디까지 왔어요?", "반품 어떻게 해요?", "집하 예약 방법 알려주세요"
- CS팀장: "이 세 가지가 전체 상담의 60%를 차지. 상담사가 매번 FAQ 뒤져서 답하고 있음"
- FAQ 300건 — 구글 시트에 정리되어 있음. 최근 3개월 내 업데이트.
- 상담 매뉴얼 50건 — Notion에 있음. 배송 절차, 클레임 처리, 요금 안내 등.
- 상담 이력은 자체 개발 CRM에 저장. PostgreSQL. REST API 있음. 최근 2년치 약 50만건.
- AWS 사용 중: EC2, RDS(PostgreSQL), S3. Bedrock 쓸 수 있는 환경.
- 개인정보: 상담 내용에 고객 이름, 전화번호, 주소 포함됨. 개인정보처리방침은 있으나 AI 관련 조항은 없음.
- AI/챗봇 경험: 1년 전에 규칙 기반 챗봇 도입했다가 "멍청해서" 3개월 만에 철거. 상담사들 불신 있음.
- 이번에 원하는 건: 고객 직접 대화 아님. 상담사가 참고하는 내부 보조 도구. 상담사가 질문 입력하면 관련 FAQ/매뉴얼 찾아주고 답변 초안 제시.
- 나중에 잘 되면 고객 직접 대면 챗봇도 검토하겠지만, 지금은 내부용으로 확정.
- CS팀장: "우선 CS팀에서 확실하게 효과 보고, 그 다음에 물류센터 현장 매뉴얼 검색이나 셀러 문의 대응으로 넓히고 싶다"
- 예산 2천만원, 6주 내 파일럿 결과 보고 원함.
- IT담당 오민수가 혼자 전담. 풀타임 투입 가능. "Bedrock이면 바로 쓸 수 있다"
- 별도 컴플라이언스팀 없음. 대표이사가 직접 승인. CS팀장이 실무 의사결정.
