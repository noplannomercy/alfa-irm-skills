## 기술 신호
- 거버넌스 무게: **Light** — 내부 직원용, 규제기관 없음, 대표이사 직접 승인
- GraphRAG 필요: **아니오** — 용어 불일치 신호 없음, FAQ가 구조화되어 있음
- 인프라 경로: **클라우드 API (AWS Bedrock)** — 온프레미스 불필요
- PoC 이력: **있음 (실패)** — 규칙 기반 챗봇 3개월 만에 철거. 원인: 규칙 기반 한계, "멍청" → LLM 기반 RAG로 접근하면 해소 가능

## 감지된 리스크
- **상담사 AI 불신**: 이전 실패 경험 → 파일럿 시 UX 중요, 점진적 도입, 상담사 피드백 루프 필수
- **개인정보 포함**: 상담 내용에 이름/전화/주소 → Bedrock 사용 시 데이터 학습 안 됨(기본), 가명처리 또는 개인정보 필터링 검토
- **개인정보처리방침 AI 조항 부재**: 추가 필요 → 법적 리스크는 낮지만 정비 권고
