# 다온로지스틱스 미팅 질문 리스트

## 미팅 기본 정보
- 예상 소요: 60분
- 거버넌스 무게: Light (현재 판단 — 물류업, 내부용 추정)
- 2차 미팅 필요 예상: 아니오 (유즈케이스 이미 명시, 1회로 충분할 가능성)

## 반드시 물어볼 것 (프로필 빈칸)

### 🟡 항목 1 — 회사 개요 (보강)
- Q1. 주력 사업이 택배/창고/포워딩 중 어디인가요?
- Q2. CS팀이 대응하는 고객이 B2B(기업)인가요, B2C(개인)인가요?
- (왜 물어보는지): 고객 유형에 따라 질문 패턴과 복잡도가 다름

### ⬜ 항목 2 — AI/LLM 현황
- Q1. 이전에 챗봇이나 AI 관련 시도해본 적 있으신가요?
- Q2. 현재 상담 채널이 뭔가요? (전화/카톡/웹챗/이메일)
- (왜 물어보는지): 기존 챗봇 실패 이력이 있으면 원인 파악 필요. 채널에 따라 연동 방식 달라짐

### ⬜ 항목 3 — 인프라 (보강)
- Q1. AWS에서 어떤 서비스 쓰고 계세요? (EC2, RDS, S3 등)
- Q2. FAQ나 매뉴얼이 어디에 저장되어 있나요? (S3, 사내 위키, 파일서버 등)
- Q3. 상담 이력 DB가 있나요? 어떤 시스템?
- (왜 물어보는지): AWS면 Bedrock 활용 가능, 리소스 위치에 따라 파이프라인 달라짐

### ⬜ 항목 4 — 규제 환경
- Q1. 상담 내용에 개인정보(이름, 전화번호, 주소)가 포함되나요?
- Q2. 고객 데이터 처리에 대한 내부 규정이 있나요?
- (왜 물어보는지): 물류는 보통 Light지만, 배송 정보에 개인정보 포함 시 처리 주의

### 🟡 항목 5 — 원하는 것 (구체화)
- Q1. "AI 챗봇"이라고 하시면, 상담사가 참고하는 보조 도구인가요, 고객이 직접 대화하는 챗봇인가요?
- Q2. 가장 많이 들어오는 반복 질문 유형 top 3이 뭔가요?
- Q3. FAQ가 몇 건 정도 되나요? 최근에 업데이트한 적 있나요?
- Q4. 매뉴얼이라 하시면 어떤 매뉴얼? (배송 절차, 클레임 처리, 요금 안내 등)
- (왜 물어보는지): 내부 보조 vs 고객 직접 = 거버넌스 무게 결정. FAQ/매뉴얼 상태가 RAG 품질 결정

### ⬜ 항목 6 — 제약사항 (보강)
- Q1. 2천만원이면 언제까지 결과를 보고 싶으신가요?
- Q2. 내부에서 이 프로젝트에 투입할 수 있는 인력이 있나요? (IT담당 등)
- Q3. 상담사 중 파일럿에 참여할 수 있는 분이 있나요?
- (왜 물어보는지): 일정/인력 확인. 파일럿 참여자가 있어야 피드백 가능

## 업종 특화 질문

### 물류에서 꼭 확인할 것
- Q. 고객이 상담 채널에서 직접 AI와 대화하게 할 계획이 있나요?
  → (놓치면 안 되는 이유): 고객 직접 대면 = 거버넌스 격상 + 오답 책임 소재
- Q. 상담 내용에 운송장번호, 주소 등 개인정보가 포함되나요?
  → (놓치면 안 되는 이유): 개인정보 포함 시 LLM 투입 범위 제한 가능

## 초반에 반드시 확인 (거버넌스 무게 결정)
- Q. "이 서비스를 고객(외부)이 직접 사용하나요, 내부 직원이 사용하나요?"
  → 고객 대면 = 거버넌스 Medium~Heavy, 내부 = Light
- Q. "규제 기관 검토를 받아야 하나요?"
  → 물류는 보통 불필요

## 미팅 중 감지할 신호
- "표현이 달라서..." / "같은 건데 다르게 불러요" → GraphRAG 필요 신호
- "고객 챗봇도 하고 싶고..." → 2단계 접근 필요 (내부→외부)
- "위에서 아직 말 안 해줬어" → 스폰서 미확정
- "전에 챗봇 해봤는데 안 됐어" → PoC 실패 이력, 실패 원인 파악 필수
- "상담사마다 답변이 달라" → FAQ 정리 안 됨, 표준화 필요
