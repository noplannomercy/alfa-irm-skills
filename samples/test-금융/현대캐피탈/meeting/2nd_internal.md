## 기술 신호
- 거버넌스: **Very Heavy (확정)** — 보조수단성 + 신용정보법 + 감사로그 + AI감사 + 모델정책문서
- GraphRAG: **필수** — "담보 평가"가 여신규정/자동차특약/리스특약에 각각 다르게 존재. 상품-규정 매핑 없으면 PoC와 같은 실패.
- 인프라: AWS Bedrock (CSAP OK) — 오토에버가 세팅

## 컴플라이언스 요구사항 정리 (구현에 직접 영향)

| 요구 | 구현 영향 |
|------|----------|
| 보조수단성 | AI 추천 ≠ 최종 결정. 심사역 override 가능 UI. |
| 차이 로그 | AI 추천 vs 심사역 결정이 다를 때 자동 기록. |
| 근거 표시 | 의견서에 "AI 보조, 참조: 여신규정 제X조 제X항" 표시. |
| AI 호출 로그 | 입력/출력/타임스탬프/사용자 전수 기록. |
| AI 감사 | 분기 1회 감사 리포트 생성 기능 필요. |
| 모델 정책 | Bedrock Claude 데이터 처리 정책 문서 제출. |
| 신용정보 차단 | CBS 신용정보가 LLM에 절대 전달되지 않는 구조적 보장. |

| 실패 원인 | 대응 | 2차에서 추가 확인된 것 |

| 규정 통째 RAG | 컬렉션 분리 + 메타 필터 | 편→장→조→항 계층 구조 활용 |
