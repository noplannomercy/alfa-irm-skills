# 현대캐피탈 2차 미팅 메모

> 일시: 2026-04-21
> 참석: 이준혁 DX팀장, 송미래 DX과장, 최동원 심사기획차장, 강서윤 컴플라이언스팀장, 박재현 정보보호팀 과장, 오토에버 김성호 PM, 우리 측 2명

## 미팅 내용 (원문)

- 컴플라이언스팀장 강서윤이 첫마디: "AI가 심사에 관여한다는 건 금감원 검사 대상이 될 수 있다는 뜻입니다. 보조수단이라고 해도 검사관이 '이거 AI가 결정한 거 아니냐'고 물으면 대답할 수 있어야 합니다."

- 보조수단성 기준 (강서윤):
  - AI는 정보를 제공만. 심사역이 AI 결과를 무시하고 다른 결정을 내릴 수 있어야 함.
  - AI 추천과 심사역 최종 결정이 다른 경우의 기록 필수.
  - "AI가 승인 추천했는데 심사역이 거절, 또는 그 반대 — 이런 케이스를 로그로 남겨야 한다."
  - 심사 의견서에 "AI 보조 활용" 명시 필요. 어떤 규정을 참조했는지 근거 표시.

- 신용정보 관련 (강서윤):
  - "신용정보를 LLM에 넣는 건 절대 안 됩니다. 신용정보법 제32조 위반."
  - "다만 규정집, 내규, 약관은 신용정보가 아니니까 RAG에 넣어도 됩니다."
  - "심사 이력을 넣으려면 완전 비식별화 후에만. 근데 비식별화하면 의미가 있는지 모르겠네."
  - 이준혁 팀장: "그러면 규정/내규 검색 + 의견서 초안(규정 근거)만 하는 거죠?"
  - 강서윤: "그 범위면 문제없습니다. 근데 '규정 근거를 AI가 찾아줬다'는 것도 기록해야 해요."

- 정보보호팀 박재현:
  - AWS Bedrock: 현대캐피탈은 AWS 금융 리전 사용 중. Bedrock은 금융권 CSAP 인증된 상태.
  - "Bedrock이면 데이터가 AWS 밖으로 안 나가고, 모델 학습에도 안 쓰인다. 이건 괜찮습니다."
  - 다만 "어떤 모델을 쓸 건지, 모델별 데이터 처리 정책 문서를 제출해야 합니다."
  - 감사 로그: 금감원 검사 대비 — 모든 AI 호출에 대해 입력/출력/타임스탬프/사용자 기록 필수.
  - "연 1회 이상 AI 활용 내부 감사를 돌려야 합니다. 파일럿 중에도 분기 1회."
  - 데이터 암호화: 전송 중 TLS 필수, 저장 시 AES-256. "AWS 기본이긴 한데 확인은 해주세요."

- 오토에버 김성호 PM:
  - "인프라 쪽은 저희가 Bedrock 활성화하고 VPC 세팅하겠습니다. IAM 권한도."
  - "AI 모델 선택이랑 RAG 설계는 여러분(우리)이 해주시면, 저희가 CI/CD 파이프라인 깔겠습니다."
  - "규정집 PDF를 텍스트로 변환하는 건 이미 해놨습니다. 근데 품질이 좀..." (PoC 잔여 자산)
  - 이준혁 팀장: "그 품질 문제 때문에 PoC가 망한 거잖아. 이번엔 파싱부터 다시 하는 게 맞지 않나?"

- 규정집 상세 (최동원 차장):
  - 규정집 = 여신규정 + 자동차금융 특약 + 리스 특약 + 내부 심사 가이드라인
  - 구조: 편(chapter) → 장(section) → 조(article) → 항(paragraph)
  - "같은 '담보 평가' 조항이 여신규정, 자동차금융 특약, 리스 특약에 각각 다르게 있어요. 이걸 구분 못 하면 의미 없습니다."
  - 규정 개정: 분기 1회 정기 개정 + 수시 개정. 개정 이력 관리 중 (버전 관리).
  - 심사 가이드라인: 약 200페이지 별도 문서. 규정 해석 방법 + 사례집.
  - "심사역들이 진짜 원하는 건 — 이 건에 적용되는 규정이 뭔지 찾아주고, 비슷한 과거 심사 사례에서 어떻게 판단했는지 알려주는 것."

- 유즈케이스 확정:
  - 강서윤: "규정 검색 + 의견서 초안, 신용정보 빼고. 이 범위면 진행 가능합니다."
  - 합의: **"자동차 할부 심사 규정 검색 + 심사 의견서 보조"** 확정.
  - 파일럿 대상: 본사 심사팀 심사역 10명.
  - 성공 기준 (이준혁 팀장): "심사역이 '이거 쓸만하다'고 말하면 성공. 정량으로는 규정 검색 시간 50% 단축."

- 고객 대면: 아님. 심사역 내부 도구. 딜러나 고객에게 노출 안 됨.
