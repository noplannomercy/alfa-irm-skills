# 현대캐피탈 미팅 질문 리스트

## 미팅 기본 정보
- 예상 소요: 60~90분
- 거버넌스 무게: Very Heavy (금융 — 금감원, 신용정보법)
- 2차 미팅 필요 예상: 예 (컴플라이언스팀 동석 필수)

## 반드시 물어볼 것 (프로필 빈칸)

### 🟡 항목 1 — 회사 개요 (보강)
- Q1. 디지털혁신팀 규모가 어떻게 되나요? 이번 프로젝트 담당자가 누구인가요?
- Q2. 심사 업무라고 하시면 자동차 할부 심사인가요, 개인대출 심사인가요, 전체인가요?
- Q3. 심사 팀 인원이 몇 명이고, 하루 심사 건수가 어느 정도인가요?
- (왜 물어보는지): "심사"가 범위가 넓음. 어떤 심사인지에 따라 데이터/규제/프로세스 완전 다름.

### 🟡 항목 2 — AI/LLM 현황 (PoC 실패 파악 핵심)
- Q1. 1년 전 PoC에서 구체적으로 뭘 시도했나요? (어떤 업무에 AI를 적용?)
- Q2. 실패 원인이 뭐라고 판단하세요? (기술 문제? 데이터 문제? 현업 수용성?)
- Q3. PoC에서 남은 자산이 있나요? (데이터, 모델, 문서 등)
- Q4. 오토에버가 이번에 어떤 방식으로 진행하겠다고 제안했나요?
- Q5. 우리(당사)의 역할이 뭔가요? 오토에버와 어떤 관계인가요? (경쟁? 협업? 자문?)
- (왜 물어보는지): PoC 실패 원인을 모르면 같은 실수 반복. 오토에버와의 관계가 프로젝트 구조를 결정.

### 🟡 항목 3 — 인프라 (보강)
- Q1. AWS 어떤 서비스 쓰고 계세요? Bedrock 사용 가능한가요?
- Q2. 심사 시스템(CBS? 자체 개발?)이 뭔가요?
- Q3. 심사 데이터는 어디에 있나요? (어떤 DB, 접근 방식)
- Q4. 현재 AI/ML 인프라가 있나요? (SageMaker 등)
- Q5. "노코드로 할지 개발로 할지 모른다"고 하셨는데, 선호하시는 방향이 있나요?
- (왜 물어보는지): 아키가 없는 상태에서 우리가 방향을 잡아줘야 함. AWS 내에서 뭘 쓸 수 있는지 파악.

### ⬜ 항목 4 — 규제 환경
- Q1. 컴플라이언스팀이 이 프로젝트를 알고 있나요?
- Q2. 금감원에 AI 활용 관련 보고/승인이 필요한가요?
- Q3. 신용정보를 LLM에 투입하는 것에 대한 내부 검토가 있었나요?
- Q4. AI 모델의 심사 의사결정 참여에 대한 규정이 있나요? ("보조수단성" 인식?)
- Q5. 이전 PoC에서 규제 관련 이슈가 있었나요?
- (왜 물어보는지): 금융은 규제가 가장 무거움. 신용정보법 + 금감원 + 보조수단성. 컴플라이언스 모르고 진행하면 나중에 엎어짐.

### 🟡 항목 5 — 원하는 것 (구체화)
- Q1. "심사 업무 AI"라 하시면 구체적으로 어떤 단계를 AI로 하고 싶은 건가요?
  - 서류 접수/확인? 신용 평가? 심사 의견서 작성? 약관/규정 검색? 사기 탐지?
- Q2. 현재 심사 프로세스가 어떻게 되나요? (단계별로)
- Q3. 심사역이 가장 시간 많이 쓰는 업무가 뭔가요?
- Q4. 6개월 파일럿에서 "이것만은 꼭 보여줘야 한다"는 게 있나요?
- Q5. 현업(심사역)이 원하는 게 뭔가요? 경영진이 원하는 게 뭔가요? 같은 건가요?
- (왜 물어보는지): 현업 니즈 vs 경영진 니즈가 다를 수 있음. 6개월 파일럿 성공 기준 합의 필수.

### ⬜ 항목 6 — 제약사항 (보강)
- Q1. 6개월 파일럿의 예산 규모가 어떻게 되나요?
- Q2. 오토에버 외에 투입 가능한 내부 인력이 있나요? (IT, 데이터팀 등)
- Q3. 파일럿 결과를 누구에게 보고하나요? (성공 기준은?)
- Q4. 심사 결과가 고객에게 직접 전달되나요, 내부 심사역 참고용인가요?
- (왜 물어보는지): 고객 대면 여부가 거버넌스 결정. 오토에버 인력 구성도 파악 필요.

## 업종 특화 질문

### 금융에서 꼭 확인할 것
- Q. AI가 심사 의사결정에 참여하나요, 보조 도구인가요?
  → (놓치면 안 되는 이유): "보조수단성" 충족 여부. AI가 의사결정하면 금감원 규제 정면충돌.
- Q. 신용정보(신용점수, 연체 이력, 소득)를 LLM에 넣을 계획인가요?
  → (놓치면 안 되는 이유): 신용정보법 위반 가능. LLM에 신용정보 직접 투입은 극히 제한적.
- Q. 컴플라이언스팀이 2차 미팅에 동석 가능한가요?
  → (놓치면 안 되는 이유): 금융은 컴플라이언스 없이 진행 불가. 나중에 엎어짐.

## 초반에 반드시 확인 (거버넌스 무게 결정)
- Q. "이 AI가 고객 대면인가요, 내부 심사역 보조인가요?"
  → 고객 대면(심사 결과 직접 통보) = Very Heavy
  → 내부 보조 = Heavy (그래도 Heavy)
- Q. "오토에버와 우리의 역할이 어떻게 되나요?"
  → 이거 안 풀면 프로젝트 구조 자체가 안 잡힘

## 미팅 중 감지할 신호
- "오토에버가 하겠다는데..." → 정치적 구도 파악 필수
- "금감원에서 뭐라 할까 봐" → 규제 불확실성, 보조수단성 검토 필요
- "심사역들이 안 쓸까 봐" → PoC 실패 트라우마, UX/신뢰 확보 핵심
- "신용정보는 못 넣지 않나?" → 맞음. LLM에 직접 투입 불가 경로 설계 필요.
- "표현이 달라서..." → GraphRAG 신호 (심사 용어 vs 약관 용어 vs 고객 용어)
- "전에 해봤는데 안 됐어" → PoC 실패 원인 깊이 파고들기
