> 참조: IRM 실행가이드 v10

## 실행(확정) — 해당 없음

## 실행(유연) — 판단 기반 프로세스

### 프로세스 1: 심사 의견서 초안 생성

```
Step 1: [접근] cli-cbs: 심사 건 정보 조회 (신용정보 차단 프록시 경유)
Step 2: [탐색] regulations + GraphRAG: 상품 유형에 맞는 규정 검색
         → GraphRAG가 "자동차 할부"에 APPLIES_TO인 조항만 필터
Step 3: [탐색] guidelines: 유사 심사 사례/해석 검색
Step 4: [탐색] review_history: 유사 과거 심사 (비식별화, 해당 시)
Step 5: [LLM 판단] 종합 → 의견서 초안 생성
         - 반드시 포함: "AI 보조 활용" 표시 + 참조 규정 근거 (제X조 제X항)
         - 설명 가능성: 왜 이 규정을 참조했는지 추적 가능
Step 6: [사람 확인] 심사역이 검토/수정/확정
         - 심사역이 AI 추천과 다른 결정 시 → 차이 로그 자동 기록
```

- 판단 포인트: Step 5
- Human-in-the-loop: Step 6 — 보조수단성 핵심
- 차이 로그: AI 추천 근거 vs 심사역 최종 결정 (다를 경우 사유 기록)
- 감사 대비: 전 과정 입출력 로그 (금감원 검사 시 제출 가능)
- 고정화 조건: 해당 없음 — 심사는 건마다 다르므로 실행(확정) 전환 대상 아님
