## GraphRAG 구성
- pgvector + Apache AGE (같은 PostgreSQL)
- 노드:
  - `Regulation` — 규정 문서 (여신규정, 자동차특약, 리스특약)
  - `Article` — 조항 (제X편 제X장 제X조 제X항)
  - `Product` — 상품 유형 (자동차할부, 자동차리스, 장기렌탈)
  - `Topic` — 심사 주제 (담보평가, 한도산정, 감면, 딜러심사)
  - `Term` — 용어 (동의어/유사어)
- 엣지:
  - `Article` -[BELONGS_TO]→ `Regulation` (조항이 어떤 규정에)
  - `Article` -[APPLIES_TO]→ `Product` (조항이 어떤 상품에 적용)
  - `Article` -[COVERS]→ `Topic` (조항이 어떤 주제를)
  - `Term` -[MEANS]→ `Topic` ("담보 가치" = "담보평가")
  - `Article` -[SUPERSEDES]→ `Article` (개정으로 대체된 관계)
- 핵심 활용:
  - "담보 평가 기준?" → Topic("담보평가") → COVERS 역탐색 → 해당 Article들 → APPLIES_TO로 상품 필터 → 할부 조항만
  - PoC 실패 원인 = 이 "APPLIES_TO 필터"가 없어서 3개 규정 조항이 뒤섞인 것
- 시드 데이터: 규정집 목차에서 편/장/조 구조 추출 + 상품 적용 관계 매핑
