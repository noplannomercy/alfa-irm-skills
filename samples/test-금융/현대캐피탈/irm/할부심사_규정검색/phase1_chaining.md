|---|------|-----------|:--------:|:------:|

## 체이닝 플로우 상세

### 패턴 1: 심사 의견서 초안 (메인 — 6단계)

```
[입력] 심사 건 ID + 질문/항목
    │
    ▼
① [접근] cli-cbs → 심사 건 정보 (신용정보 차단 프록시)
    │   → 상품유형, 차량정보, 서류상태, 딜러정보
    │
    ▼
② [탐색] regulations + GraphRAG
    │   → 상품유형으로 APPLIES_TO 필터 → 해당 조항만 검색
    │   → PoC 실패 원인 해결 지점
    │
    ▼ (병렬)
③ [탐색] guidelines: 유사 사례/해석
④ [탐색] review_history: 유사 과거 심사 (해당 시)
    │
    ▼
⑤ [실행(유연)] LLM 의견서 초안
    │   → "AI 보조 활용" + "참조: 자동차금융특약 제X조 제X항"
    │
    ▼
⑥ [Human-in-the-loop] 심사역 검토/수정/확정
    │   → 차이 시 로그 기록
```

병렬: ③④는 ②와 병렬 가능

### 패턴 2: 상품별 규정 비교 (GraphRAG 핵심)

```
[입력] "담보 평가 기준이 할부랑 리스가 다른가?"
    │
    ▼
① [GraphRAG] Topic("담보평가") → COVERS → Article들
    │   → APPLIES_TO → Product("할부"), Product("리스") 분기
    │
    ▼
② [탐색] regulations: 할부 담보 조항 + 리스 담보 조항 병렬
    │
    ▼
③ [응답] 상품별 차이점 비교 + 각 조항 출처 표시
```

## 예상 소요 시간
- 패턴 1: 15~20초 (현재 규정 검색 반나절 → 95% 단축)
- 패턴 2: 5~10초 (현재 3개 규정 수동 대조 → 자동)
