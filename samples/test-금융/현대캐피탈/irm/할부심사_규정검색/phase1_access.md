## CLI 설계

### cli-cbs (CBS 심사 건 조회)
- 경계: CBS 심사 시스템 (신용정보 제외 항목만)
- 접근: REST API (내부 — 오토에버 세팅)
- 명령어:
  - `cbs case get --id <심사ID>` → 심사 건 상세 (상품유형, 차량정보, 서류상태, 딜러정보)
  - `cbs case list --product <상품유형> --status <상태> --limit` → 심사 건 목록
- 출력: JSON 표준 형식
- 접근 모드: **읽기전용**
- **신용정보 차단**: API 응답에서 신용점수/연체이력/소득 필드 **구조적 제거** (프록시 레이어)
  - CBS API → 프록시(필드 필터링) → CLI → LLM
  - 프록시에서 허용 필드 화이트리스트만 통과 (블랙리스트 아님)
- 감사로그: 모든 조회 기록 (사용자/일시/심사ID/조회 항목)

## REST 게이트웨이
- 오토에버가 CI/CD + VPC 내 API 세팅 담당
- 우리는 CLI + 프록시 필터링 설계

## 신용정보 차단 아키텍처

```
CBS REST API (전체 필드)
    │
    ▼
프록시 레이어 (화이트리스트 필터링)
  허용: product_type, vehicle_info, document_status, dealer_info, application_date
  차단: credit_score, delinquency_history, income, debt_ratio
    │
    ▼
cli-cbs (필터링된 JSON만)
    │
    ▼
LLM (신용정보 절대 도달 불가 — 구조적 보장)
```

**컴플라이언스 요구: "신용정보가 LLM에 절대 전달되지 않는 구조적 보장"** → 프록시 화이트리스트로 충족.
