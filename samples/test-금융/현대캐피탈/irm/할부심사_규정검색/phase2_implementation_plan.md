## 구현 순서

```
Phase A: 인프라 (1주) — 오토에버
  └─ Bedrock 활성화 + VPC + pgvector + AGE + IAM

Phase B: 파싱 (2주)
  └─ 규정집 2,000p → Forge → MD → 조항 단위 청킹
  └─ 가이드라인 200p → Forge → MD
  └─ 의존: Phase A

Phase C: RAG 컬렉션 (1.5주) — Phase B 완료 후
  ├─ C-1: regulations (1주)
  └─ C-2: guidelines (0.5주) — C-1과 병렬

Phase D: GraphRAG (3주) — Phase C와 일부 병렬
  └─ 목차 기반 시드 (편/장/조 구조)
  └─ APPLIES_TO (상품-조항 매핑) ← PoC 실패 해결 핵심
  └─ COVERS (주제-조항 매핑)
  └─ 의존: Phase B (조항 단위 청킹 완료 후)

Phase E: 접근 (1주) — Phase A 이후
  └─ cli-cbs + 신용정보 차단 프록시
  └─ 컴플라이언스팀 프록시 구조 승인

Phase F: 실행 + 감사 (2.5주)
  ├─ F-1: 의견서 생성 + 보조수단성 UI (1.5주)
  └─ F-2: 감사 로그 + AI 감사 리포트 (1주) — F-1과 병렬
  └─ 의존: Phase C + D + E

Phase G: 통합 + 검증 (1.5주)
  └─ 체이닝 + 신용정보 차단 전수 검증
  └─ 컴플라이언스 최종 검증
  └─ 의존: Phase F

Phase H: 파일럿 (4주)
  └─ 심사역 10명 실 운영
  └─ 분기 AI 감사 1회 포함
  └─ 의존: Phase G

Phase I: 효과 분석 + 보고 (2주)
  └─ 경영진 보고서 작성
```

**병렬화:**
```
A → B → C ──→ D ──→ F → G → H → I
       └→ E ──┘
```

**합계: 약 20주 (5월~10월)**
