## 1. 회사 개요
- 업종: 여신금융 (자동차 할부/리스)
- 규모: 심사역 약 80명, 하루 심사 1,500건 (수동 450건) [1차 미팅 확인]
- 접점: 디지털혁신팀 이준혁 팀장 (팀 8명) [1차 미팅 확인]

## 2. AI/LLM 현황
- 기존 시도: 오토에버 PoC 실패 (4원인 식별) [1차 미팅 확인]
- 남은 자산: 규정집 PDF 전환본 (품질 문제), AWS 계정 [1차/2차 확인]

## 3. 인프라
- 클라우드: AWS 금융 리전 [1차 미팅 확인]
- Bedrock: CSAP 인증 완료, 활성화만 필요 [2차 미팅 확인]
- CBS: Oracle DB, 내부 REST API [1차 미팅 확인]
- 오토에버: Bedrock 활성화 + VPC + IAM + CI/CD 담당 [2차 미팅 확인]

## 4. 규제 환경
- 감독기관: 금감원, 금융위 [확인]
- 컴플라이언스: 강서윤 팀장 — 승인 완료 [2차 미팅 확인]
- 보조수단성: AI 정보 제공만, 심사역 무시 가능, 차이 로그 필수 [2차 미팅 확인]
- 신용정보: LLM 투입 절대 불가 (신용정보법 32조). 규정/내규/약관은 OK [2차 미팅 확인]
- 감사로그: 모든 AI 호출 입출력/타임스탬프/사용자 기록 필수 [2차 미팅 확인]
- AI 감사: 분기 1회 (파일럿), 연 1회 (운영) [2차 미팅 확인]
- 의견서: "AI 보조 활용" + 참조 규정 근거 표시 필수 [2차 미팅 확인]
- 모델 정책: 모델별 데이터 처리 정책 문서 제출 필요 [2차 미팅 확인]

## 5. 원하는 것
- **유즈케이스: 자동차 할부 심사 규정 검색 + 심사 의견서 보조 (확정)** [2차 미팅]
- 규정집: 여신규정 + 자동차금융 특약 + 리스 특약 + 심사 가이드라인(200p) [2차 미팅]
- 구조: 편→장→조→항, 같은 조항이 상품별로 다르게 존재 [2차 미팅]
- 개정: 분기 1회 + 수시, 버전 관리 중 [2차 미팅]
- 성공 기준: 규정 검색 50% 단축 + 심사역 "쓸만하다" [2차 미팅]
- 고객 대면: 아님, 내부 도구 [2차 미팅]

## 6. 제약사항
- 예산: 전체 7억 (우리 2~3억) [1차 미팅]
- 일정: 5월~10월, 11월 경영진 보고 [1차 미팅]
- 인력: DX PM + 오토에버 5명 + 우리 2~3명 [1차 미팅]
- 역할: 오토에버=PM+인프라+CI/CD, 우리=AI설계+구현+자문 [확인]
