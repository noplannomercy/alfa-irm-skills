## 미팅 흐름 요약

**오프닝 (10분)**
- 안전관리본부장: "작년에 중대재해처벌법 관련 감사가 강화되면서 안전관리 기록 완결성 요구가 높아졌습니다. 우리 팀 25명이 현장별로 흩어져 있는데, 각자가 제각각 다른 버전의 매뉴얼을 보고 있어요."
- 본부장: "고소작업이랑 높이작업이 같은 건데 현장마다 다르게 불러요. 신입 안전관리자는 용어 통일이 안 되어 있어서 혼선이 많아요."

**현황 파악 (30분)**

### 안전관리 프로세스
- 안전관리자 25명, 현장별 2~3명 배치
- 연 2,400건 점검 (현장별 약 300건/년)
- TBM: 매 작업 전 5~10분, 하루 2~4회
- 정보 탐색: PDF 직접 열거나 동료에게 전화로 문의 → 30% 시간 소모 체감
- "과거 사고 사례는 엑셀로 있는데 어떤 기준으로 검색해야 할지 모르겠어요. 그냥 Ctrl+F로 찾아요."

### 데이터 현황 (IT팀장 설명)
- 안전관리 매뉴얼: 2024년 12월 최종 갱신, 6개월~1년 주기 업데이트
- SOP 30종: 고소작업, 굴착, 크레인, 용접 등 — 가장 자주 쓰는 것은 고소작업/굴착/크레인 3종
- 법규집: 행정처에서 나올 때마다 PDF 추가, 내부 반영 지연 2~3개월
- 사고 사례 180건: 엑셀+PDF, 5년치, 사고유형/원인/조치/재발방지 컬럼 있음
- 스캔 도면: 현장별 150~300장, OCR 미적용, 구형 스캐너 사용한 도면은 텍스트 판독 거의 불가

**핵심 발언 기록**

> "스캔 도면에서 안전 관련 표기를 AI가 찾아줬으면 좋겠는데, 품질이 너무 들쭉날쭉해서 될지 모르겠어요." — IT팀장

> "사고 사례를 쉽게 검색해서 TBM에 활용하고 싶어요. 크레인 작업 전에 관련 사고 사례 3개를 자동으로 뽑아줬으면." — 안전팀장

> "법규가 바뀌면 매뉴얼 업데이트가 늦어서 현장에서 옛날 기준으로 작업해요. 실시간으로 법규 적용이 됐으면 좋겠어요." — 안전팀장

> "기상 악화 시 작업 중지 판단을 시스템이 자동으로 해줬으면. 지금은 담당자 판단인데 기준이 사람마다 달라요." — 본부장

> "중대재해처벌법 때문에 안전관리 기록이 완벽해야 해요. 감사 시 기록 누락이 있으면 바로 과태료예요." — 본부장

### 시스템 현황
- 안전관리 시스템(자체개발): PostgreSQL, 웹 기반, 현재 100% 가동 중
- 현장관리 API: REST, JSON 응답, 현장ID/인력수/장비목록 조회 가능
- 기상청 API: 현재 수동으로 날씨 확인 후 담당자가 판단, 자동화 없음
- AWS EC2+RDS 사용 중, IT팀 3명(백엔드1, 인프라1, 데이터1)

**요구사항 정리 (20분)**

### 우선순위 (본부장 직접 답변)
1. **사고 사례 검색 + TBM 활용** — "이게 가장 급해요"
2. **매뉴얼/법규 Q&A** — "현장에서 바로 물어볼 수 있어야 해요"
3. **기상 연동 작업 중지 판단** — "자동화해주세요"
4. **도면 안전 표기 검색** — "되면 좋겠지만 어려우면 나중에"

### 제약 확인
- 예산 4천만원: 인프라+라이선스 1천만원, 개발 3천만원 예상
- 10월 PoC: 성공 기준 = 안전관리자 만족도 + 정보탐색 시간 20% 단축
- 도면 외부 전송: 검토 필요 (보안팀 확인 예정)
- 외부 AI API: "OpenAI 쓰는 건 괜찮은데, 도면 데이터는 외부로 보내기 어려울 것 같아요"
- IT팀: 이 프로젝트에 약 50% 가용 (다른 업무 병행)
