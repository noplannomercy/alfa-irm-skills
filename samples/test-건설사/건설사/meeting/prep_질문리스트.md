# 사전 질문리스트 — 대림건설 (1차 미팅 준비)

> 작성일: 2026-03-27
> 미팅 목적: AI 도입 요구사항 파악 및 기술 검토 준비
> 대상: 안전관리본부, IT팀

## A. 현황 파악

### A-1. 안전관리 업무 흐름
1. 현재 안전관리자가 매뉴얼/법규를 찾는 구체적인 방식은 어떻게 되나요? (PDF 직접 열기, 사내 포털 검색, 구두 문의 등)
2. 정보 탐색에 30%의 시간이 걸린다고 하셨는데, 하루 기준으로 어떤 시점에 가장 자주 찾아보시나요? (TBM 전, 점검 중, 사고 발생 후)
3. 현장마다 안전 조치가 다르게 적용되는 사례를 구체적으로 설명해주실 수 있나요?
4. TBM(작업전 안전미팅)은 하루 몇 회, 얼마나 걸리나요? 현재 어떤 자료를 활용하시나요?

### A-2. 데이터 현황
5. 안전관리 매뉴얼(PDF 200p)은 마지막으로 갱신된 게 언제인가요? 갱신 주기는 어떻게 되나요?
6. SOP 30종 중 가장 자주 참조하는 상위 5종은 무엇인가요?
7. 과거 사고 사례집 180건 — 이걸 현재 어떻게 활용하고 계신가요? 직접 엑셀을 열어서 찾으시나요?
8. 스캔 도면의 품질 문제가 구체적으로 어느 정도 심각한가요? 흐릿한 도면이 전체의 몇 % 정도 되나요?
9. 도면은 현장별로 수백 장이라고 하셨는데, 한 현장당 평균 몇 장 정도인가요?

### A-3. 시스템 현황
10. 안전관리 시스템(PostgreSQL)에서 가장 자주 조회하는 데이터는 무엇인가요? (점검이력, 사고이력, 교육이력 중)
11. 현장관리 시스템 API — 현재 이걸 직접 쓰는 담당자가 있나요, 아니면 시스템이 자동으로 호출하나요?
12. 기상청 API는 현재 어떻게 활용 중인가요? 작업 중지 판단을 누가, 어떻게 하고 있나요?

## B. 요구사항 확인

### B-1. 핵심 기능
13. AI가 가장 먼저 해결해줬으면 하는 문제를 하나만 고르신다면 무엇인가요?
14. "고소작업=높이작업" 같은 동의어 문제가 다른 용어에도 있나요? 현장마다 다르게 부르는 용어 목록이 있나요?
15. 기상 악화 시 작업 중지 판단의 기준이 현재 문서화되어 있나요? (풍속 몇 m/s, 강수량 몇 mm 등)
16. 중대재해처벌법 대응으로 안전관리 기록에서 가장 취약한 부분이 어디라고 보시나요?

### B-2. 사용 시나리오
17. 안전관리자가 현장에서 AI를 사용할 때 주로 사용하는 디바이스는 무엇인가요? (PC, 태블릿, 스마트폰)
18. 현장사무소 8곳 모두 네트워크 환경이 안정적인가요? 인터넷 연결이 불안정한 현장도 있나요?
19. AI 응답을 안전관리자가 그냥 쓸 건지, 아니면 검수 후 사용할 건지 어떤 프로세스를 원하시나요?

## C. 제약사항 및 우선순위

### C-1. 기술/보안
20. 도면은 "사내 한정" 데이터라고 하셨는데, 도면 데이터를 외부 클라우드에 저장하는 것에 제약이 있나요?
21. AWS 이외 다른 클라우드나 온프레미스 서버를 추가로 사용하는 것에 대해 제약이 있나요?
22. OpenAI 같은 외부 AI API 사용 시 데이터가 외부로 전송되는 것에 대해 보안 정책상 제한이 있나요?

### C-2. 일정/예산
23. 10월 PoC 완료 시 성공 기준을 어떻게 정의하시나요? (예: 정보 탐색 시간 20% 단축, 정확도 80% 이상 등)
24. IT팀 3명의 가용 시간이 얼마나 되나요? 이 프로젝트에 전담인가요, 아니면 다른 업무와 병행인가요?
25. 4천만원 예산 중 인프라/라이선스와 개발 공수 비율을 어떻게 생각하고 계신가요?

### C-3. 조직
26. 안전관리자 25명이 AI 도구를 사용하게 될 때, 교육/온보딩 계획이 있으신가요?
27. 안전관리본부와 IT팀 외에 이해관계자가 더 있나요? (법무팀, 품질팀, 경영진 등)
28. AI 도입에 반대하거나 우려하는 부서나 담당자가 있나요?
