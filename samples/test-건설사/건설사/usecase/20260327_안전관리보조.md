## 2. 핵심 시나리오

### 시나리오 1: TBM 사고 사례 검색
```
안전관리자 입력: "오늘 크레인 작업 전 TBM 자료 필요해요"
AI 응답: 크레인 관련 과거 사고 사례 3건 제시 (사고유형/원인/재발방지 포함)
         관련 SOP 조항 참조
         오늘 기상 데이터 기반 특이 주의사항 포함
```

### 시나리오 2: 법규/매뉴얼 Q&A
```
안전관리자 입력: "고소작업 시 안전대 부착 기준이 어떻게 돼요?"
AI 응답: 산업안전보건법 관련 조항 + 안전관리 매뉴얼 해당 섹션 + 고소작업 SOP 기준
         출처 문서명 및 조항 번호 포함
```

### 시나리오 3: 기상 조건 작업 중지 판단
```
상황: 기상청 API에서 특정 현장 풍속 14m/s 수신
AI 동작: "○○ 현장 풍속 기준 초과 → 크레인 작업 중지 권고" 자동 알림
         관련 법규 조항 및 작업 중지 기준 함께 제시
```

### 시나리오 4: 용어 동의어 처리
```
안전관리자 입력: "높이작업 주의사항 알려줘"
AI 동작: "높이작업 = 고소작업" 동의어 인식
         고소작업 관련 매뉴얼, SOP, 법규, 사고 사례 통합 검색
```

## 6. 제약사항 및 리스크

| 구분 | 내용 | 대응 |
|------|------|------|
| 스캔 도면 품질 | OCR 미적용, 품질 불균일 → 1차 제외 | 2차 PoC에서 OCR 전처리 후 재검토 |
| 도면 외부 전송 제한 | 보안팀 확인 중 | 로컬 임베딩 준비 (BGE-m3-ko 로컬) |
| 법규 실시간 반영 | 자동화 파이프라인 없음 | 갱신 감지 + 재인덱싱 파이프라인 구축 |
| 용어 동의어 | "고소작업=높이작업" 외 미확인 동의어 | GraphRAG 동의어 노드 + 안전팀 워크숍 |
| IT팀 가용성 | 50% (다른 업무 병행) | 일정 버퍼 2~3주 반영 |

## 7. 데이터 흐름

```
안전관리자 질의
    ↓
질의 분류 (법규/사례/기상/절차/이력)
    ↓
[문서 검색] pgvector 하이브리드 검색
  + [그래프 탐색] Apache AGE (동의어/관계 추적)
    ↓
[DB 조회] 안전관리 시스템 (이력 조회)
+ [API 조회] 현장관리 API, 기상청 API
    ↓
컨텍스트 조립 (문서 + 관계 + 실시간 데이터)
    ↓
LLM 응답 생성 (출처 포함)
    ↓
안전관리자에게 전달
```
