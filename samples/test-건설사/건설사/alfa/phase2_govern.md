## 1. 거버넌스 구조

| 역할 | 담당자 | 책임 |
|------|--------|------|
| AI 도입 총괄 | 안전관리본부장 | 도입 방향, 예산, 최종 승인 |
| AI 운영 담당 | 안전팀장 | 사용 기준, 품질 모니터링, 이슈 보고 |

## 2. AI 사용 정책

### 2-1. 허용 사용

- 안전 매뉴얼, 법규, SOP 내용 질의응답
- 작업 유형별 과거 사고 사례 검색
- 기상 데이터 기반 작업 중지 권고
- 안전 점검 체크리스트 참조

### 2-2. 제한 사용 (검수 필수)

- AI 답변을 공식 안전 지침으로 활용 시 → 안전관리자 검토 필수
- 감사 자료에 AI 생성 내용 포함 시 → 안전팀장 승인 필수
- 신규 법규 적용 시 → 법규 반영 확인 후 사용

### 2-3. 금지 사용

- AI 답변을 근로자에게 최종 안전 지침으로 직접 전달 (반드시 안전관리자 판단 포함)
- AI를 통한 사고 처리 공식 보고서 생성
- 개인정보(근로자 이름, 연락처)가 포함된 질의

## 3. 규제 요건

### 3-1. 산업안전보건법

| 조항 | 내용 | AI 시스템 적용 |
|------|------|-------------|
| 제36조 | 위험성 평가 실시 의무 | AI는 보조 도구, 최종 판단은 안전관리자 |
| 제17조 | 안전관리자 선임 의무 | AI는 대체 불가, 보조 역할 |
| 제29조 | 근로자 교육 의무 | AI 답변을 교육 자료로 활용 가능 (검수 후) |
| 제155조 | 서류 보존 의무 | AI 조회 이력 포함 보존 필요 |

### 3-2. 중대재해처벌법

| 항목 | 요건 | AI 시스템 대응 |
|------|------|-------------|
| 안전보건관리체계 구축 | 체계적 안전관리 증명 | AI 사용 이력, 답변 출처 기록 |
| 위험요인 파악 의무 | 위험 정보 수집/검토 증명 | 사고 사례 검색 이력 보존 |
| 안전조치 이행 기록 | 조치 사항 문서화 | AI 권고사항 수용/거부 이력 기록 |

### 3-3. 개인정보보호법

| 항목 | 내용 |
|------|------|
| 대상 데이터 | 근로자 이름, 연락처, 교육이력 |
| 분류 | 일반 개인정보 (민감정보 아님) |
| 처리 원칙 | 최소 수집, 목적 외 사용 금지 |
| AI 입력 제한 | 개인식별 정보 마스킹 후 사용 |

## 4. 책임 한계 명확화

### 4-1. AI 시스템의 역할 한계

```
AI 시스템 역할:
  ✅ 관련 법규/매뉴얼 검색 및 제시
  ✅ 유사 사고 사례 검색 및 제시
  ✅ 기상 데이터 기반 권고
  ✅ 출처 문서 참조 제공

안전관리자 역할 (AI 대체 불가):
  ✅ 현장 상황 직접 판단
  ✅ 안전 조치 최종 결정
  ✅ 법적 책임 수반 서류 서명
  ✅ 근로자 직접 교육 및 지도
```

### 4-2. 오답/오류 발생 시 처리

| 상황 | 처리 절차 |
|------|---------|
| AI 오답 발견 | IT팀 즉시 보고 → 해당 답변 로그 보존 → 원인 분석 → 데이터 수정 |
| 법규 미반영 오답 | 즉시 사용 중단 통보 → 법규 데이터 갱신 → 재검증 후 재개 |
| 시스템 장애 | IT팀 즉시 대응 → 기존 수동 프로세스로 복귀 → 사후 보고 |

## 6. 개인정보 처리

| 항목 | 내용 |
|------|------|
| 수집 목적 | 안전관리 보조, 교육이력 확인 |
| 처리 방식 | 마스킹 처리 후 AI 입력 (이름 → ○○○, 연락처 → 마스킹) |
| 보관 위치 | AWS RDS (국내 리전, 서울 ap-northeast-2) |
| 외부 전송 | 원칙 금지 (외부 AI API 사용 시 개인정보 포함 금지) |
| 접근 권한 | 안전관리자(조회), IT팀(관리), 본부장(전체) |

## 7. 기술 보안 정책

| 항목 | 기준 |
|------|------|
| 접근 인증 | 사내 계정 SSO 연동 |
| 네트워크 | VPN 통한 현장 접속 |
| 데이터 암호화 | RDS 저장 암호화 (AES-256), 전송 TLS 1.3 |
| 도면 데이터 | 사내 서버 한정, 외부 API 전송 금지 (임베딩 로컬 처리) |
| 감사 로그 | 모든 API 호출 이력 보존 (CloudWatch) |
| 접근 권한 관리 | 최소 권한 원칙, 역할 기반 접근 제어 |
