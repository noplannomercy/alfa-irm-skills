> 작성일: 2026-03-27
> 신뢰도 전파: 리소스 인벤토리 → 컬렉션 설계 → 질의 매핑

## 분류 원칙

리소스를 검색 목적에 따라 컬렉션으로 분류한다. 컬렉션 분리 3기준 (갱신주기 / 메타데이터 / 질의패턴) 으로 판단하며, 2/3 이상 해당 시 분리한다.

---

## 1. 컬렉션 분리 판단

### 판단 대상: 문서형 리소스 (R-01~R-05)

| 리소스 | 갱신주기 | 메타데이터 | 질의패턴 | 분리 판단 |
|--------|---------|---------|---------|---------|
| R-01 매뉴얼 | 6개월~1년 | 공종, 챕터 | "기준/절차 알려줘" | - |
| R-02 SOP | 부정기 | 작업유형, 단계 | "작업 전 점검 항목" | - |
| R-03 법규집 | 법 개정 시 (연 2~3회) | 법령명, 조항번호 | "법적 기준이 뭐야?" | **분리** |
| R-04 사고 사례 | 사고 발생 시 | 사고유형, 날짜, 공종, 심각도 | "유사 사고 사례 찾아줘" | **분리** |
| R-05 체크리스트 | 부정기 | 공종, 점검항목 | "점검 항목 알려줘" | - |

### 3기준 상세 판단

#### R-03 (법규집) 분리 근거

| 기준 | 판단 | 근거 |
|------|------|------|
| 갱신주기 | **다름** | 법 개정 시 즉시 갱신 필요 (현재 지연 2~3개월), 매뉴얼/SOP는 6개월~1년 |
| 메타데이터 | **다름** | 법령명, 조항번호, 시행일 등 법규 고유 메타데이터 필요 (매뉴얼에는 불필요) |
| 질의패턴 | **다름** | "법적 기준", "법령에서 규정하는" 등 법규 특정 질의 → 별도 검색 범위로 정밀도 향상 |
| **판단** | **✅ 분리** | **3/3 해당, 분리 타당** |

#### R-04 (사고 사례) 분리 근거

| 기준 | 판단 | 근거 |
|------|------|------|
| 갱신주기 | **다름** | 사고 발생 시마다 추가 (이력형), 매뉴얼/SOP는 정적 문서 (최신형) |
| 메타데이터 | **다름** | 사고유형, 심각도, 원인분류, 공종, 날짜 등 이력형 고유 메타데이터 필요 |
| 질의패턴 | **다름** | "유사 사고 사례 3건", "크레인 관련 사고" → 이력 검색 패턴 (매뉴얼과 혼재 시 정밀도 하락) |
| **판단** | **✅ 분리** | **3/3 해당, 분리 타당** |

#### R-01+R-02+R-05 (매뉴얼+SOP+체크리스트) 통합 근거

| 기준 | 판단 | 근거 |
|------|------|------|
| 갱신주기 | **유사** | 모두 부정기, 6개월~1년 주기 |
| 메타데이터 | **유사** | 공종, 작업유형 — 통일 가능 |
| 질의패턴 | **유사** | "기준 알려줘", "절차가 어떻게 돼?" — 함께 검색 시 더 풍부한 답변 |
| **판단** | **통합** | **0/3 분리 기준 해당, 통합이 유리** |

```
├── R-08 현장관리 API (REST 호출)
└── R-09 기상청 API (REST 호출)
```

## 3. 컬렉션별 메타데이터 스키마

### 공통 메타데이터 (전 컬렉션)

```json
{
  "doc_type": "string",      // "manual" / "sop" / "checklist" / "regulation" / "accident"
  "work_type": "string",     // "고소작업" / "굴착" / "크레인" / "용접" / "전기" / "all"
  "date": "date",            // 문서 작성/발행일
  "source_file": "string"    // 원본 파일명 (출처 표시용)
}
```

### safety_knowledge 도메인 메타데이터

```json
{
  "specialty": "string",     // "토목" / "건축" / "설비" / "전기" / "공통"
  "doc_category": "string",  // "manual" / "sop" / "checklist"
  "chapter": "string",       // 챕터명 (자동 추출)
  "version": "string"        // 문서 버전 (가능한 경우)
}
```

### regulation 도메인 메타데이터

```json
{
  "law_name": "string",      // "산업안전보건법" / "시행령" / "시행규칙" / "고시"
  "article_no": "string",    // "제44조" 등 (규칙 기반 추출)
  "effective_date": "date",  // 시행일
  "regulation_type": "string" // "osh_act" (산업안전보건법) / "serious_accident_act" (중대재해처벌법)
}
```

### accident_cases 도메인 메타데이터

```json
{
  "accident_type": "string",  // "추락" / "협착" / "감전" / "화재" / "기타"
  "severity": "string",       // "사망" / "중상" / "경상"
  "root_cause": "string",     // LLM 추출: "안전수칙 미준수" / "장비 결함" / "기상 악화" 등
  "site_id": "string",        // 발생 현장 ID (마스킹 가능)
  "year": "integer"           // 발생 연도 (시간 범위 필터용)
}
```

## 4. 신뢰도 전파 (Confidence Propagation)

### 신뢰도 정의

| 등급 | 의미 | 적용 |
|------|------|------|
| High (H) | 데이터 품질 양호, 즉시 사용 가능 | 검색 결과 신뢰, 출처 표시 |
| Medium (M) | 일부 전처리 필요, 사용 가능 | 검색 결과 사용, 품질 주의 표시 |
| Low (L) | 전처리 필요, 현재 제외 | 인덱싱 제외, 재검토 필요 |

### 리소스 → 컬렉션 신뢰도 전파

| 리소스 | 원본 신뢰도 | 컬렉션 | 전파 신뢰도 | 근거 |
|--------|-----------|--------|-----------|------|
| R-01 매뉴얼 | H | safety_knowledge | **H** | PDF 품질 양호, 구조 명확 |
| R-02 SOP | H | safety_knowledge | **H** | PDF 품질 양호, 작업유형 명확 |
| R-05 체크리스트 | H | safety_knowledge | **H** | Excel 구조화, 항목 명확 |
| R-03 법규집 | H (단, 갱신 지연 이슈) | regulation | **M** | 갱신 파이프라인 미구축 → 현재 2~3개월 지연 위험 |
| R-04 사고 사례 | H (단, 전처리 필요) | accident_cases | **M** | Excel+PDF 혼합, 파싱 전처리 필요, 개인정보 마스킹 필요 |
| R-06 도면 | L | (제외) | **L** | OCR 미적용, 품질 불균일 |

### 컬렉션 신뢰도 → 질의 매핑 전파

| 컬렉션 | 신뢰도 | 질의 매핑 영향 |
|--------|--------|-------------|
| safety_knowledge | H | 법규/매뉴얼 Q&A 답변: 높은 신뢰, 출처 표시 |
| regulation | M | 법규 Q&A 답변: "최신 법규 반영 여부 확인 권고" 문구 추가 필요 |
| accident_cases | M | 사고 사례 검색: "개인정보 마스킹 처리됨" 표시, 전처리 완료 후 H로 승급 |

## 5. 인덱싱 범위 결정

### 컬렉션별 유형 분류

| 컬렉션 | 유형 | 인덱싱 전략 |
|--------|------|-----------|
| safety_knowledge | **최신형** | 구버전 삭제 후 신버전 삽입 (구버전 히트 시 잘못된 기준 안내 위험) |
| regulation | **최신형** | 법 개정 시 구버전 즉시 삭제 + 신버전 인덱싱 (법 위반 방지) |
| accident_cases | **이력형** | 전체 5년치 보존, 신규 사고 발생 시 추가 |

### regulation 갱신 파이프라인 (즉시 긴급도)

```
고용노동부 법령정보 웹사이트 변경 감지
    ↓
신규/개정 PDF 다운로드
    ↓
기존 구버전 청크 삭제 (doc_id 기준)
    ↓
신버전 파싱 + 청킹 + 임베딩
    ↓
regulation 컬렉션 삽입
    ↓
알림 발송 (안전팀장)
```
