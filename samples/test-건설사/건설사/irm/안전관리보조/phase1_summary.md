## 1. 설계 결정 요약

### 1-1. 핵심 아키텍처 결정

| 항목 | 결정 | 근거 |
|------|------|------|
| 벡터DB | pgvector (PostgreSQL 확장) | 기존 AWS RDS 활용, 관리 단순 |
| 그래프DB | Apache AGE (PostgreSQL 확장) | 동일 DB, 동의어/관계 처리 |

| 리랭킹 | BGE-reranker-v2-m3 (로컬) | 한국어, 보안, 비용 |
| 검색 전략 | 하이브리드 (Dense+BM25) + GraphRAG | 키워드+의미+관계 통합 |
| 컬렉션 | 3개 분리 | 갱신주기/메타/질의패턴 각각 판단 |

## 2. 일정 추정

### 2-1. 일정 기준표 (컬렉션 규모별)

| 컬렉션 규모 | 인덱싱 소요 | 기준 |
|-----------|-----------|------|
| 소규모 (1만 청크 이하) | 1~2주 | 파싱+임베딩+인덱싱 |
| 중규모 (1~10만 청크) | 2~4주 | 대용량 배치 처리 |
| 대규모 (10만 청크 이상) | 4~8주 | GPU 병렬 처리 필요 |

**본 프로젝트**: 총 ~15,000청크 → **소규모** → 인덱싱 1~2주

### 2-2. GraphRAG 복잡도별 추가 리스크

| GraphRAG 복잡도 | 리스크 가산 |
|---------------|-----------|
| 단순 (동의어만) | +0주 |
| 중간 (계층 관계 + 동의어) | +1~2주 |
| 복잡 (멀티도메인 + 동적 관계) | +3~4주 |

**본 프로젝트**: 동의어 + 계층 관계 (법규-매뉴얼-SOP) → **중간** → +1주 가산

### 2-3. 리스크 가산 요인

| 리스크 | 일정 가산 |
|--------|---------|
| IT팀 50% 가용 | +2주 |
| 스캔 도면 품질 이슈 (1차 제외로 영향 최소) | +0주 |
| 법규 갱신 파이프라인 신규 구축 | +1주 |
| 안전팀 데이터 확인/검수 대기 | +1주 |
| 전처리 파이프라인 (사고 사례 Excel+PDF) | +1주 |

**총 리스크 가산: +5주**
