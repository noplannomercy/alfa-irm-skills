# IRM Phase 0 — 질의 매핑 (Query Mapping) — 안전관리 보조

> 고객: 대림건설
> 유즈케이스: 안전관리 보조
> 작성일: 2026-03-27
> 신뢰도 전파: 컬렉션 분류 결과 → 질의 매핑 → Phase 1 탐색 설계

---

## 질의 매핑 원칙

- 각 질의 유형에 대해 검색 대상 컬렉션과 실시간 조회 리소스를 결정
- 컬렉션 신뢰도를 답변 생성 시 반영 (regulation M → "최신 법규 확인 권고" 문구)
- GraphRAG 적용 여부를 질의 유형별로 판단

---

## 1. 질의 유형별 매핑

### Q-01: 법규/기준 Q&A

| 항목 | 내용 |
|------|------|
| 예시 질의 | "고소작업 시 안전대 부착 기준이 뭐예요?" |
| 예시 질의 | "굴착 작업 시 토사 붕괴 방지 기준은?" |
| 예시 질의 | "크레인 최대 풍속 기준이 어떻게 돼요?" |
| 검색 컬렉션 | `regulation` (1순위) + `safety_knowledge` (2순위) |
| 실시간 조회 | 없음 |
| GraphRAG | ✅ 적용 — 법규→매뉴얼→SOP 관계 탐색 (법 조항이 매뉴얼의 어느 섹션으로 구체화되는지) |
| 신뢰도 전파 | regulation: M → 답변에 "최신 법규 반영 여부 확인 권고" 문구 포함 |
| 메타 필터 | `work_type`, `law_name`, `article_no` |

### Q-02: TBM 사고 사례 검색

| 항목 | 내용 |
|------|------|
| 예시 질의 | "오늘 크레인 작업 전 TBM 자료 필요해요" |
| 예시 질의 | "고소작업 관련 사고 사례 3건 주세요" |
| 예시 질의 | "용접 작업 시 화재 사고 사례 알려줘" |
| 검색 컬렉션 | `accident_cases` (1순위) + `safety_knowledge` SOP (참조) |
| 실시간 조회 | R-08 현장관리 API (오늘 현장 작업 유형 확인) |
| GraphRAG | ✅ 적용 — 사고원인→위험요인→재발방지 관계 탐색 |
| 신뢰도 전파 | accident_cases: M → 전처리 완료 후 H 승급, 개인정보 마스킹 확인 |
| 메타 필터 | `work_type`, `accident_type`, `severity`, `year` (최근 5년) |
| 출력 형식 | 사고유형 / 발생원인 / 조치 / 재발방지 카드 형태 |

### Q-03: 기상 조건 작업 중지 판단

| 항목 | 내용 |
|------|------|
| 예시 질의 | "오늘 ○○ 현장 크레인 작업 가능한가요?" |
| 예시 질의 | "현재 풍속이 얼마인지, 작업 중지 해야 하나요?" |
| 검색 컬렉션 | `regulation` (작업 중지 기준 법규) + `safety_knowledge` (SOP) |
| 실시간 조회 | R-09 기상청 API (현장 좌표 기반 풍속/강수량/기온) + R-08 현장관리 API (현장 정보) |
| GraphRAG | ⚠️ 선택 — 기상조건→법규기준→작업중지 관계 (단순 규칙이므로 GraphRAG 효과 제한적) |
| 판단 로직 | 풍속 ≥10m/s → 크레인 중지 / 강수량 ≥1mm/h → 전기작업 중지 / 기상청 위험 기상 경보 → 전면 중지 |
| 출력 형식 | 현장명 / 현재 기상 / 판단 결과 / 관련 법규 조항 |

### Q-04: 절차/체크리스트 확인

| 항목 | 내용 |
|------|------|
| 예시 질의 | "굴착 작업 전 점검 절차가 어떻게 돼요?" |
| 예시 질의 | "안전 점검 체크리스트 전기 공종 버전 주세요" |
| 검색 컬렉션 | `safety_knowledge` (SOP + 체크리스트) |
| 실시간 조회 | 없음 |
| GraphRAG | ✅ 적용 — 절차→법규기준→점검항목 관계 탐색 |
| 메타 필터 | `work_type`, `specialty`, `doc_category` |

### Q-05: 현장 이력 조회

| 항목 | 내용 |
|------|------|
| 예시 질의 | "이 현장 최근 3개월 점검 결과 알려줘" |
| 예시 질의 | "작년에 이 현장에서 사고가 있었나요?" |
| 검색 컬렉션 | 없음 (벡터 검색 불필요) |
| 실시간 조회 | R-07 안전관리 시스템 DB (inspection_history, accident_history) + R-08 현장관리 API |
| GraphRAG | ❌ 미적용 — 구조화 DB 조회로 충분 |
| 쿼리 방식 | SQL 파라미터 쿼리 (site_id, date_range 기반) |

### Q-06: 교차 검색 (복합 질의)

| 항목 | 내용 |
|------|------|
| 예시 질의 | "크레인 작업 관련 법규, 매뉴얼, 사고 사례 모두 주세요" |
| 예시 질의 | "고소작업(=높이작업) 관련 정보 총정리" |
| 검색 컬렉션 | `regulation` + `safety_knowledge` + `accident_cases` (교차 컬렉션 검색) |
| 실시간 조회 | 없음 |
| GraphRAG | ✅ 적용 — 동의어 처리 + 법규-매뉴얼-SOP-사고사례 전체 관계 탐색 |
| 리랭킹 | 교차 컬렉션 통합 후 1회 리랭킹 |
| 신뢰도 전파 | regulation 포함 시 → "최신 법규 반영 여부 확인 권고" |

## 2. GraphRAG 핵심 관계 정의

### 동의어 관계 (안전관리 도메인)

```
고소작업 --[동의어]--> 높이작업
고소작업 --[동의어]--> 고공작업
굴착작업 --[동의어]--> 터파기작업
협착 --[동의어]--> 끼임
```

### 법규-문서 관계

```
산업안전보건법 제44조
    --[구체화]--> 안전관리 매뉴얼 3장 (고소작업 기준)
    --[구체화]--> 고소작업 SOP

산업안전보건법 제38조
    --[구체화]--> 크레인 작업 SOP
    --[규정]--> 크레인 풍속 기준 (10m/s)

중대재해처벌법 제4조
    --[요구]--> 위험성 평가 (체크리스트)
    --[요구]--> 안전관리 기록 (안전관리 시스템)
```

### 사고 인과 관계

```
고소작업 안전수칙 미준수
    --[원인]--> 추락 사고
    --[재발방지]--> 안전대 부착 의무 강화

크레인 강풍 작업 강행
    --[원인]--> 크레인 전도 사고
    --[재발방지]--> 풍속 기준 초과 시 즉시 중지
```

## 3. 질의 분류 로직 (라우팅)

```python
질의 분류 프롬프트:
  - "법규", "법적", "조항", "법령" → Q-01 (regulation 우선)
  - "사고 사례", "TBM", "유사 사고" → Q-02 (accident_cases)
  - "기상", "풍속", "비", "작업 가능", "중지" → Q-03 (기상 판단)
  - "절차", "체크리스트", "점검 항목" → Q-04 (safety_knowledge)
  - "이력", "점검 결과", "현장 사고" + 현장명 → Q-05 (DB 조회)
  - 복합 키워드 → Q-06 (교차 검색)
  - "고소작업", "높이작업" → 동의어 처리 후 해당 유형으로
```

## 4. 신뢰도 전파 요약 (Classification → Query Mapping)

| 컬렉션 | 신뢰도 | 답변 생성 시 적용 사항 |
|--------|--------|-------------------|
| safety_knowledge | H | 출처 파일명+챕터 표시, 추가 주의 문구 없음 |
| regulation | M | "법규 갱신 파이프라인 구축 전까지 최신 법규 직접 확인 권고" 문구 포함 |
| accident_cases | M | "개인정보 마스킹 처리됨" 표시, 전처리 후 H 승급 예정 |
