## 1. 체이닝 개요

단일 질의로 해결되지 않는 복합 업무 흐름을 다단계 에이전트 체인으로 처리한다.

### 체이닝이 필요한 시나리오

| 시나리오 | 이유 |
|---------|------|
| TBM 자료 자동 생성 | 현장정보 조회 → 오늘 작업유형 확인 → 기상 판단 → 사고 사례 검색 → 종합 보고서 |
| 기상 작업 중지 전체 현장 점검 | 전체 현장 목록 조회 → 현장별 기상 조회 → 중지 필요 현장 필터링 → 알림 발송 |
| 법규 갱신 반영 확인 | 법규 변경 감지 → 관련 매뉴얼 조항 검색 → 불일치 항목 식별 → 담당자 알림 |

## 2. Chain 1: TBM 자동 자료 생성

### 트리거
- 안전관리자가 "오늘 TBM 자료 만들어줘 + 현장명" 입력
- 또는 매일 아침 7시 자동 실행 (Cron 옵션)

### 체인 흐름

```
Step 1: 현장 정보 조회
  cli-anything-field-api --json sites get --site-id {SITE_ID}
  → 오늘 예정 작업 목록 추출

Step 2: 기상 판단
  cli-anything-weather --json check-work-stop --site-id {SITE_ID}
  → 작업 중지 대상 작업유형 식별

Step 3: 사고 사례 검색 (작업유형별)
  for each work_type in today_works:
    cli-anything-safety-search --json search \
      --query "{work_type} 사고 사례" \
      --collection accident_cases --top-k 3 \
      --work-type "{work_type}"
  → 작업유형별 사고 사례 3건씩

Step 4: 법규/매뉴얼 핵심 사항 검색
  cli-anything-safety-search --json search \
    --query "{work_type} 작업 전 주의사항" \
    --collection safety_knowledge --top-k 3

Step 5: TBM 자료 생성 (GPT-4o)
  입력: 현장정보 + 기상판단 + 사고사례 + 법규핵심
  출력: TBM 자료 (A4 1장 형식)
    - 오늘 작업 개요
    - 기상 특이사항 (작업 중지 권고 포함)
    - 유사 사고 사례 3건 (원인/재발방지)
    - 오늘 특별 주의사항
    - 출처 목록

Step 6: 로깅
  TBM 자료 생성 이력 저장 (중대재해처벌법 기록 요건)
```

### 출력 형식 (TBM 자료)

```
=== ○○현장 TBM 자료 (2026-03-27) ===

[오늘 작업]
- 크레인 작업 (9:00~17:00)
- 굴착 작업 B구역 (13:00~17:00)

[기상 주의]
⚠️ 오후 풍속 12m/s 예보 → 크레인 작업 오후 중지 예정 (법적 기준: 10m/s)
- 관련 법규: 산업안전보건법 제38조

[유사 사고 사례]
1. 크레인 전도 (2023.08, 건축현장) — 강풍 경보 중 작업 강행 → 재발방지: 풍속 10m/s 시 즉시 중지
2. 크레인 줄걸이 사고 (2022.11) — 불량 와이어 미확인 → 재발방지: 작업 전 와이어 전수 점검
3. 굴착 붕괴 (2024.02) — 흙막이 미설치 → 재발방지: 1.5m 이상 굴착 시 흙막이 필수

[오늘 특별 주의]
- 크레인 와이어 로프 점검 필수
- 오후 풍속 모니터링 상시 실시
- 굴착 작업 시 흙막이 설치 상태 확인

[출처]
- 안전관리매뉴얼_건축편.pdf 4장 / 크레인작업_SOP.pdf / 기상청 단기예보 14:00
```

## 3. Chain 2: 전체 현장 기상 일일 점검

### 트리거
- 매일 오전 6시 자동 실행 (Cron)
- 기상 특보 발령 시 즉시 실행 (이벤트 트리거)

### 체인 흐름

```
Step 1: 전체 현장 목록 조회
  cli-anything-field-api --json sites list
  → 8개 현장 ID 목록

Step 2: 현장별 기상 판단 (병렬)
  for each site_id in sites:
    cli-anything-weather --json check-work-stop --site-id {site_id}
  → 작업 중지 필요 현장 식별

Step 3: 요약 보고서 생성
  → 전체 현장 기상 요약 + 중지 필요 현장 알림

Step 4: 알림 발송 (IT팀 이메일/슬랙 연동)
  작업 중지 대상 현장 안전팀장에게 알림
```

## 4. Chain 3: 법규 갱신 반영 확인

### 트리거
- 법규집 갱신 감지 시 (이벤트 트리거)
- 월 1회 정기 점검 (Cron)

### 체인 흐름

```
Step 1: 신규 법규 버전 파싱
  신규 PDF → Docling 파싱 → 변경 조항 추출

Step 2: 관련 매뉴얼/SOP 검색
  변경 조항 → safety_knowledge 검색
  → 관련 매뉴얼/SOP 섹션 식별

Step 3: 불일치 분석 (GPT-4o)
  법규 변경 내용 vs 현재 매뉴얼 내용 비교
  → 업데이트 필요 항목 목록 생성

Step 4: 담당자 보고
  안전팀장에게 "법규 변경 → 매뉴얼 업데이트 필요 항목" 리포트 전송

Step 5: regulation 컬렉션 갱신
  구버전 청크 삭제 → 신버전 인덱싱
```

## 5. 체이닝 오케스트레이션

### 오케스트레이터 선택: Python 직접 구현 (PoC) → n8n (운영)

| 단계 | 도구 | 이유 |
|------|------|------|
| PoC | Python 스크립트 | 단순, IT팀 역량 적합 |
| 운영 | n8n (오픈소스) | 시각적 워크플로우, 스케줄/이벤트 트리거 |

### 에러 처리

| 체인 단계 | 실패 시 |
|---------|---------|
| 현장 API 실패 | 해당 체인 중단, 담당자 알림, 수동 처리 유도 |
| 기상 API 실패 | 마지막 성공 데이터 사용 + 타임스탬프 경고 표시 |
| 사고 사례 검색 0건 | "관련 사고 사례 없음" 명시 후 다음 단계 진행 |
| LLM 오류 | 3회 재시도 → 실패 시 원본 데이터 그대로 전달 |
