# 자동차보험 심사 보조 — 체이닝 설계

> 작성일: 2026-04-03
> 설계 신뢰도: ✅

## 체이닝 패턴

| # | 질의 | 체이닝 순서 | GraphRAG | 신뢰도 |
|---|------|-----------|:--------:|:------:|
| 1 | "이 건 심사해줘" | 접근→탐색→탐색→실행(유연)→실행(확정) | Y | ✅ |
| 2 | "이 특약 적용 가능?" | 탐색→접근→실행(유연) | Y | ✅ |
| 3 | "거절 건 재심사" | 접근→탐색→실행(확정) | N | ✅ |
| 4 | "유사 사례 + 판단 근거" | 탐색→탐색 | Y | ✅ |

## 체이닝 플로우 상세

### 패턴 1: 전체 심사 보조 (가장 복잡)
```
"이 건 심사해줘" →
  ① 접근: core contract → 계약 정보 [✅]
  ② 접근: uw history → 과거 심사 이력 [✅]
  ③ 탐색: uw_guidelines에서 약관+기준 (GraphRAG 관계 추적) [✅]
  ④ 탐색: uw_cases에서 유사 사례 (GraphRAG 질의 확장) [✅]
  ⑤ 실행(유연): LLM 의견 초안 + 판단 근거 → 심사역 확인 [✅]
  ⑥ 실행(확정): 감사로그 기록 [✅]
```

### 패턴 2: 특약/할증 판단
```
"이 특약 적용 가능?" →
  ① 탐색: uw_guidelines에서 약관 조항 (GraphRAG: 특약→면책→할증 관계) [✅]
  ② 접근: core contract → 계약 정보 [✅]
  ③ 실행(유연): LLM 적용 판단 + 근거 → 심사역 확인 [✅]
```

### 패턴 3: 재심사
```
"거절 건 재심사" →
  ① 접근: uw case → 원심사 건 [✅]
  ② 탐색: uw_cases + uw_guidelines → 유사 재심사 사례 + 기준 [✅]
  ③ 실행(확정): 재심사 접수 가능 판단 + 절차 안내 [✅]
```
