# ALFA 미팅 준비 어시스턴트

> 고객 프로필 빈칸을 찾아내고, 미팅에서 물어볼 질문을 업종 맥락에 맞게 생성한다.

## 폴더 구조

이 스킬은 아래 구조로 파일을 생성한다. 저장 위치는 Step 1에서 확인한다.

```
[저장위치]/[회사명]/
├── STATUS.md                    ← 현재 상태 한눈에
├── meeting/
│   ├── prep_질문리스트.md       ← 미팅용 질문 (내부 참고)
│   └── prep_사전요청서.md       ← 고객 발송용 숙제 (프로필 질의 + 리소스 조사)
└── alfa/
    └── profile.md               ← 프로필 초안 (이후 LIVE에서 업데이트)
```

## 실행 흐름

### Step 1: 고객 정보 수집

AskUserQuestion으로 아래를 물어본다:

```
고객 미팅 준비를 시작합니다.

1. 저장 위치를 알려주세요 (기본: 현재 작업 디렉토리)
2. 고객 정보를 자유롭게 입력해주세요
   (회사명, 업종, 규모, 현재 상황, 원하는 것 등 — 아는 만큼만)
```

회사명을 기반으로 `[저장위치]/[회사명]/` 폴더를 생성한다.
이미 해당 폴더가 존재하면 기존 프로필(`alfa/profile.md`)을 로드하여 이어간다.

### Step 2: 프로필 6항목 매핑

입력받은 정보를 아래 6항목에 매핑하고, 각 항목의 **채움 수준**을 판단한다.

| # | 항목 | 핵심 질문 | 채움 수준 |
|---|------|----------|----------|
| 1 | **회사 개요** | 업종, 규모, 주력 사업 | ⬜ 없음 / 🟡 부분 / 🟢 충분 |
| 2 | **AI/LLM 현황** | 기존 PoC 경험, 결과, 남은 자산 | ⬜ / 🟡 / 🟢 |
| 3 | **인프라** | 클라우드/온프레미스, DB, 네트워크 | ⬜ / 🟡 / 🟢 |
| 4 | **규제 환경** | 감독기관, 컴플라이언스팀, 데이터 분류 | ⬜ / 🟡 / 🟢 |
| 5 | **원하는 것** | 경영진 지시, 현업 니즈, 구체 유즈케이스 | ⬜ / 🟡 / 🟢 |
| 6 | **제약사항** | 예산, 일정, 투입 인력, 고객 대면 범위 | ⬜ / 🟡 / 🟢 |

### Step 3: 업종 분기 판단

입력된 업종에 따라 **추가 확인 포인트**를 결정한다.

```
금융 → 규제 Heavy
  - 금감원/금융위 대상 여부
  - 컴플라이언스팀 존재 여부 → 2차 미팅 동석 필요
  - 개인정보/신용정보 LLM 투입 여부
  - "보조수단성" 충족 가능 여부

제조 → 온프레미스 분기
  - 데이터 외부 반출 가능 여부
  - 온프레미스 GPU 인프라 (A100 등)
  - 용어 표현 차이 여부 → GraphRAG 신호

IT서비스 → 경량 거버넌스
  - 고객사 데이터 혼재 여부 (NDA)
  - 내부용 한정인지 확인
  - CTO 구두 승인이면 바로 진행 가능

이커머스/리테일 → 2단계 접근
  - 1차: 내부 직원 보조 (규제 light)
  - 2차: 고객 대면 (규제 격상) → 별도 거버넌스
  - 오답 책임 소재

의료/공공 → 규제 Very Heavy (금융과 유사하되 더 무거움)
  - 개인건강정보/민감정보 처리
  - 인허가/인증 요건
  - 데이터 주권 이슈

기타/불명 → 범용 질문으로 시작
```

### Step 4: 질문 리스트 생성

**파일:** `[회사명]/meeting/prep_질문리스트.md`

```markdown
# [회사명] 미팅 질문 리스트

## 미팅 기본 정보
- 예상 소요: 60분
- 거버넌스 무게: [Light / Medium / Heavy] (현재 판단)
- 2차 미팅 필요 예상: [예/아니오/판단 불가]

## 반드시 물어볼 것 (프로필 빈칸)

### [빈칸 항목 1] — [항목명]
- Q1. ...
- Q2. ...
- (왜 물어보는지): ...

### [빈칸 항목 2] — [항목명]
- Q1. ...
- ...

## 업종 특화 질문

### [업종]에서 꼭 확인할 것
- Q. ...
- Q. ...
- (놓치면 안 되는 이유): ...

## 초반에 반드시 확인 (거버넌스 무게 결정)
- Q. "이 서비스를 고객(외부)이 직접 사용하나요, 내부 직원이 사용하나요?"
  → 고객 대면 = 거버넌스 Heavy, 내부 = Light
- Q. "규제 기관 검토를 받아야 하나요?"
  → 금융/의료 = Heavy

## 미팅 중 감지할 신호
- "표현이 달라서..." / "같은 건데 다르게 불러요" → GraphRAG 필요 신호
- "데이터 외부 반출 금지" → 온프레미스/로컬 LLM 분기
- "고객 챗봇도 하고 싶고..." → 2단계 접근 필요
- "위에서 아직 말 안 해줬어" → 스폰서 미확정, 진행 보류 가능성

## 현재까지 파악된 프로필 요약
(6항목 채움 현황 표)
```

### Step 6: 사전 요청서 생성

**파일:** `[회사명]/meeting/prep_사전요청서.md`

프로필 빈칸 기반 질의 + 리소스 조사 요청을 하나의 문서로 생성한다. 고객에게 미팅 전 발송하여 사전 준비를 요청하는 용도.

```markdown
# [회사명] 사전 요청서

> 작성일: [날짜]
> 미팅 예정일: [미정]
> 회신 요청: 미팅 전까지

안녕하세요, 미팅 전 아래 내용을 준비해주시면 미팅 시간을 효율적으로 쓸 수 있습니다.

---

## A. 사전 질의 (담당자용)

아래 항목 중 현재 파악 가능한 내용을 정리해주세요.

### 현황 관련
(프로필 빈칸 기반으로 생성 — 빈칸인 항목만)
- [빈칸 항목에 대한 질의 — 고객 톤으로]
- ...

### 목표/기대 관련
- AI 도입으로 기대하시는 구체적인 효과가 있으시면 알려주세요
- 예산/일정 계획이 잡혀있으시면 공유 부탁드립니다

---

## B. 리소스 조사 요청 (IT팀용)

AI 도입 범위를 구체화하기 위해 아래 정보가 필요합니다.
미팅 전 또는 미팅 후 1주 내 회신 부탁드립니다.

### 문서/파일
- [ ] AI 활용 대상 문서 목록 (매뉴얼, FAQ, 가이드 등)
- [ ] 각 문서의 저장 위치, 파일 형식, 대략적 건수
- [ ] 샘플 문서 3~5건 (내용 확인용)

### 시스템/DB
- [ ] 관련 시스템 목록 (사내 시스템, 외부 API 등)
- [ ] 각 시스템의 접근 방식 (DB 직접 / REST API / 파일 등)
- [ ] DB 테이블 구조 또는 ERD (해당 시)
- [ ] 데이터 규모 (건수, 기간)

### 절차/매뉴얼
- [ ] 업무 절차서 목록 (처리 프로세스가 정해진 것)
- [ ] 각 절차의 단계 수, 판단 기준 유무

### 보안/접근
- [ ] 데이터 접근 제한 사항 (개인정보, 보안등급 등)
- [ ] 외부 반출 가능 여부
- [ ] 클라우드/온프레미스 제약
```

**톤 규칙:**
- ALFA, IRM, GraphRAG 등 내부 용어 절대 사용 금지
- 고객이 읽고 바로 행동할 수 있는 체크리스트 형태
- A(담당자용)와 B(IT팀용)를 분리하여 각각 전달 가능하게

## 주의 사항

- ALFA, IRM 등 프레임워크 용어는 **내부용**이므로 고객용 질문에 사용하지 않는다
- "행위 유형", "GraphRAG", "거버넌스 무게" 등은 내부 메모에만 기록
- 고객 대면 여부는 **반드시 초반에 확인** — 거버넌스 무게를 결정하는 핵심 트리거
- 업종이 불분명할 때는 범용 질문으로 시작하되, 업종 확인 질문을 최우선으로 배치
- 기존 폴더가 있으면 기존 프로필을 로드하여 이어간다 (새로 만들지 않음)
