# IRM Phase 1: 종합 요약 (Design Summary)

> /irm-search, /irm-access, /irm-exec 결과를 취합하여 종합 아키텍처와 일정을 정리한다.

## 참조 문서 (필수 — 건너뛰면 안 됨)

**아래 파일을 반드시 Read 도구로 읽은 후 종합한다. 읽지 않고 작성하면 안 된다.**

- `~/.claude/skills/irm-references/4.IRM_실행가이드_v10.md` — Phase 1 전체 흐름

## 선행 조건

아래 파일이 이미 존재해야 한다. 없으면 해당 스킬을 먼저 실행하라고 안내한다.

- `phase1_search.md` ← `/irm-search`에서 생성
- `phase1_access.md` ← `/irm-access`에서 생성
- `phase1_execution.md` + `phase1_chaining.md` ← `/irm-exec`에서 생성

## 핵심 원칙

- **이 스킬은 새로운 설계를 하지 않는다** — 이미 만들어진 3개 결과를 종합만 한다
- **신뢰도 전파**: 각 설계의 신뢰도를 종합하여 전체 설계 신뢰도를 산출한다

## 실행 흐름

`irm/[유즈케이스명]/phase1_search.md`, `phase1_access.md`, `phase1_execution.md`, `phase1_chaining.md`를 로드한다.

## 일정 추정 규칙

| 작업 | 기본 소요 | 리스크 가산 |
|------|----------|-----------|
| 인프라 (확장 설치) | 0.5~1주 | — |
| 컬렉션 구축 (1만건 이하) | 0.5주 | — |
| 컬렉션 구축 (1~10만건) | 1주 | — |
| 컬렉션 구축 (10만건 이상) | 1.5~2주 | 전처리 +1주 |
| GraphRAG 그래프 구축 | 2주 | 전문가 감수 +1주, 복잡도 높음 +1주 |
| CLI 1개 | 0.5주 | 인증 복잡 +0.5주 |
| LLM + Human-in-the-loop | 1주 | 판단 복잡도 높음 +0.5주 |
| 통합 테스트 | 0.5~1주 | 체이닝 3개 이상 +0.5주 |

## 주의 사항

- **이 스킬은 종합만 한다** — search/access/exec 결과를 취합
- 새로운 설계 결정을 하지 않음
- 일정은 기준표 + 리스크 가산 적용
- 다른 유즈케이스와 인프라 공유 가능 여부 반드시 확인
