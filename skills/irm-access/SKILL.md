# IRM 접근 배치 설계 (Access Deployment)

> Phase 0의 접근 유형 리소스를 기반으로, CLI + REST 게이트웨이를 설계한다.

## 참조 문서 (필수 — 건너뛰면 안 됨)

**아래 파일을 반드시 Read 도구로 읽은 후 설계를 시작한다. 읽지 않고 설계하면 안 된다.**

- `~/.claude/skills/irm-references/3.IRM_CLI_구성요소_런북_v10.md` — CLI 설계 패턴, REST 게이트웨이

## 핵심 원칙

- **시스템 단위 경계**: CLI는 시스템별로 분리
- **JSON 표준 출력**: 모든 CLI 출력은 JSON
- **신뢰도 전파**: Phase 0의 ✅/⚠️를 이어받는다

## 실행 흐름

### Step 0: Phase 0 결과 로드

- `irm/[유즈케이스명]/phase0_*.md` 로드
- 접근 유형 리소스만 추출
- 없으면 `/irm-init` 먼저 실행하라고 안내

### Step 1: 접근 설계

**파일:** `irm/[유즈케이스명]/phase1_access.md`

```markdown
# [유즈케이스명] — 접근 배치 설계

> 작성일: [날짜]
> 접근 질의: [N]건
> 참조: CLI 런북 v10

## 대상 시스템

| 1 | [이름] | DB/API/파일 | Y/N | ★★★ | ✅/⚠️ | [메모] |
```

## 주의 사항

- 이 스킬은 **접근(CLI+REST)만** 설계한다
- 탐색(RAG)은 `/irm-search`, 실행(파이프라인)은 `/irm-exec`에서 별도 설계
- CLI 우선순위: ★★★ = 즉시, ★★ = 운영 중, ★ = 온디맨드
- 규제 관련 제한(가명처리, 신용정보 비노출 등)은 Phase 2 GOVERN 참조
