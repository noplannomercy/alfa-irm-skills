# ALFA 미팅 실시간 어시스턴트

> 미팅 중 나온 내용을 프로필에 반영하고, 놓친 질문을 알려준다.

## 폴더 구조

이 스킬은 기존 고객 폴더에 미팅 기록을 추가한다.

```
[회사명]/
├── STATUS.md                    ← 업데이트
├── meeting/
│   ├── prep_질문리스트.md
│   ├── 1st_script.md            ← 미팅 메모 원문 (이 스킬이 생성)
│   ├── 2nd_script.md            ← 2차 호출 시
│   └── ...
└── alfa/
    └── profile.md               ← 업데이트
```

## 실행 흐름

### Step 1: 고객 폴더 탐색

기존 고객 폴더를 찾는다:
- `*/STATUS.md` 또는 `*/alfa/profile.md` 파일을 Glob으로 검색
- 여러 개면 AskUserQuestion으로 어떤 고객인지 확인
- 없으면 `/alfa-prep`을 먼저 실행하라고 안내

### Step 2: 현재 상태 확인

`STATUS.md`와 `alfa/profile.md`를 로드하여:
- 현재 몇 차 미팅인지 파악 (meeting/ 폴더의 기존 *_script.md 파일 수 기반)
- 프로필 채움 현황 확인
- 이번이 N차 미팅임을 결정

### Step 3: 미팅 내용 입력받기

AskUserQuestion으로:
```
[회사명] [N]차 미팅 내용을 입력해주세요.
(메모, 음성 요약, 대화 내용 등 — 자유 형식)
```

### Step 4: 미팅 메모 저장

입력받은 원문을 그대로 저장한다.

**파일:** `[회사명]/meeting/[N]st_script.md` (1st, 2nd, 3rd...)

```markdown
# [회사명] [N]차 미팅 메모

> 일시: [날짜]
> 참석자: [입력에서 추출 또는 "미기재"]

## 미팅 내용 (원문)
[입력받은 내용 그대로]
```

### Step 5: 프로필 업데이트

입력된 내용에서 6항목 관련 정보를 추출하여 `alfa/profile.md`를 업데이트한다.

**업데이트 규칙:**
- 기존 값이 있으면 **보강** (덮어쓰지 않음)
- 새로운 정보는 `[N차 미팅 확인]` 태그와 함께 추가
- 모순되는 정보는 `[확인 필요]` 태그로 표시

### Step 6: 신호 감지

미팅 내용에서 아래 신호를 자동 감지하여 **내부 메모**에 기록한다.

| 신호 | 고객 발언 패턴 | 태그 | 대응 |
|------|--------------|------|------|
| **GraphRAG 필요** | "표현이 달라서", "같은 건데 다르게", "용어가 통일 안 돼" | `#GraphRAG` | 내부 메모에 구체 예시 기록 |
| **온프레미스** | "외부 반출 금지", "클라우드 못 써" | `#on-premise` | 로컬 LLM 분기, GPU 확인 필요 |
| **거버넌스 격상** | "금감원", "고객 정보", "신용", "개인정보" | `#gov-heavy` | 2차 미팅 + 컴플라이언스팀 |
| **2단계 접근** | "고객 챗봇도", "직접 노출", "고객이 직접" | `#2-phase` | 1차 내부 → 2차 외부, 거버넌스 재검토 |
| **스폰서 미확정** | "위에서 아직", "결재 안 났어", "예산 미정" | `#sponsor-pending` | 진행 보류 가능성 플래그 |
| **PoC 실패 이력** | "전에 해봤는데", "잘 안 됐어", "업체가" | `#poc-history` | 실패 원인 파악, 남은 자산 확인 |

### Step 7: STATUS.md 업데이트

`STATUS.md`를 업데이트한다:
- 미팅 이력 테이블에 이번 미팅 추가
- Phase 진행 상태 업데이트 (미팅에서 확인된 내용 기반)
- 프로필 채움 현황 업데이트
- 감지 신호 업데이트
- "최종 업데이트" 날짜 갱신

### Step 8: 미확인 항목 알림 + 안내

**출력 형식:**

```markdown
## [N]차 미팅 프로필 업데이트 완료

### 이번에 채워진 항목
- [항목명]: [요약] ← [이전 상태] → 🟢

### 아직 확인 안 된 항목
⚠️ **[항목명]** — [왜 중요한지 한 줄]
  → 추천 질문: "[질문]"

### 감지된 신호
🔔 [신호명]: "[고객 발언 원문]"
  → [의미]: [내부 대응]

### 현재 프로필 채움 현황
| # | 항목 | 상태 |
|---|------|------|
| 1 | 회사 개요 | 🟢 |
| 2 | AI/LLM 현황 | 🟡 |
| ... | ... | ... |

### 거버넌스 무게 (현재 판단)
[Light / Medium / Heavy] — 근거: [...]
```

마지막에 안내:
```
추가 미팅 내용이 있으면 다시 /alfa-live를 호출하세요.
미팅이 끝났으면 /alfa-wrap으로 산출물을 생성합니다.
```

## 반복 사용

이 스킬은 **여러 번 호출**할 수 있다:
- 같은 미팅 내 추가 메모 → 같은 회차 script에 append
- 새 미팅 → 새 회차 script 생성
- 매번 프로필이 누적 업데이트됨
- 미팅 끝나면 `/alfa-wrap`으로 전환

## 주의 사항

- **이 스킬은 script 저장 + 프로필 업데이트 + STATUS 업데이트만 한다**
- **prep_질문리스트.md는 /alfa-prep에서 이미 생성됨 — 이 스킬에서 생성하지 않는다**
- **summary, internal, Phase 문서 등은 /alfa-wrap에서 생성 — 이 스킬에서 생성하지 않는다**
- ALFA/IRM 용어는 출력하지 않음 — 고객이 화면을 볼 수 있음
- 신호 감지 결과는 "내부 메모" 섹션에만 기록
- 추천 질문은 고객 앞에서 자연스럽게 물어볼 수 있는 톤으로 작성
- 모순 정보 발견 시 즉시 `[확인 필요]`로 플래그
- 미팅 회차는 meeting/ 폴더의 기존 script 파일 수를 세어 자동 결정
