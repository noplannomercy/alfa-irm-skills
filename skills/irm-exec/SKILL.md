# IRM 실행 + 체이닝 설계 (Execution + Chaining)

> Phase 0의 실행 유형 리소스와 체이닝 패턴을 기반으로, 파이프라인과 체이닝 플로우를 설계한다.

## 참조 문서 (필수 — 건너뛰면 안 됨)

**아래 파일을 반드시 Read 도구로 읽은 후 설계를 시작한다. 읽지 않고 설계하면 안 된다.**

- `~/.claude/skills/irm-references/4.IRM_실행가이드_v10.md` — 실행(확정/유연) 설계, 체이닝 관계

## 핵심 원칙

- **실행(확정)**: 같은 입력 → 같은 결과 (결정적 파이프라인)
- **실행(유연)**: LLM 판단 + Human-in-the-loop (보조수단성)
- **고정화 조건**: 실행(유연)에서 패턴 안정 시 실행(확정)으로 전환
- **신뢰도 전파**: Phase 0의 ✅/⚠️를 이어받는다

## 실행 흐름

### Step 0: Phase 0 결과 로드

- `irm/[유즈케이스명]/phase0_*.md` 로드
- 실행 유형 리소스 + 체이닝 패턴 추출
- 없으면 `/irm-init` 먼저 실행하라고 안내

### Step 1: 실행 설계

**파일:** `irm/[유즈케이스명]/phase1_execution.md`

```markdown
# [유즈케이스명] — 실행 배치 설계

> 작성일: [날짜]
> 참조: IRM 실행가이드 v10

## 실행(확정) — 결정적 파이프라인

### [절차명] 파이프라인
Step 1: [접근] [시스템] → [입력 데이터 획득]
Step 2: [탐색] [컬렉션] → [기준/규정 조회]
Step 3: [판단] [규칙 기반] → [결과]
Step 4: [기록] [시스템] → [저장]

- 입력: [무엇]
- 출력: [무엇]
- 성공/실패 경계: [각 Step별]

## 실행(유연) — 판단 기반 프로세스

### [절차명] 프로세스
Step 1: [접근] 데이터 수집
Step 2: [탐색] 유사 이력/기준 조회
Step 3: [LLM 판단] 분석/추천
Step 4: [사람 확인] 최종 결정

- 판단 포인트: Step [N]
- Human-in-the-loop: Step [N]
- 고정화 조건: [언제 실행(확정)으로 전환]
```

### Step 2: 체이닝 설계

**파일:** `irm/[유즈케이스명]/phase1_chaining.md`

```markdown
# [유즈케이스명] — 체이닝 설계

> 작성일: [날짜]

## 체이닝 패턴

| # | 질의 | 체이닝 순서 | GraphRAG | 신뢰도 |
|---|------|-----------|:--------:|:------:|
| 1 | "[질의]" | 접근→탐색→실행 | Y/N | ✅/⚠️ |

## 체이닝 플로우 상세

### 패턴 1: [패턴명]
[질의] →
  ① [행위유형]: [상세]
  ② [행위유형]: [상세]
  ③ [행위유형]: [상세]
```

## 주의 사항

- 이 스킬은 **실행(확정/유연) + 체이닝만** 설계한다
- 탐색(RAG)은 `/irm-search`, 접근(CLI)은 `/irm-access`에서 별도 설계
- 체이닝은 `/irm-search`와 `/irm-access` 결과를 참조하여 연결
- 실행(유연)에는 반드시 Human-in-the-loop 위치를 명시
- 고정화 조건 명시 (동일 판단 N회 연속 → 규칙화 검토)
